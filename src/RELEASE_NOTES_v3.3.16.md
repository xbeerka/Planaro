# Release Notes v3.3.16 - Исправление залипания флага Undo/Redo

## Критическое исправление: Бесконечный цикл блокировки

### Проблема

После исправления в первой версии v3.3.16 проблема залипания флага `isUndoRedoInProgressRef` осталась, но проявлялась иначе:

**Сценарий**:
1. Пользователь нажимает Ctrl+Z
2. Флаг `isUndoRedoInProgressRef` установлен в `true`
3. Происходит ошибка → флаг НЕ сбрасывается
4. Пользователь нажимает Ctrl+Z снова
5. Обнаруживается залипший флаг → сбрасывается в `false`
6. Код **продолжает выполнение** → на строке 439 флаг **снова устанавливается в `true`**
7. При следующей попытке Ctrl+Z → флаг снова `true` → цикл повторяется

**Логи**:
```
⏸️ UNDO/REDO: Undo уже выполняется, ожидайте завершения...
⚠️ DEBUG: Флаг isUndoRedoInProgressRef не был сброшен!
⚠️ Принудительный сброс флага...
(повторяется при каждой попытке)
```

### Причина

**Отсутствие `return` после сброса флага**:

```typescript
// ❌ НЕПРАВИЛЬНО (v3.3.16 первая версия)
if (isUndoRedoInProgressRef.current) {
  console.warn('⚠️ Принудительный сброс флага...');
  isUndoRedoInProgressRef.current = false;
  // Код продолжает выполняться
}

const state = historyUndo();
if (!state) return;

isUndoRedoInProgressRef.current = true; // ← СНОВА устанавливаем в true!
```

→ Флаг сбрасывается, но сразу же устанавливается обратно в `true` → **бесконечный цикл**

### Решение

**Добавить `return` после сброса флага**:

```typescript
// ✅ ПРАВИЛЬНО (v3.3.16 финальная версия)
if (isUndoRedoInProgressRef.current) {
  console.warn('⚠️ Принудительный сброс флага...');
  isUndoRedoInProgressRef.current = false;
  console.log('✅ Флаг сброшен. Повторите попытку.');
  
  queueMicrotask(() => {
    showToast({
      type: 'info',
      message: 'Повторите попытку',
      description: 'Предыдущая операция была сброшена'
    });
  });
  
  return; // ✅ КРИТИЧНО: Выходим из функции!
}

// Эта часть НЕ выполняется при залипшем флаге
const state = historyUndo();
if (!state) return;

isUndoRedoInProgressRef.current = true; // ← Выполняется только при нормальном flow
```

→ При обнаружении залипшего флага:
1. Флаг сбрасывается в `false`
2. Показывается toast уведомление
3. Функция завершается с `return`
4. При следующей попытке (повторный Ctrl+Z) флаг уже `false` → Undo выполняется

## Почему это работает

### Без `return` (НЕ РАБОТАЕТ)

```
[Цикл 1]
Флаг: true → сбрасываем в false → продолжаем → устанавливаем в true

[Цикл 2] 
Флаг: true → сбрасываем в false → продолжаем → устанавливаем в true

[Цикл 3]
...бесконечный цикл
```

### С `return` (РАБОТАЕТ)

```
[Попытка 1 - залипание]
Флаг: true → сбрасываем в false → return → выход

[Попытка 2 - нормальный flow]
Флаг: false → проверка пройдена → устанавливаем в true → Undo → finally: сбрасываем в false

[Попытка 3 - нормальный flow]
Флаг: false → проверка пройдена → ...
```
