## 🔍 Диагностика проблем

### Проблема: Gaps НЕ убираются

**Проверка 1**: Включен ли DEBUG?

**Проверка 2**: Есть ли логи RULE 1.5 в консоли?
- Если НЕТ → покрытие неполное (проверьте высоту соседей)
- Если ДА → проверьте стили в DevTools

### Проблема: Gaps убираются когда НЕ должны

**Причина**: Покрытие определяется неправильно

**Проверка**:
```
// Логи:
📏 RULE 1.5: e... expandLeft=1 (other-project coverage)

// Но соседи покрывают ЧАСТИЧНО (например, 0-50%, а событие 0-100%)
```

**Решение**:
- Проверьте `analyzeOtherProjectCoverage()` в `/utils/eventNeighbors.ts:249`
- Убедитесь что проверяются ОБА условия:
  ```typescript
  return hasTopCovered && hasBottomCovered; // ← AND, не OR!
  ```

## 📝 Примечания

### Порядок применения правил (STAGE 3)
1. **RULE 1**: Base horizontal glue (same-project) → `expandLeft/Right = 1`
2. **RULE 1.5**: Vertical coverage (other-project) → `expandLeft/Right = 1` (ТОЛЬКО если нет same-project)
3. **RULE 2**: Stacking А/Б/В/Г → может менять expand соседей
4. **RULE 3**: Biting → может уменьшать expand на -1

### Приоритет
- Same-project всегда выигрывает (RULE 1 > RULE 1.5)
- Проверка `!geometry.left.hasInnerConnection` гарантирует отсутствие конфликта
