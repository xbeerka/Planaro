# 🛡️ Обработка Cloudflare ошибок

**Дата**: 2025-11-18  
**Версия**: v3.4.0-clean  
**Проблема**: Cloudflare Error 1105: Temporarily unavailable

---

## 🚨 Проблема

Supabase база данных защищена **Cloudflare CDN**, который иногда возвращает **Error 1105**:

```
Error 1105: Temporarily unavailable
The page could not be rendered due to a temporary fault.
```

**Причины**:
- Временная перегрузка базы данных
- Техническое обслуживание Supabase
- Проблема с Cloudflare CDN
- Rate limiting

**Проблемы ДО исправления**:
- ❌ Огромные HTML ошибки в логах (весь Cloudflare error page)
- ❌ Пользователь видит непонятные технические ошибки
- ❌ Нет повторных попыток
- ❌ Приложение может "зависнуть"

---

## ✅ Решение

### 1. Автоматический Retry с экспоненциальной задержкой

**Файл**: `/services/api/base.ts`

**Что добавили**:
- `retries: 2` - по умолчанию 2 повторных попытки
- `retryDelay: 1000` - базовая задержка 1 секунда
- Экспоненциальная задержка: 1s → 2s → 4s

**Как работает**:
```typescript
// Попытка 1: сразу
// Попытка 2: через 1 секунду (2^0 * 1000ms)
// Попытка 3: через 2 секунды (2^1 * 1000ms)
// Всего: 3 попытки за ~3 секунды
```

**Код**:
```typescript
export interface ApiOptions {
  retries?: number; // По умолчанию 2
  retryDelay?: number; // По умолчанию 1000ms
}

for (let attempt = 0; attempt <= retries; attempt++) {
  try {
    // ... делаем запрос
    
    if (cloudflareError && attempt < retries) {
      const delayMs = retryDelay * Math.pow(2, attempt); // Экспоненциально
      console.log(`⏳ Повтор через ${delayMs}ms...`);
      await delay(delayMs);
      continue; // Retry
    }
    
  } catch (error) {
    // ... обработка
  }
}
```

### 3. Toast уведомления для пользователя

**Файл**: `/utils/cloudflareErrorHandler.ts`

**Функция**: `handleCloudflareError(error, showToast)`

**Что делает**:
- Проверяет является ли ошибка Cloudflare Error 1105
- Показывает **один toast** вместо спама
- Сбрасывает флаг через 60 секунд

**Логика**:
```typescript
let cloudflareErrorShown = false;

export function handleCloudflareError(error, showToast) {
  if (!isCloudflareError(error)) return;
  
  // Показываем toast только ОДИН РАЗ
  if (!cloudflareErrorShown && showToast) {
    showToast('База данных временно недоступна. Повторная попытка...', 'error');
    cloudflareErrorShown = true;
    
    // Сбрасываем через 60 секунд
    setTimeout(() => {
      cloudflareErrorShown = false;
    }, 60000);
  }
  
  // Короткое логирование
  console.error('❌ Cloudflare Error 1105: Database temporarily unavailable');
}
```

**Результат**:
- ✅ Пользователь видит: "База данных временно недоступна. Повторная попытка..."
- ✅ Показывается **только один toast**, не спамит
- ✅ Через 60 секунд может показаться снова (если проблема не решилась)

## 📊 Как это работает

### Сценарий 1: Cloudflare Error 1105

**Что происходит**:
1. Пользователь открывает календарь
2. Начинается синхронизация (Delta Sync, Projects, Resources)
3. Cloudflare возвращает Error 1105
4. **Автоматически** делается 2 повторных попытки (1s, 2s)
5. Если не помогло - показывается toast **один раз**
6. Через 4 секунды следующий Delta Sync попробует снова

**Логи** (короткие):
```
❌ Cloudflare Error 1105: Temporarily unavailable (попытка 1/3)
⏳ Повтор через 1000ms...
❌ Cloudflare Error 1105: Temporarily unavailable (попытка 2/3)
⏳ Повтор через 2000ms...
❌ Cloudflare Error 1105: Temporarily unavailable (попытка 3/3)
❌ Cloudflare Error 1105: Database temporarily unavailable
```

**Toast** (для пользователя):
```
🔴 База данных временно недоступна. Повторная попытка...
```

---

### Сценарий 2: Временная недоступность (30 секунд)

**Что происходит**:
1. Cloudflare Error 1105 в 15:00:00
2. Toast показался **один раз**: "База данных временно недоступна"
3. Delta Sync продолжает пытаться каждые 4 секунды (в фоне)
4. В 15:00:30 база данных восстановилась
5. Delta Sync успешно загружает данные
6. **НЕТ** новых toast (флаг не сброшен)

**Результат**: Пользователь не видит спам, приложение восстанавливается автоматически

---

### Сценарий 3: Длительная недоступность (>60 секунд)

**Что происходит**:
1. Cloudflare Error 1105 в 15:00:00
2. Toast показался: "База данных временно недоступна"
3. Delta Sync продолжает пытаться каждые 4 секунды
4. Через 60 секунд флаг сбрасывается
5. Если база всё ещё недоступна - показывается новый toast

**Результат**: Пользователь знает что проблема не решена

## 📝 Технические детали

### Retry стратегия

**Exponential Backoff**:
- Попытка 1: сразу
- Попытка 2: через 1s (2^0 * 1000ms)
- Попытка 3: через 2s (2^1 * 1000ms)

**Почему экспоненциальная**:
- Даём время серверу восстановиться
- Не перегружаем сервер частыми запросами
- Стандартная практика для retry логики

### Toast throttling

**Один toast на 60 секунд**:
- Не спамим пользователя
- Если проблема решилась за 60 сек - пользователь не видит повторных уведомлений
- Если не решилась - показываем новый toast

### Типы ошибок

**Cloudflare Error 1105** - retry enabled:
- Автоматически повторяем запрос
- Показываем toast

**Другие ошибки** - retry disabled:
- HTTP 401 → выход из системы
- HTTP 404 → обычная ошибка
- HTTP 500 → ошибка сервера (не Cloudflare)

## 🧪 Тестирование

### Проверка retry логики

1. Откройте консоль браузера
2. Найдите сообщения:
   ```
   ❌ Cloudflare Error 1105 (попытка 1/3)
   ⏳ Повтор через 1000ms...
   ```
3. Убедитесь что делается **3 попытки**

### Проверка toast

1. Дождитесь Cloudflare ошибки
2. Должен показаться **ОДИН** toast
3. Следующие ошибки (в течение 60 сек) **НЕ** показывают toast

### Проверка логов

1. Откройте консоль
2. Логи должны быть **короткими**:
   ```
   ❌ Cloudflare Error 1105: Database temporarily unavailable
   ```
3. **НЕТ** огромных HTML блоков
