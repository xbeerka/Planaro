# 🧪 Тестовые сценарии для проверки Undo/Redo

## Цель
Проверить что Undo/Redo НЕ удаляет все события из воркспейса.

---

## Сценарий 1: Простой Undo/Redo

1. ✅ Открой календарь воркспейса
2. ✅ Убедись что события отображаются (запиши количество: `_____`)
3. ✅ Создай новое событие (drag из топ-панели)
4. ✅ Нажми **Cmd+Z (Undo)**
5. ✅ Проверь: событие удалилось, остальные на месте? ⬜ ДА / ⬜ НЕТ
6. ✅ Нажми **Cmd+Shift+Z (Redo)**
7. ✅ Проверь: событие вернулось, остальные на месте? ⬜ ДА / ⬜ НЕТ

**Ожидаемый результат:**
- ✅ Все события остались на месте
- ✅ Только созданное событие удалилось/вернулось

---

## Сценарий 2: Drag & Drop + Undo/Redo

1. ✅ Открой календарь воркспейса
2. ✅ Перетащи событие на новую позицию (drag)
3. ✅ Нажми **Cmd+Z (Undo)**
4. ✅ Проверь: событие вернулось на место, остальные на месте? ⬜ ДА / ⬜ НЕТ
5. ✅ Нажми **Cmd+Shift+Z (Redo)**
6. ✅ Проверь: событие снова на новой позиции, остальные на месте? ⬜ ДА / ⬜ НЕТ

**Ожидаемый результат:**
- ✅ Все события остались на месте
- ✅ Только перемещённое событие изменило позицию

---

## Сценарий 3: Множественные операции + Undo/Redo

1. ✅ Открой календарь воркспейса
2. ✅ Создай 3 новых события
3. ✅ Перетащи 2 события на новые позиции
4. ✅ Удали 1 событие
5. ✅ Нажми **Cmd+Z (Undo)** 5 раз подряд
6. ✅ Проверь: все изменения откатились, старые события на месте? ⬜ ДА / ⬜ НЕТ
7. ✅ Нажми **Cmd+Shift+Z (Redo)** 5 раз подряд
8. ✅ Проверь: все изменения вернулись, старые события на месте? ⬜ ДА / ⬜ НЕТ

**Ожидаемый результат:**
- ✅ Все старые события остались на месте
- ✅ Только новые/изменённые события откатились/вернулись

---

## Сценарий 4: Undo/Redo + Full Sync (30 сек ожидания)

1. ✅ Открой календарь воркспейса
2. ✅ Создай новое событие
3. ✅ Нажми **Cmd+Z (Undo)** - событие удалилось
4. ✅ Жди **30 секунд** (должен сработать Full Sync)
5. ✅ Проверь: все события на месте, удалённое НЕ вернулось? ⬜ ДА / ⬜ НЕТ
6. ✅ Нажми **Cmd+Shift+Z (Redo)** - событие вернулось
7. ✅ Жди **30 секунд** (должен сработать Full Sync)
8. ✅ Проверь: все события на месте, включая восстановленное? ⬜ ДА / ⬜ НЕТ

**Ожидаемый результат:**
- ✅ Full Sync НЕ вернул удалённое событие после Undo
- ✅ Full Sync НЕ удалил восстановленное событие после Redo
- ✅ Все старые события остались на месте

---

## Сценарий 5: Быстрые Undo/Redo (стресс-тест)

1. ✅ Открой календарь воркспейса
2. ✅ Создай событие
3. ✅ Нажми **Cmd+Z / Cmd+Shift+Z** 20 раз подряд (быстро!)
4. ✅ Проверь: все события на месте? ⬜ ДА / ⬜ НЕТ
5. ✅ Проверь логи в консоли: есть ошибки? ⬜ ДА / ⬜ НЕТ

**Ожидаемый результат:**
- ✅ Все события остались на месте
- ✅ Нет ошибок в консоли
- ✅ Блокировка синхронизации работает (логи `⏸️ Сброс таймера`)

---

## Сценарий 6: Undo/Redo после изменения высоты события

1. ✅ Открой календарь воркспейса
2. ✅ Измени высоту события (resize вертикальный)
3. ✅ Нажми **Cmd+Z (Undo)**
4. ✅ Проверь: высота вернулась, остальные события на месте? ⬜ ДА / ⬜ НЕТ
5. ✅ Жди **30 секунд** (Full Sync)
6. ✅ Проверь: высота НЕ изменилась обратно? ⬜ ДА / ⬜ НЕТ
7. ✅ Нажми **Cmd+Shift+Z (Redo)**
8. ✅ Жди **30 секунд** (Full Sync)
9. ✅ Проверь: высота НЕ вернулась обратно? ⬜ ДА / ⬜ НЕТ

**Ожидаемый результат:**
- ✅ Full Sync НЕ перезаписывает измененную высоту после Undo/Redo
- ✅ `syncRestoredEventsToServer` синхронизировала UPDATE операцию
- ✅ Все старые события остались на месте

## Возможные причины бага (если он всё ещё есть)

### 1. История не сохраняется
**Симптом:** `Восстановлено 0 событий`
**Причина:** `saveHistory()` не вызывается или блокируется
**Проверка:** Поищи в логах `💾 Сохранение в историю`

### 2. История сохраняется без событий
**Симптом:** События исчезают после Undo
**Причина:** `saveHistory()` вызван с пустым массивом событий
**Проверка:** В `/hooks/useHistory.ts` есть защита, но может быть race condition

### 3. Full Sync перезаписывает восстановленные события
**Симптом:** События исчезают через 30 секунд после Redo
**Причина:** `syncRestoredEventsToServer` не успела синхронизировать
**Проверка:** Должен быть лог `✅ События успешно синхронизированы с сервером`

### 4. Orphaned events cleanup удаляет события
**Симптом:** События исчезают через 5-10 секунд после Undo/Redo
**Причина:** Блокировка `lastUndoRedoTimeRef` не работает
**Проверка:** Должен быть лог `🛡️ Блокировка orphaned cleanup на 5 секунд`
