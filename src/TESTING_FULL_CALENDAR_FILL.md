# 🧪 Тестирование: Полное заполнение календаря (v2.3)

## ✅ Что изменилось

### Старая версия (v2.2)
- Максимум **10 событий** на сотрудника
- Заполнение **до 100%**, но с ограничением по количеству событий
- Могли оставаться **пустые ячейки** если достигнут лимит 10 событий

### Новая версия (v2.3)
- От **1 до 12 событий** на сотрудника (случайное количество)
- **100% заполнение БЕЗ ПУСТОТ** - все 208 ячеек (52 недели × 4 юнита) заполнены
- Гарантия: **НИ ОДНОЙ пустой ячейки**

## 🎯 Алгоритм заполнения

### 1. Целевое количество событий
Для каждого сотрудника генерируется случайное количество событий:
```typescript
const targetEventCount = Math.floor(Math.random() * 12) + 1; // 1-12 событий
```

### 2. Прогрессивное заполнение
```
Пока есть свободные ячейки:
  1. Найти первую свободную ячейку (слева-направо, сверху-вниз)
  2. Определить максимальные размеры события от этой позиции
  3. ЕСЛИ это последнее событие (достигнут лимит):
       → Создать максимальный прямоугольник (заполнить ВСЁ оставшееся)
     ИНАЧЕ:
       → Создать событие случайного размера:
          * 60% - короткое (1-6 недель)
          * 40% - среднее/длинное (7+ недель)
          * 70% - высота 1 юнит
          * 30% - высота 2-4 юнита
  4. Пометить ячейки как занятые
  5. Перейти к следующей свободной ячейке
```

### 3. Гарантия заполнения
Последнее событие (или когда достигнут `targetEventCount - 1`) создаётся **максимального размера**:
```typescript
if (createdCount >= targetEventCount - 1) {
  width = maxWidth;   // Всё доступное по горизонтали
  height = maxHeight; // Всё доступное по вертикали
}
```

Это гарантирует что ВСЕ оставшиеся ячейки будут заполнены.

### Сотрудник A: 1 событие
```
Week: 0                                                     51

Unit 1: [========================================]
Unit 2: [========================================]
Unit 3: [========================================]
```
Одно большое событие на все 52 недели и все 4 юнита.

### Сотрудник B: 4 события
```

Unit 2: [Event 1    ][Event 2     ][Event 3    ][Event 4  ]
Unit 3: [Event 1    ][Event 2     ][Event 3    ][Event 4  ]
```
Четыре вертикальные полосы, каждая высотой 4 юнита.

### Сотрудник C: 12 событий (шахматный порядок)
```
Week: 0  4  8 12 16 20 24 28 32 36 40 44 48              51
Unit 0: [1][2][3][4][5][6]............................[12]
Unit 1: [1][2][3][4][5][6]............................[12]

```
После 11 событий остаются пробелы - 12-е событие заполняет ВСЁ оставшееся.

## 🧮 Статистика

### Общие цифры
- **Всего ячеек на сотрудника**: 52 × 4 = **208 ячеек**
- **Количество событий**: от **1 до 12** (случайно)
- **Заполнение**: всегда **100%** (208/208 ячеек)

### Средние размеры событий
- **При 1 событии**: 208 ячеек = 52 недели × 4 юнита
- **При 12 событиях**: ~17 ячеек на событие (например, 4 недели × 4 юнита)
- **Минимальный размер события**: 1 неделя × 1 юнит = 1 ячейка
- **Максимальный размер события**: 52 недели × 4 юнита = 208 ячеек

## ✅ Как проверить

### 1. Генерация тестовых данных
1. Открыть календарь воркспейса
2. Нажать кнопку **"Генерация тестовых данных"** в тулбаре
3. Подтвердить действие

### 2. Проверка в консоли
Для каждого сотрудника будет лог:
```
✅ Иванов Иван: создано 7 событий, заполнено 208/208 ячеек (100.0%)
```

### 3. Визуальная проверка
- Прокрутить календарь по всем 52 неделям
- Проверить что **нет пустых ячеек** (серых промежутков)
- Каждый сотрудник должен иметь **сплошное цветное заполнение** всех 4 юнитов

### 4. Проверка разнообразия
- Разные сотрудники имеют **разное количество событий** (1-12)
- События имеют **разные размеры** (короткие, средние, длинные)
- События имеют **разную высоту** (1-4 юнита)

## ⚠️ Важные детали

### Последнее событие - "заполнитель"
Когда достигнут целевой лимит событий (`targetEventCount - 1`), последнее событие создаётся максимального размера чтобы заполнить **ВСЕ** оставшиеся пробелы.

Пример:
```typescript
// Цель: 5 событий
// События 1-4: случайные размеры (могут оставить пробелы)
// Событие 5: МАКСИМАЛЬНЫЙ прямоугольник (заполняет ВСЕ оставшееся)
```

### Защита от бесконечного цикла
```typescript
const maxIterations = 1000;
```
Если алгоритм не может заполнить календарь за 1000 итераций - прерывается и логирует предупреждение.

### Минимальный размер события
События меньше 2 ячеек автоматически расширяются:
```typescript
if (width * height < 2 && maxWidth * maxHeight >= 2) {
  width = Math.min(2, maxWidth);
  height = 1;
}
```

Это делает события более заметными и красивыми.
