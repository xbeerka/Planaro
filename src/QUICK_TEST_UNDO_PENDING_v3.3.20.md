## 🎯 Что тестируем

Исправление двух багов системы Undo/Redo:
1. ✅ Undo блокируется когда событие грузится (симметрия с Redo)
2. ✅ Redo разблокируется после того как событие загрузилось (обновление истории)

## 🐛 Известные проблемы (должны быть исправлены)

### ❌ Баг 1: Undo НЕ блокируется при pending событиях
**Статус:** ✅ Исправлено в v3.3.20

**Как проявлялся:**
- Создал событие → нажал Ctrl+Z сразу → Undo сработал
- Событие удалилось → createEvent завершился → событие "воскресло"

**Исправление:**
- Добавлена проверка `hasPendingEvents` в `handleUndo`
- Симметрия с `handleRedo`

---

### ❌ Баг 2: Redo блокируется после загрузки
**Статус:** ✅ Исправлено в v3.3.20

**Как проявлялся:**
- Создал событие → дождался загрузки → Ctrl+Z → Ctrl+Y
- Toast: "Подождите - Дождитесь завершения создания событий"
- Redo заблокирован навсегда

**Причина:**
- История НЕ обновлялась после создания на сервере
- Временный ID оставался в истории
- Проверка видела временный ID → блокировала Redo

**Исправление:**
- Обновление ID в истории в `syncRestoredEventsToServer`
- Временный ID заменяется на реальный ID во ВСЕЙ истории

---

## ✅ Критерии успеха

1. ✅ **Undo блокируется** при pending событиях (toast показан)
2. ✅ **Redo НЕ блокируется** после загрузки события
3. ✅ **История обновляется** с реальными ID после создания
4. ✅ **События не "воскресают"** из БД

2. Найди `syncRestoredEventsToServer`
3. Убедись что есть обновление истории:
   ```typescript
   if (updateHistoryEventId && results.created.length > 0) {
     const tempToRealIdMap = new Map<string, string>();
     eventsToCreate.forEach((tempEvent, index) => {
       const createdEvent = results.created[index];
       if (createdEvent) {
         tempToRealIdMap.set(tempEvent.id, createdEvent.id);
       }
     });
     tempToRealIdMap.forEach((realId, tempId) => {
       updateHistoryEventId(tempId, realId);
     });
   }
   ```
