# 🔧 Исправление: Лимит загрузки событий (PostgREST 1000 rows)

**Дата**: 21 октября 2025  
**Версия**: 2.3.7 - ПРАВИЛЬНАЯ ПАГИНАЦИЯ  
**Проблема**: Создано 8053 событий, но загружено только 1000

## 📊 Проблема

При генерации большого количества тестовых событий (например, 8053 событий):
- ✅ Batch создание успешно создаёт все события в БД
- ❌ GET `/events` endpoint возвращает только 1000 событий (лимит PostgREST по умолчанию)
- ❌ Разница: 7053 событий потеряны при загрузке

### Причина

**PostgREST ВСЕГДА возвращает максимум 1000 записей за один запрос**, независимо от указанного лимита.

- `.limit(50000)` → вернёт 1000 записей
- `.range(0, 49999)` → вернёт 1000 записей
- **Решение**: цикл с пагинацией по 1000 записей

### Код до исправления (НЕПРАВИЛЬНО)

```typescript
// ❌ НЕПРАВИЛЬНО - вернёт только 1000 событий
const { data: events, error } = await query
  .order('id', { ascending: true })
  .range(0, 49999); // ← PostgREST всё равно вернёт max 1000!
```

## ✅ Решение: ПАГИНАЦИЯ

Загружаем события циклом порциями по 1000 записей:

```typescript
// ✅ ПРАВИЛЬНО - загружаем ВСЕ события через пагинацию
const PAGE_SIZE = 1000;
const allEvents: any[] = [];
let currentPage = 0;
let hasMore = true;

while (hasMore) {
  const rangeStart = currentPage * PAGE_SIZE;
  const rangeEnd = rangeStart + PAGE_SIZE - 1;
  
  let query = supabase
    .from('events')
    .select('*, event_patterns(name, pattern)')
    .order('id', { ascending: true })
    .range(rangeStart, rangeEnd);
  
  if (workspaceId) {
    query = query.eq('workspace_id', workspaceId);
  }
  
  const { data: pageEvents, error } = await query;
  
  if (error) {
    return c.json({ error: error.message }, 500);
  }
  
  if (pageEvents && pageEvents.length > 0) {
    allEvents.push(...pageEvents);
  }
  
  // Если получили меньше чем PAGE_SIZE - это последняя страница
  hasMore = pageEvents.length === PAGE_SIZE;
  currentPage++;
  
  // Защита от бесконечного цикла (макс 100,000 событий)
  if (currentPage >= 100) break;
}

console.log(`✅ Загружено ${allEvents.length} событий за ${currentPage} запросов`);
```

### Как это работает?

1. **Страница 1**: `.range(0, 999)` → 1000 событий → `hasMore = true`
2. **Страница 2**: `.range(1000, 1999)` → 1000 событий → `hasMore = true`
3. **...**
4. **Страница 9**: `.range(8000, 8999)` → 53 события → `hasMore = false` (последняя)
5. **Итого**: 8053 события за 9 запросов

## ⚠️ Известные ограничения

1. **Максимум 100,000 событий** на воркспейс (защита от бесконечного цикла)
   - Для календаря с 100 сотрудниками это ~1000 событий на сотрудника
   - Достаточно для любого реалистичного сценария
   - Можно увеличить лимит страниц если нужно больше

2. **Batch размер 50 событий** (серверный лимит для создания)
   - Генерация 10,000 событий = 200 batch запросов
   - Каждый batch занимает ~200-500ms
   - Общее время: ~40-100 секунд

3. **PostgREST пагинация**
   - Загрузка по 1000 событий за запрос
   - 10,000 событий = 10 запросов (~2-5 секунд)
   - Автоматически останавливается когда событий меньше 1000
