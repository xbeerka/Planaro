# ✅ Gap Handles v1.5.0 - Ready to Test!

## 🚀 Что добавлено

**Двусторонний resize границ между событиями** - при зажатой **Cmd/Ctrl** можно перетаскивать промежутки между касающимися событиями, изменяя **оба события одновременно**.

---

## 🎯 Быстрый старт

1. **Открой календарь воркспейса**
2. **Создай 2 события которые касаются**:
   - Event A: unit 0-2
   - Event B: unit 2-4 (касается Event A снизу)
3. **Зажми Cmd/Ctrl** клавишу
4. **Наведи курсор на промежуток** между событиями
5. **Появится синяя пипка** с 3 кружками
6. **Перетащи пипку** вверх или вниз
7. **Оба события изменятся** одновременно! 🎉

## 🎨 Примеры использования

### Вертикальный gap (сверху-снизу)

```
До drag:
[Event A: 0-2 units]  ← касается Event B
[Event B: 2-4 units]

Зажать Cmd → drag handle вверх на 1 unit:

После drag:
[Event A: 0-1 units]  ← уменьшилось на 1
[Event B: 1-4 units]  ← сдвинулось на -1, увеличилось на 1
```

### Горизонтальный gap (слева-справа)

```
До drag:
[Event C: week 0-5]  [Event D: week 5-10]
                     ↑ касаются

Зажать Cmd → drag handle влево на 2 weeks:

После drag:
[Event C: week 0-3]  [Event D: week 3-10]
                     ↑ граница сдвинулась
```

## 🔑 Ключевые особенности

### ✨ Умный поиск gaps

- Вертикальные: события на одном ресурсе, одной неделе, касаются по вертикали
- Горизонтальные: события на одном ресурсе, касаются по горизонтали, перекрываются по вертикали
- Один gap на пару событий (нет дубликатов)

### 🎯 Визуальные handles

- Синие пипки с 3 кружками
- Растягиваются на всю ширину/высоту пересечения событий
- Появляются **только** при зажатой Cmd/Ctrl
- z-index: 200 (выше событий)

### 🛡️ Валидация

- События не могут исчезнуть (минимум 1 unit / 1 week)
- События не могут выйти за пределы сетки (0-15 units, 0-51 weeks)
- Невалидные изменения игнорируются

### 🔄 Интеграция

- ✅ Undo/Redo работает
- ✅ Polling блокируется на 2 секунды
- ✅ Изменения сохраняются на сервер параллельно
- ✅ История обновляется

## 📊 Производительность

- **Поиск gaps**: `O(n log n)` где n - количество событий
- **Рендер handles**: `O(g)` где g - количество gaps (обычно < 50)
- **Оптимизация**: Gaps вычисляются только при `isCtrlPressed = true`

## 🐛 Troubleshooting

### Gap handle не появляется?

- Проверь что Cmd/Ctrl зажата
- Проверь что события **касаются** (нет промежутка между ними)
- Проверь что события на **одном ресурсе**
- Вертикальный gap: события должны быть на **одной неделе** (или пересекаться)
- Горизонтальный gap: события должны **перекрываться по вертикали**

### Drag не работает?

- Проверь консоль браузера на ошибки
- Проверь что курсор изменился на `ns-resize` или `ew-resize`
- Проверь логи: `🎯 Gap drag начат`

### Изменения откатились?

- Проверь валидацию: событие не может быть < 1 unit/week
- Проверь что событие не выходит за пределы (unit 0-15, week 0-51)
