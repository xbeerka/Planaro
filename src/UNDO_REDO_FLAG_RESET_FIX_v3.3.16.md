# Исправление сброса флага isUndoRedoInProgressRef (v3.3.16)

## Проблема

**Симптом**: При попытке выполнить Undo/Redo появляется ошибка:
```
⏸️ UNDO/REDO: Undo уже выполняется, ожидайте завершения...
```

И Undo/Redo перестаёт работать навсегда (требует перезагрузки страницы).

## Причина

Флаг `isUndoRedoInProgressRef.current` устанавливается в `true` в начале операции Undo/Redo, но **НЕ сбрасывается** в `false` после завершения операции.

### Возможные причины несброса флага:

1. **Ошибка в коде ПЕРЕД try блоком** (между строками 436-439)
   - Флаг устанавливается на строке 436
   - `try` начинается на строке 439
   - Если ошибка происходит на строках 437-438, `finally` НЕ выполнится
   
2. **React Hot Reload** во время выполнения Undo/Redo
   - Ref сохраняется между перезагрузками
   - Компонент перемонтируется, но ref остаётся `true`

3. **Асинхронная ошибка** внутри `try` блока, которая НЕ перехватывается

## Решение

### Часть 1: Принудительный сброс при обнаружении "залипшего" флага + return

```typescript
// ✅ v3.3.11: КРИТИЧНО - блокируем одновременные Undo операции
if (isUndoRedoInProgressRef.current) {
  console.warn('⏸️ UNDO/REDO: Undo уже выполняется, ожидайте завершения...');
  console.warn('⚠️ DEBUG: Флаг isUndoRedoInProgressRef не был сброшен!');
  console.warn('⚠️ Принудительный сброс флага...');
  isUndoRedoInProgressRef.current = false;
  console.log('✅ Флаг сброшен. Повторите попытку.');
  
  // ✅ Показываем toast чтобы пользователь знал что нужно повторить
  queueMicrotask(() => {
    showToast({
      type: 'info',
      message: 'Повторите попытку',
      description: 'Предыдущая операция была сброшена'
    });
  });
  return; // ✅ КРИТИЧНО: Возвращаемся чтобы НЕ устанавливать флаг снова
}
```

**Логика**:
- Если флаг `true` → это "залипание" (предыдущий Undo завершился но флаг не сбросился)
- Принудительно сбрасываем флаг
- **КРИТИЧНО**: Возвращаемся с `return` чтобы НЕ продолжать выполнение
- Показываем toast уведомление пользователю
- Пользователь повторяет попытку → флаг уже `false` → Undo выполняется

**Почему нужен `return`**:
- БЕЗ `return`: код продолжает выполнение → на строке 439 флаг снова устанавливается в `true` → при следующем вызове флаг опять `true` → **бесконечный цикл**
- С `return`: флаг остаётся `false` → при следующем вызове (повторная попытка) Undo выполняется нормально

### Часть 2: Детальное логирование для диагностики

```typescript
} catch (err) {
  console.error('🔄 UNDO/REDO: ❌ Критическая ошибка в handleUndo:', err);
} finally {
  // ✅ v3.3.11: Гарантированно снимаем блокировку даже при ошибках
  console.log('🔄 UNDO/REDO: 🔓 Сброс флага isUndoRedoInProgressRef (было:', isUndoRedoInProgressRef.current, ')');
  isUndoRedoInProgressRef.current = false;
  console.log('🔄 UNDO/REDO: 🔓 Блокировка снята (теперь:', isUndoRedoInProgressRef.current, ')');
}
```

**Логика**:
- Внешний `catch` перехватывает ВСЕ ошибки (даже те, что произошли между `try` и внутренними `try-catch`)
- `finally` ВСЕГДА выполняется
- Детальное логирование показывает значение флага до и после сброса

### Часть 3: Перемещение установки флага ВНУТРЬ try блока

**ВАЖНО**: Мы НЕ делаем этого, потому что:
1. Флаг должен быть установлен ДО начала асинхронных операций
2. Проверка `if (!state) return` должна быть ДО установки флага (иначе флаг залипнет)

Вместо этого мы добавляем внешний `catch` для перехвата ошибок.

## Как это решает проблему

1. **Принудительный сброс** → защита от "залипания" флага
2. **Return после сброса** → КРИТИЧНО! Предотвращает повторную установку флага в `true`
3. **Toast уведомление** → пользователь знает что нужно повторить попытку
4. **Внешний catch** → перехват ВСЕХ ошибок (даже необработанных)
5. **Finally гарантирует сброс** → флаг ВСЕГДА сбрасывается
6. **Детальное логирование** → легко диагностировать проблемы

### Почему нужен `return` после сброса флага

**БЕЗ `return`** (v3.3.16 первая версия - НЕ РАБОТАЕТ):
```typescript
if (isUndoRedoInProgressRef.current) {
  isUndoRedoInProgressRef.current = false;
  // Код продолжает выполняться
}
const state = historyUndo();
if (!state) return;
isUndoRedoInProgressRef.current = true; // ← СНОВА устанавливаем в true!
```
→ При следующем вызове флаг снова `true` → **бесконечный цикл**

**С `return`** (v3.3.16 финальная версия - РАБОТАЕТ):
```typescript
if (isUndoRedoInProgressRef.current) {
  isUndoRedoInProgressRef.current = false;
  return; // ← Выходим из функции
}
// Эта часть НЕ выполняется при залипшем флаге
isUndoRedoInProgressRef.current = true;
```
→ При следующем вызове флаг остаётся `false` → **Undo выполняется**
