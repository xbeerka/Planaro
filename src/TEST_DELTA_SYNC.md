## 🧪 Тест 1: Быстрая синхронизация изменений (4 секунды)

### Цель
Проверить что изменения появляются у другого пользователя через 4 секунды.

### Шаги
1. Откройте воркспейс в двух разных браузерах (пользователь A и пользователь B)
2. Пользователь A: создайте новое событие или переместите существующее
3. Пользователь B: откройте консоль браузера и наблюдайте логи

### Ожидаемый результат
```
⏸️ Delta Sync: пропуск (недавнее локальное изменение)  // ← 0-2 сек (пропуск)
📥 Delta Sync: получено 1 изменений                    // ← через 4 сек!
✅ Delta Sync: применено 1 изменений
```

### ✅ Критерий успеха
- Изменение появилось у пользователя B через 4-6 секунд
- В консоли видно сообщение "📥 Delta Sync: получено N изменений"

## 🧪 Тест 2: Full Sync для обнаружения удалений (30 секунд)

### Цель
Проверить что удалённые события исчезают у других пользователей.

### Шаги
1. Откройте воркспейс в двух браузерах
2. Пользователь A: удалите событие
3. Пользователь B: подождите 30 секунд и наблюдайте консоль

### Ожидаемый результат
```
// Через 4-8 секунд: Delta Sync (изменения есть)
📥 Delta Sync: получено 0 изменений  // ← удалённое событие НЕ придёт в delta

// Через 30 секунд: Full Sync
🔄 Full Sync: загрузка ВСЕХ событий (для обнаружения удалений)
🗑️ Full Sync: обнаружено 1 удалённых событий другими пользователями: ["e12345"]
✅ Full Sync: загружено 99 событий (было 100)
```

### ✅ Критерий успеха
- Событие исчезло у пользователя B через 30-35 секунд
- В консоли видно "🗑️ Full Sync: обнаружено N удалённых событий"

## 🧪 Тест 3: Защита от конфликтов при drag/drop

### Цель
Проверить что синхронизация не мешает перетаскиванию.

### Шаги
1. Откройте воркспейс
2. Начните перетаскивать событие (не отпускайте мышь)
3. Наблюдайте консоль во время перетаскивания

### Ожидаемый результат
```
⏸️ Delta Sync: пропуск (пользователь взаимодействует)  // ← КАЖДЫЕ 4 сек во время drag
⏸️ Delta Sync: пропуск (пользователь взаимодействует)
⏸️ Delta Sync: пропуск (пользователь взаимодействует)

```

### ✅ Критерий успеха
- Во время перетаскивания синхронизация заблокирована
- После отпускания мыши синхронизация возобновляется

## 🧪 Тест 7: Delta Sync timestamp

### Цель
Проверить что timestamp правильно обновляется.

### Шаги
1. Откройте воркспейс
2. Откройте Network tab и посмотрите на запрос `/events/changes`
3. Проверьте параметр `since` в следующих запросах

### Ожидаемый результат
```
Запрос 1: /events/changes?workspace_id=X
          Response: { events: [...], timestamp: "2025-11-18T10:00:00.000Z" }

Запрос 2: /events/changes?workspace_id=X&since=2025-11-18T10:00:00.000Z
          Response: { events: [...], timestamp: "2025-11-18T10:00:04.000Z" }

Запрос 3: /events/changes?workspace_id=X&since=2025-11-18T10:00:04.000Z

```

### ✅ Критерий успеха
- Первый запрос без `since` параметра
- Последующие запросы используют `timestamp` из предыдущего ответа
- Timestamp монотонно возрастает

## 🐛 Известные проблемы

### Проблема 1: "Воскрешение" удалённых событий
**Симптом**: Удалённое событие появляется снова после синхронизации  
**Причина**: `deletedEventIdsRef` не содержит ID удалённого события  
**Проверка**: Посмотрите консоль - должно быть "Отмечаем что было локальное изменение"

### Проблема 2: "Мигание" событий
**Симптом**: После перемещения событие прыгает туда-сюда  
**Причина**: Синхронизация происходит слишком быстро после локального изменения  
**Проверка**: Должно быть "⏸️ Delta Sync: пропуск (недавнее локальное изменение)"

### Проблема 3: Задержка > 10 секунд
**Симптом**: Изменения появляются слишком долго  
**Причина**: `DELTA_SYNC_INTERVAL` установлен неправильно  
**Проверка**: В `SchedulerContext.tsx` должно быть `const DELTA_SYNC_INTERVAL = 4000;`
