# Оптимизация загрузки онлайн пользователей в календаре

## Проблема

**До оптимизации:**
```
Пользователь входит в календарь
  ↓
OnlineUsers делает запрос к /presence/online/:workspaceId
  ↓ (200-500ms задержка)
Блок онлайн пользователей появляется с задержкой
```

**Проблема**: Видимая задержка при загрузке календаря, блок появляется не сразу.

## Решение

**После оптимизации:**
```
WorkspaceListScreen уже загрузил batch данные
  ↓
Данные сохранены в кэше cache_online_users_batch
  ↓
Пользователь входит в календарь
  ↓
OnlineUsers читает кэш (TTL 45 сек)
  ↓ (0ms - мгновенно!)
Блок онлайн пользователей показывается сразу
  ↓ (в фоне)
Запрос к /presence/online/:workspaceId для обновления
```

**Эффект**: Мгновенное отображение, такой же UX как в списке воркспейсов.

## Технические детали

### Структура кэша

```typescript
// Ключ: cache_online_users_batch
// TTL: 45 секунд
{
  data: {
    "1": [
      {
        userId: "uuid",
        email: "user@kode.ru",
        displayName: "Иван Иванов",
        avatarUrl: "https://...supabase.co/storage/.../avatars/...",
        lastSeen: "2025-10-21T12:00:00Z"
      },
      ...
    ],
    "14": [...],
    ...
  },
  timestamp: 1698765432000
}
```

### Логика загрузки

```typescript
// При монтировании OnlineUsers (первый раз)
fetchOnlineUsers(isInitial: true)
  ↓
loadFromCache()
  ↓
Проверка TTL (age < 45000ms)
  ↓
if (cache valid) {
  setOnlineUsers(cached.data[workspaceId]) // Мгновенно!
  hasLoadedCache.current = true
}
  ↓
Запрос к серверу (в фоне)
  ↓
setOnlineUsers(serverData) // Обновление
```

### Периодическое обновление

```typescript
// Каждые 15 секунд
fetchOnlineUsers(isInitial: false)
  ↓
Запрос к серверу (БЕЗ проверки кэша)
  ↓
setOnlineUsers(serverData)
```

## Преимущества

✅ **Мгновенное отображение** - 0ms задержка при входе в календарь
✅ **Консистентность** - те же данные что в списке воркспейсов
✅ **Актуальность** - автоматическое обновление в фоне
✅ **Снижение нагрузки** - переиспользование уже загруженных данных

## Связь с другими компонентами

1. **WorkspaceListScreen** - загружает batch данные каждые 15 секунд, сохраняет в кэш
2. **OnlineUsers (календарь)** - читает кэш при монтировании, затем обновляет в фоне
3. **App.tsx** - очищает кэш при выходе из системы

## Альтернативные подходы

❌ **Websockets/Realtime** - слишком сложно для простого presence
❌ **Polling каждые 5 секунд** - DDOS база данных
✅ **Кэширование с TTL** - баланс между актуальностью и производительностью
