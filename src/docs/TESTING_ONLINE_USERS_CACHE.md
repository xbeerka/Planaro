# Тестирование оптимизации загрузки онлайн пользователей

## Что тестируем

Проверяем что при входе в календарь воркспейса онлайн пользователи показываются **мгновенно** (из кэша), а не после задержки 200-500ms.

## Подготовка

1. **Откройте 2 браузера** (или 2 вкладки в режиме инкогнито):
   - Браузер A: войдите как `sa@kode.ru`
   - Браузер B: войдите как `test@kode.ru`

2. **В браузере B**:
   - Откройте календарь воркспейса (например, "Разработка 2025")
   - Heartbeat отправится → пользователь станет "онлайн" в этом воркспейсе
   - Оставьте эту вкладку открытой (heartbeat каждые 30 сек)

3. **В браузере A**:
   - Будем тестировать мгновенную загрузку

## Сценарий 1: Мгновенное отображение из кэша

### Шаги

1. **В браузере A**: Откройте список воркспейсов
2. **Подождите 2 секунды** - batch запрос загрузит онлайн пользователей
3. **Проверьте карточку воркспейса** - должен показаться кружок пользователя `test@kode.ru`
4. **Откройте DevTools → Console**
5. **Кликните на воркспейс** - войдите в календарь

### Ожидаемый результат

**В консоли должны быть логи:**

```
🔍 OnlineUsers: проверка кэша для workspace 14
💾 OnlineUsers: загружено из кэша (5с назад): 1 пользователей
  👤 test@kode.ru: displayName=Тест Тестированный, avatarUrl=да
💡 OnlineUsers: показываем кэшированные данные мгновенно, запрос в фоне...
👥 OnlineUsers: запрос к серверу для workspace: 14
📦 OnlineUsers: ответ сервера - 1 пользователей
  👤 test@kode.ru: displayName=Тест Тестированный, avatarUrl=да
```

**В UI:**
- ✅ Блок онлайн пользователей появляется **СРАЗУ** при входе в календарь (0ms)
- ✅ Показывается кружок с аватаркой `test@kode.ru`
- ✅ Через ~200-500ms данные обновляются с сервера (визуально без изменений)

### ❌ Признаки ошибки

- Блок появляется с задержкой 200-500ms
- В консоли нет лога `💾 OnlineUsers: загружено из кэша`
- Видно "мигание" - блок появляется → исчезает → появляется снова

## Сценарий 2: Кэш устарел (TTL > 45 сек)

### Шаги

1. **В браузере A**: Откройте список воркспейсов
2. **Подождите 50 секунд** - кэш устареет (TTL 45 сек)
3. **Откройте DevTools → Console**
4. **Кликните на воркспейс** - войдите в календарь

### Ожидаемый результат

```
🔍 OnlineUsers: проверка кэша для workspace 14
⏰ OnlineUsers: кэш устарел (52с > 45с)
👥 OnlineUsers: запрос к серверу для workspace: 14
📦 OnlineUsers: ответ сервера - 1 пользователей
  👤 test@kode.ru: displayName=Тест Тестированный, avatarUrl=да
✅ OnlineUsers: состояние обновлено с сервера - 1 пользователей
```

**В UI:**
- Блок появляется с небольшой задержкой (200-500ms) - нормально
- Кэш не используется, данные загружаются с сервера

## Сценарий 3: Кэш отсутствует (первый вход)

### Шаги

1. **Очистите IndexedDB**:
   - DevTools → Application → IndexedDB → `keyval-store` → Удалить
2. **Перезагрузите страницу**
3. **Войдите в систему**
4. **Сразу кликните на воркспейс** (НЕ ждите batch запроса)

### Ожидаемый результат

```
🔍 OnlineUsers: проверка кэша для workspace 14
📦 OnlineUsers: кэш отсутствует
👥 OnlineUsers: запрос к серверу для workspace: 14
📦 OnlineUsers: ответ сервера - 1 пользователей
✅ OnlineUsers: состояние обновлено с сервера - 1 пользователей
```

**В UI:**
- Блок появляется с небольшой задержкой (200-500ms) - нормально
- Нет кэша, данные загружаются с сервера

## Сценарий 4: Периодическое обновление

### Шаги

1. **Войдите в календарь воркспейса**
2. **Оставьте открытым на 30 секунд**
3. **Наблюдайте в консоли**

### Ожидаемый результат

Каждые **15 секунд** должен быть запрос к серверу:

```
👥 OnlineUsers: запрос к серверу для workspace: 14
📦 OnlineUsers: ответ сервера - 1 пользователей
```

❗ **НЕ должно быть** лога `🔍 OnlineUsers: проверка кэша` - кэш проверяется только при первой загрузке.

## Метрики производительности

### До оптимизации
- Время до появления блока: **200-500ms**
- Количество запросов: **1** (при входе в календарь)

### После оптимизации
- Время до появления блока: **0ms** (из кэша)
- Количество запросов: **1** (в фоне для обновления)
- **Выигрыш**: Мгновенное отображение, отсутствие задержки

## Проверка DevTools Performance

1. **DevTools → Performance**
2. **Запись** во время входа в календарь
3. **Проверьте Timeline**:
   - Блок онлайн пользователей должен рендериться **синхронно** с компонентом календаря
   - Не должно быть отдельного "фетча" перед рендером блока

## Troubleshooting

### Проблема: Кэш не используется

**Симптомы:**
- Лог `📦 OnlineUsers: кэш отсутствует` каждый раз
- Задержка при входе в календарь

**Решение:**
1. Проверьте что WorkspaceListScreen делает batch запрос
2. Проверьте что batch запрос сохраняет данные в IndexedDB:
   ```javascript
   await setStorageJSON('cache_online_users_batch', { data: {...}, timestamp: Date.now() })
   ```
3. Проверьте IndexedDB → `keyval-store` → должен быть ключ `cache_online_users_batch`

### Проблема: Блок "мигает"

**Симптомы:**
- Блок появляется → исчезает → появляется снова

**Решение:**
1. Проверьте что `hasLoadedCache.current` корректно работает
2. Проверьте что `setOnlineUsers` вызывается только при изменении данных (JSON.stringify сравнение)

### Проблема: Данные устаревшие

**Симптомы:**
- Показываются пользователи которые уже вышли из календаря

**Решение:**
1. Подождите 15 секунд - должно прийти обновление с сервера
2. Проверьте что сервер правильно фильтрует пользователей (TTL 2 минуты для presence)
