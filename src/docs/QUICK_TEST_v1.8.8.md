# Быстрая проверка v1.8.8 - Исправление "призраков" и "мигания"

## 🎯 Что тестируем

v1.8.8 исправляет две проблемы presence системы:
1. **"Призраки"** - пользователи оставались "онлайн" 2+ минуты после закрытия календаря
2. **"Мигание"** - текущий пользователь появлялся на 1-2 секунды при возврате из календаря

## ✅ Быстрый тест (5 минут)

### Тест 1: Нет "мигания" текущего пользователя (быстрый переход)

**Шаги** (1 браузер) - КРИТИЧНЫЙ ТЕСТ:
1. Войдите как `test@kode.ru`
2. **БЫСТРО** откройте календарь любого воркспейса (клик)
3. **МГНОВЕННО** нажмите "Назад" (<1 секунды после входа)
4. **Смотрите на карточки воркспейсов**

**✅ Ожидаемое поведение**:
- Вашей аватарки НЕТ в онлайн списке сразу после возврата
- **НЕ должно быть "мигания"** даже на долю секунды
- Плавный переход, никаких визуальных артефактов
- **Это главный тест** - проверяет временную блокировку presence

**❌ Проблема, если**:
- Ваша аватарка появляется на 1-2 секунды, затем исчезает
- Видны задержки или скачки

**Логи в консоли** (должны быть):
```
🔙 Возврат к списку воркспейсов
🧹 Очистка текущего пользователя из presence кэша: test@kode.ru
✅ Кэш очищен от текущего пользователя
🔒 Установлена временная блокировка presence для: test@kode.ru
[... может быть batch запрос ...]
🔒 Активна блокировка presence для test@kode.ru (0с / 5с)
  🚫 Workspace X: отфильтрован текущий пользователь (1 удалено)
✅ Batch данные отфильтрованы - текущий пользователь скрыт
👋 Отправка leave для workspace: X
✅ Leave успешно отправлен - пользователь удалён из онлайн списка
```

### Тест 3: Fallback при закрытии вкладки

**Шаги** (2 браузера):
1. Браузер A: войдите как `sa@kode.ru` → откройте список воркспейсов
2. Браузер B: войдите как `test@kode.ru` → откройте календарь воркспейса #14
3. В браузере A: видите аватарку `test@kode.ru`
4. В браузере B: **закройте вкладку** (крестик) или весь браузер
5. В браузере A: подождите **60 секунд**

**✅ Ожидаемое поведение**:
- Аватарка `test@kode.ru` исчезает **максимум через 60 секунд**
- Leave запрос может не успеть отправиться (вкладка закрылась)
- Но TTL истекает → автоматическое удаление

**❌ Проблема, если**:
- Аватарка остаётся больше 90 секунд
- Это означает что TTL не работает

## 🐛 Troubleshooting

### "Мигание" всё равно есть

**Проверьте**:
1. Консоль браузера - есть ли логи `🧹 Очистка кэша`?
2. Если нет → очистка не запускается
3. Если есть, но "мигание" есть → проверьте timing (см. `/docs/TESTING_CACHE_CLEANUP.md`)

### Аватарка не исчезает 60+ секунд

**Проверьте**:
1. Серверные логи - есть ли `👋 Leave`?
2. Если нет → leave не дошёл → проверьте Network tab
3. Если есть, но аватарка остаётся → проверьте batch запрос (каждые 15 сек)
4. DevTools → Network → фильтр `/presence/online-batch` → смотрите response

### Другие пользователи пропадают при возврате

**Проверьте**:
1. IndexedDB → `cache_online_users_batch`
2. Должны остаться только ДРУГИЕ пользователи (не вы)
3. Если все пропали → ошибка в фильтре (см. код в App.tsx)
