# Фикс "Not Found" при выборе фото в ProfileModal

## Проблема

**Симптом:**
- Пользователь открывает ProfileModal (редактирование профиля)
- Кликает на аватарку для выбора фото
- **Выброс на страницу "Not Found"** (404)
- Приходится обновлять страницу вручную
- Изображение не сохраняется

## Причина

### Техническая деталь:
HTML `<button>` элементы по умолчанию имеют **`type="submit"`** если не указано иное.

В ProfileModal было несколько кнопок без явного `type="button"`:
```tsx
// ❌ БЫЛО - button без type
<button onClick={() => fileInputRef.current?.click()}>
  <Upload />
</button>

// ❌ При клике браузер думал что это submit формы
// ❌ Пытался отправить form → 404 → редирект на Not Found
```

### Почему это ломалось:
1. Пользователь кликает на кнопку Upload
2. Браузер видит `<button>` без `type` → считает это `type="submit"`
3. Пытается отправить несуществующую форму
4. Отправка на текущий URL через POST
5. Сервер не обрабатывает POST на этом URL → 404
6. React роутинг ловит 404 → показывает "Not Found"

## Решение

### 1. Добавлен `type="button"` ко всем кнопкам

```tsx
// ✅ ИСПРАВЛЕНО
<button
  type="button"  // ← КРИТИЧНО!
  onClick={(e) => {
    e.preventDefault();
    e.stopPropagation();
    fileInputRef.current?.click();
  }}
>
  <Upload />
</button>
```

### 2. Исправлены ВСЕ кнопки в модалке:

#### Кнопка закрытия (X в заголовке):
```tsx
<button
  type="button"
  onClick={(e) => {
    e.preventDefault();
    e.stopPropagation();
    handleCancel();
  }}
>
  <X />
</button>
```

#### Кнопка Upload (иконка на аватарке):
```tsx
<button
  type="button"
  onClick={(e) => {
    e.preventDefault();
    e.stopPropagation();
    fileInputRef.current?.click();
  }}
>
  <Upload />
</button>
```

#### Вся область аватарки (div):
```tsx
<div 
  onClick={(e) => {
    e.preventDefault();
    e.stopPropagation();
    fileInputRef.current?.click();
  }}
>
  {/* Avatar */}
</div>
```

#### Кнопка "Отмена" в футере:
```tsx
<button
  type="button"
  onClick={(e) => {
    e.preventDefault();
    handleCancel();
  }}
>
  Отмена
</button>
```

#### Кнопка "Сохранить" в футере:
```tsx
<button
  type="button"
  onClick={(e) => {
    e.preventDefault();
    handleSave();
  }}
>
  Сохранить
</button>
```

## Best Practices (для будущего)

### ✅ Всегда указывай type для кнопок:

```tsx
// ✅ ПРАВИЛЬНО - кнопка действия (не submit)
<button type="button" onClick={handleClick}>
  Действие
</button>

// ✅ ПРАВИЛЬНО - явный submit (только внутри <form>)
<button type="submit" onClick={handleSubmit}>
  Отправить форму
</button>

// ❌ НЕПРАВИЛЬНО - браузер считает это submit
<button onClick={handleClick}>
  Действие
</button>
```

### ✅ Используй preventDefault для безопасности:

```tsx
<button
  type="button"
  onClick={(e) => {
    e.preventDefault();      // Предотвращает отправку формы
    e.stopPropagation();     // Останавливает всплытие события
    handleClick();
  }}
>
  Действие
</button>
```

### ✅ Проверяй все модальные окна:

Места где часто забывают `type="button"`:
- ❌ Кнопки закрытия (X)
- ❌ Кнопки "Отмена" / "Закрыть"
- ❌ Кнопки действий внутри форм
- ❌ IconButtons (Upload, Delete, Edit и т.д.)
