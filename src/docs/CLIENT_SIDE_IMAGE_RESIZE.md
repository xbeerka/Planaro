# Клиентское сжатие изображений

## Обзор

Реализовано **безопасное клиентское сжатие** аватарок до 200px по меньшей стороне перед загрузкой на сервер.

## Почему клиентское сжатие?

### ✅ Преимущества:
1. **100% безопасность** - не затрагивает серверный код, нет риска падения Edge Function
2. **Экономия ресурсов** - сервер не тратит CPU на обработку изображений
3. **Быстрая загрузка** - пользователь загружает только маленький файл (~10-50 КБ вместо 1-5 МБ)
4. **Мгновенный preview** - пользователь видит результат до загрузки
5. **Простая отладка** - всё в DevTools браузера
6. **Graceful degradation** - если сжатие не сработает, загрузится оригинал

### ❌ Почему НЕ серверное сжатие:
- `imagescript` и другие библиотеки для обработки изображений нестабильны в Deno Edge Functions
- Требуют нативные зависимости → высокий риск падения всего сервера
- Увеличивают время выполнения функции → дополнительные расходы
- Сложнее отлаживать

## Реализация

### 1. Утилита `/utils/imageResize.ts`

```typescript
export async function resizeImageOnClient(file: File): Promise<File>
```

**Алгоритм:**
1. Проверяет что файл - изображение
2. Загружает через FileReader
3. Создаёт Image элемент
4. Вычисляет новые размеры (200px по меньшей стороне, пропорции сохраняются)
5. Рисует на Canvas с новыми размерами (high quality smoothing)
6. Конвертирует в JPEG (качество 90%)
7. Возвращает новый File объект

**Оптимизация:**
- Если изображение уже ≤200px → возвращает оригинал без обработки
- При ошибке → возвращает оригинал (fallback)
- Подробное логирование для отладки

### 2. Интеграция в ProfileModal

```typescript
import { resizeImageOnClient } from '../../utils/imageResize';

const handleAvatarSelect = async (e: React.ChangeEvent<HTMLInputElement>) => {
  const file = e.target.files?.[0];
  if (!file) return;

  // Валидация (размер, тип)
  // ...

  // 🎨 Клиентское сжатие

  setAvatarFile(resizedFile);
  
  // Preview из сжатого файла
  // ...

};
```

## Результаты

### Типичные сценарии:
- **Фото с телефона** (4000x3000, 2.5MB) → 267x200px, ~40KB (экономия 98%)
- **Обычная фотография** (1920x1080, 800KB) → 355x200px, ~35KB (экономия 95%)
- **Уже маленькое изображение** (150x150, 20KB) → без изменений (оригинал)

### Performance:
- Сжатие 2MB изображения: ~100-300ms (зависит от устройства)
- Не блокирует UI (асинхронная операция)
- Практически незаметно для пользователя

## Альтернативы (НЕ используются)

1. **Supabase Storage Transformations** - платно, меньше контроля
2. **Серверное сжатие (imagescript)** - нестабильно в Edge Functions, риск падения
3. **CDN трансформации** - зависимость от внешнего сервиса

## Тестирование

### Проверить:
1. ✅ Большое изображение (>2MB) → сжимается корректно

3. ✅ Некорректный файл → показывает ошибку
4. ✅ Toast уведомление появляется
5. ✅ Preview показывает сжатое изображение
6. ✅ Загрузка на сервер работает
