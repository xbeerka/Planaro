## 🎯 Ключевые моменты

### ✅ Преимущества

1. **Мгновенное обновление для отправителя** (0ms)
   - State обновляется локально сразу после сохранения
   - Пользователь не ждёт broadcast

2. **Быстрое обновление для получателей** (~1 секунда)
   - Broadcast через WebSocket: ~100ms
   - Загрузка событий: ~500-800ms

3. **Умная фильтрация**
   - Отправитель игнорирует свой broadcast
   - Нет двойного обновления или лишних запросов

4. **Легкая нагрузка**
   - Размер broadcast: <100 байт
   - Полная загрузка только для получателей

### 🔒 Защита от проблем

1. **Проверка userId**
   - Каждый broadcast содержит `userId` отправителя
   - Получатель проверяет: `if (userId === currentUserId) return`

2. **Fallback на HTTP**
   - Если broadcast не дошёл - не критично
   - Другие пользователи получат обновление через обычный polling
   - WebSocket автоматически fallback на long-polling при блокировке firewall

3. **Graceful degradation**
   - Если Realtime отключен - приложение работает
   - Пользователь видит свои изменения всегда
   - Чужие изменения при перезагрузке страницы

## 📦 Размер данных

### Broadcast сообщение (отправляется через WebSocket):
```json
{
  "type": "broadcast",
  "event": "event_changed",
  "payload": {
    "action": "create",
    "userId": "abc-123-def-456"
  }
}
```

**Размер**: ~80 байт  
**Частота**: только при изменениях (не периодически)

### Ответ GET /events (загружается после broadcast):
```json
[
  {
    "id": "e1",
    "resourceId": "u1",
    "projectId": "p1",
    "startWeek": 10,
    "weeksSpan": 4,
    "unitStart": 0,
    "unitsTall": 1
  }
]
```

**Размер**: зависит от количества событий (~500 байт - 5 КБ)  
**Частота**: только при broadcast от других пользователей

## 🔧 Код (упрощённо)

### Подключение к Realtime:
```typescript
const channel = supabase
  .channel(`workspace:${workspaceId}:events`)
  .on('broadcast', { event: 'event_changed' }, (payload) => {
    const { action, userId } = payload.payload;
    
    // Игнорируем свои изменения
    if (userId === currentUserId) return;
    
    // Перезагружаем события
    refetchEvents();
  })
  .subscribe();
```

### Отправка broadcast:
```typescript
const broadcastChange = async (action: 'create' | 'update' | 'delete') => {
  await supabase.channel(`workspace:${workspaceId}:events`)
    .send({
      type: 'broadcast',
      event: 'event_changed',
      payload: { action, userId: currentUserId }
    });
};
```

### В операциях:
```typescript
const createEvent = async (event) => {
  // 1. Сохраняем на сервере
  const created = await eventsApi.create(event);
  
  // 2. Обновляем state
  setEvents(prev => [...prev, created]);
  
  // 3. Отправляем broadcast
  await broadcastChange('create');
};
```
