# Быстрый тест блокировки временных событий v3.3.9

## Цель
Проверить что временные события (`ev_temp_...`) заблокированы для взаимодействия до создания на сервере.

## Как тестировать

### ✅ Тест 1: Визуальная индикация
1. Откройте календарь
2. Создайте событие (любым способом: кликом, copy+paste, context menu)
3. **Ожидаемое поведение**:
   - Событие появляется СРАЗУ с временным ID `ev_temp_...`
   - Отображается **спиннер** рядом с названием проекта
   - Событие имеет **сниженную прозрачность** (opacity 0.6)
   - **Диагональные полоски** на фоне (stripes pattern)
   - Через ~500ms спиннер исчезает (событие создано на сервере)

### ✅ Тест 2: Блокировка drag
1. Создайте событие
2. **Сразу** попробуйте перетащить его (< 1 сек после создания)
3. **Ожидаемое поведение**:
   - Событие **НЕ перетаскивается**
   - Курсор не меняется
   - Через ~500ms можно перетащить

### ✅ Тест 3: Блокировка resize
1. Создайте событие
2. **Сразу** попробуйте изменить размер (< 1 сек после создания)
3. **Ожидаемое поведение**:
   - **Ручки resize НЕ отображаются**
   - Hover на краях события не показывает ручки
   - Через ~500ms ручки появляются

### ✅ Тест 4: Блокировка edit (контекстное меню)
1. Создайте событие
2. **Сразу** правый клик → "Edit"
3. **Ожидаемое поведение**:
   - Модалка редактирования **НЕ открывается**
   - Появляется **toast** с текстом: "Событие ещё создаётся на сервере, подождите..."
   - Через ~500ms можно редактировать

### ✅ Тест 5: Блокировка delete
1. Создайте событие
2. **Сразу** правый клик → "Delete"
3. **Ожидаемое поведение**:
   - Событие **НЕ удаляется**
   - Появляется **toast** с текстом: "Событие ещё создаётся на сервере, подождите..."
   - Через ~500ms можно удалить

### ✅ Тест 6: Блокировка copy
1. Создайте событие
2. **Сразу** правый клик → "Copy"
3. **Ожидаемое поведение**:
   - Событие **НЕ копируется**
   - Появляется **toast** с текстом: "Событие ещё создаётся на сервере, подождите..."
   - Через ~500ms можно копировать

### ✅ Тест 7: Undo/Redo корректность (главная проверка)
1. Создайте событие через copy+paste
2. **Подождите ~1 секунду** (чтобы создалось на сервере)
3. Перетащите событие в другое место
4. Нажмите **Ctrl+Z** (Undo)
5. **Ожидаемое поведение**:
   - Событие **возвращается** на исходную позицию
   - Событие **НЕ удаляется** (это был баг в v3.3.8)
6. Нажмите **Ctrl+Shift+Z** (Redo)
7. **Ожидаемое поведение**:
   - Событие **возвращается** на новую позицию
   - Событие **НЕ удаляется**

### ✅ Тест 8: Full Sync не возвращает удалённые события
1. Создайте событие
2. Удалите его через context menu
3. Нажмите **Ctrl+Z** (Undo - восстанавливаем)
4. Подождите **65 секунд** (больше 60 сек TTL + 1 Full Sync)
5. **Ожидаемое поведение**:
   - Событие **остаётся удалённым**
   - НЕ возвращается через Full Sync
   - Консоль: `🧹 Очистка пометки удаления: {id}` через ~60 сек

## Если что-то не работает

### Событие не блокируется
- Проверьте `isBlocked` в React DevTools
- Убедитесь что ID начинается с `ev_temp_`
- Логи: `console.log('isBlocked:', event.id.startsWith('ev_temp_'))`

### Спиннер не исчезает
- Проверьте логи Edge Function - возможно CREATE не завершился
- Проверьте `pendingEventIds` в React DevTools
- Timeout: должен исчезнуть через ~500ms

### Undo удаляет событие
- Проверьте что в истории НЕТ временных ID: `console.log('History:', history)`
- Проверьте `updateHistoryEventId` - должен заменить временный ID на реальный
- Если есть временные ID → это баг!
