# 🧪 Тестирование статических импортов Supabase

**Дата**: 2025-11-18  
**Цель**: Проверить работает ли **статический импорт** (вместо динамического)

---

## 🎯 Проблема

**Динамический импорт** вызывает ошибку сборки:
```typescript
const { createClient } = await import('@supabase/supabase-js'); // ❌
```

**Ошибка**:
```
ERROR: [plugin: npm] Failed to fetch
```

## 💡 Гипотеза

Возможно **статический импорт** сработает:
```typescript
import { createClient } from '@supabase/supabase-js'; // ❓
```

**Почему может сработать**:
- Статический импорт обрабатывается на этапе сборки
- Bundler может резолвить его по-другому
- В Figma Make некоторые пакеты требуют статический импорт

---

## 🧪 Варианты для тестирования

### Вариант 1: Статический импорт БЕЗ версии
```typescript
import { createClient } from '@supabase/supabase-js';
```

**Файл**: `/utils/supabase/client-static.ts` (УЖЕ СОЗДАН)

### Вариант 2: Статический импорт С версией
```typescript
import { createClient } from '@supabase/supabase-js@2';
```

**Аналогия**: В Guidelines указано что `react-hook-form@7.55.0` требует версию.

### Вариант 3: Только Realtime клиент (лёгкий пакет)
```typescript
import { RealtimeClient } from '@supabase/realtime-js';
```

**Обоснование**: Возможно основной пакет слишком тяжёлый, а realtime - отдельный лёгкий пакет.

### Вариант 4: Gotrue для Auth
```typescript
import { GoTrueClient } from '@supabase/gotrue-js';
```

**На случай** если нужна только авторизация для Realtime.

## 🔍 Что проверяем

### Разница между динамическим и статическим импортом

**Динамический**:
```typescript
const module = await import('package'); // Runtime, async
```

- Загружается **во время выполнения**
- Bundler может не включить в итоговый bundle
- Может требовать fetch пакета в runtime

**Статический**:
```typescript
import { ... } from 'package'; // Build time, sync
```

- Загружается **на этапе сборки**
- Bundler **обязан** резолвить и включить в bundle
- Все зависимости инлайнятся в итоговый файл

### Почему статический может сработать?

В **Figma Make**:
- Есть whitelist разрешённых пакетов
- Bundler настроен автоматически
- Статический импорт может быть в whitelist, а динамический - нет

---

## ⚠️ Важное замечание

**Если статический импорт НЕ сработает**:
- Это подтвердит что пакет **точно** не в whitelist
- Никакие варианты импорта не помогут
- HTTP polling - единственное решение

**Но стоит попробовать** - может быть bundler просто не умеет обрабатывать динамический `import()` для этого пакета!
