# Отладка аватарок в presence системе

## Проблема
Аватарка не показывается в онлайн пользователях (логи показывают `avatarUrl=нет`)

## Причины

### 1. Аватарка не загружена в профиль
**Проверка**: Откройте профиль (клик на аватар в хедере) и посмотрите - есть ли там аватарка?

**Решение**: Загрузите аватарку через профиль:
1. Клик на ваш аватар/инициалы в хедере
2. "Редактировать профиль"
3. Загрузить аватарку
4. "Сохранить"
5. Страница автоматически перезагрузится через 2 секунды

### 2. Токен не обновился после загрузки аватарки
**Симптомы**: Аватарка загружена, но в логах всё равно `avatarUrl=нет`

**Решение**: После сохранения профиля страница ДОЛЖНА перезагрузиться автоматически. Если не перезагрузилась - обновите вручную (F5).

### 3. Аватарка не сохранилась в user_metadata
**Проверка**: Посмотрите логи сервера (Supabase Dashboard → Edge Functions → Logs):
```
💓 Heartbeat от sa@kode.ru в workspace 14
   user_metadata: {"name":"Александр Синицкий","avatar_url":"https://..."}
✅ Presence сохранён: presence:14:...
   displayName: Александр Синицкий
   avatarUrl: https://...
```

**Если в `user_metadata` нет `avatar_url`**: 
- Проблема в endpoint `/profile/update`
- Проверьте логи загрузки аватара
- Убедитесь что сервер возвращает `avatar_url` и вызывает `admin.updateUserById`

## Диагностика

### Шаг 1: Проверьте клиентские логи
Откройте консоль браузера (F12) и найдите:
```
🖼️ Avatar URL из токена: https://... или отсутствует
```

Если "отсутствует" - токен не содержит avatarUrl → нужно загрузить аватарку в профиль.

### Шаг 2: Проверьте серверные логи (heartbeat)
В Supabase Dashboard → Edge Functions → Logs найдите:
```
💓 Heartbeat от ВАШ_EMAIL в workspace ID
   user_metadata: {...}
   avatarUrl: https://... или нет
```

Если `avatarUrl: нет` - значит в user_metadata нет avatar_url.

### Шаг 3: Проверьте серверные логи (online endpoint)
```
✅ Активных пользователей: 1
  👤 ВАШ_EMAIL: displayName=ВашеИмя, avatarUrl=да или нет
```

Если `avatarUrl=нет` - значит presence данные не содержат avatarUrl (проблема из шага 2).

### Шаг 4: Проверьте batch endpoint
```
📊 Workspace ID: 1 онлайн
   👤 ВАШ_EMAIL: displayName=ВашеИмя, avatarUrl=да или нет
```

## Решение "я всё загрузил, но не показывается"

1. **Выйдите из системы** (кнопка "Выход" в хедере)
2. **Войдите заново** - это гарантирует получение свежего токена
3. **Откройте календарь воркспейса** - heartbeat отправится с новыми данными
4. **Проверьте логи** - должны показать `avatarUrl: да`

## Если ничего не помогло

Откройте DevTools → Application → IndexedDB → auth_access_token

Скопируйте токен и вставьте на https://jwt.io/

Найдите секцию `user_metadata`:
```json
{
  "user_metadata": {
    "name": "Александр Синицкий",
    "avatar_url": "https://..."  // ← Должна быть здесь
  }
}
```

Если `avatar_url` отсутствует - проблема на сервере при сохранении профиля.
Проверьте логи endpoint `/profile/update`.
