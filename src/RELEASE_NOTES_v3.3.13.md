## 🐛 Что исправлено

### Критический баг: неправильный паттерн временных ID в cleanup

**Проблема**: В системе удаления orphaned событий (cleanup) использовался неправильный паттерн для проверки временных ID:
- **Было**: `!event.id.startsWith('ev_temp')` ← без подчеркивания в конце
- **Стало**: `!event.id.startsWith('ev_temp_')` ← с подчеркиванием

**Последствия бага**:
- ❌ Временные события отправлялись на удаление через API
- ❌ Сервер отвечал ошибкой `Cannot delete temporary events via API`
- ❌ Ложные ошибки в консоли: `❌ Ошибка удаления события ev_temp_XXX`
- ❌ Лишняя нагрузка на сеть (DELETE запросы для несуществующих событий)

**Где проявлялось**:
- При удалении департамента с только что созданными событиями
- При удалении ресурса с pending событиями
- В любом сценарии с orphaned событиями, созданными < 5 секунд назад
