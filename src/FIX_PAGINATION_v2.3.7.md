## 🔥 Проблема

**PostgREST ВСЕГДА возвращает максимум 1000 записей за один запрос**, независимо от указанного `.limit()` или `.range()`.

### До исправления (НЕПРАВИЛЬНО)

```typescript
// ❌ Вернёт только 1000 событий, даже если в БД их 8053!
const { data: events } = await supabase
  .from('events')
  .select('*, event_patterns(name, pattern)')
  .order('id', { ascending: true })
  .range(0, 49999); // ← PostgREST игнорирует, вернёт max 1000
```

**Результат**: Создано 8053 событий → Загружено 1000 → Потеряно 7053 (87.6%)

## ✅ Решение: ПАГИНАЦИЯ

Реализована автоматическая пагинация в GET `/events` endpoint:

```typescript
// ✅ Загружаем ВСЕ события циклом
const PAGE_SIZE = 1000;
const allEvents: any[] = [];
let currentPage = 0;
let hasMore = true;

while (hasMore) {
  const rangeStart = currentPage * PAGE_SIZE;
  const rangeEnd = rangeStart + PAGE_SIZE - 1;
  
  const { data: pageEvents } = await query.range(rangeStart, rangeEnd);
  
  if (pageEvents && pageEvents.length > 0) {
    allEvents.push(...pageEvents);
  }
  
  hasMore = pageEvents.length === PAGE_SIZE; // Продолжаем если получили полную страницу
  currentPage++;
  
  if (currentPage >= 100) break; // Защита: max 100,000 событий
}

return c.json(allEvents); // Вернём ВСЕ события
```

### Как это работает

1. **Страница 1**: `.range(0, 999)` → 1000 событий
2. **Страница 2**: `.range(1000, 1999)` → 1000 событий  
3. **...**
4. **Страница 9**: `.range(8000, 8999)` → 53 события (последняя)

**Итого**: 8053 события за 9 запросов (~2-5 секунд)
