## 🐛 Суть проблемы

**До исправления**:
```typescript
// ❌ НЕПРАВИЛЬНО - отсутствует подчеркивание!
if (event.id.startsWith('e') && !event.id.startsWith('ev_temp')) {
  eventsApi.delete(event.id, accessToken).catch(err => ...)
}
```

**Проблема**: Проверка `!event.id.startsWith('ev_temp')` пропускала правильные временные ID вида `ev_temp_1732005123456_789`, потому что они начинаются с `'ev_temp'`.

**Последствия**:
- Временные события отправлялись на удаление через API
- Сервер возвращал ошибку `Cannot delete temporary events via API`
- Ложные ошибки в консоли: `❌ Ошибка удаления события ev_temp_XXX`
- Лишняя нагрузка на сеть (DELETE запросы для несуществующих событий)

**После исправления**:
```typescript
// ✅ ПРАВИЛЬНО - с подчеркиванием в конце!
// ✅ v3.3.13: КРИТИЧЕСКОЕ ИСПРАВЛЕНИЕ - правильный паттерн временных ID
if (event.id.startsWith('e') && !event.id.startsWith('ev_temp_')) {
  eventsApi.delete(event.id, accessToken).catch(err => 
    console.error(`❌ Ошибка удаления события ${event.id}:`, err)
  );
}
```

**Результат**:
- ✅ Временные события корректно пропускаются
- ✅ Нет ложных ошибок в консоли
- ✅ Нет лишних DELETE запросов
- ✅ Согласованность паттернов во всём коде
