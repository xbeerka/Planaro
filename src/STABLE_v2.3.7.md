## 📊 Обзор проекта

**Planaro (Resource Scheduler)** - продвинутое веб-приложение для планирования ресурсов с годовым календарём на 52 недели.

### Основные возможности
- ✅ Годовой календарь (52 недели) с sticky заголовками
- ✅ Drag & Drop события с курсорным позиционированием
- ✅ Resize событий (4 направления) с плавной анимацией
- ✅ Режим ножниц для разделения событий
- ✅ Undo/Redo с горячими клавишами (Ctrl+Z / Ctrl+Shift+Z)
- ✅ Z-order управление для перекрывающихся событий
- ✅ Фильтрация по департаментам и проектам

- ✅ Генерация тестовых данных (100% заполнение календаря)

- ✅ Аутентификация только для @kode.ru

## 🔥 КРИТИЧЕСКОЕ ИСПРАВЛЕНИЕ v2.3.7

### Проблема
**PostgREST ВСЕГДА возвращает максимум 1000 записей за один запрос**, независимо от `.limit()` или `.range()`.

**Результат**: Создано 8053 событий → Загружено 1000 → **Потеряно 7053 (87.6%)**

### Решение
Реализована **автоматическая пагинация** в GET `/events` endpoint:

```typescript
// Загружаем ВСЕ события циклом
const PAGE_SIZE = 1000;
const allEvents: any[] = [];
let currentPage = 0;
let hasMore = true;

while (hasMore) {
  const rangeStart = currentPage * PAGE_SIZE;
  const rangeEnd = rangeStart + PAGE_SIZE - 1;
  
  const { data: pageEvents } = await query.range(rangeStart, rangeEnd);
  
  if (pageEvents && pageEvents.length > 0) {
    allEvents.push(...pageEvents);
  }
  
  hasMore = pageEvents.length === PAGE_SIZE;
  currentPage++;
  
  if (currentPage >= 100) break; // Защита: max 100,000 событий
}

return c.json(allEvents); // Вернём ВСЕ события
```

### Результаты
- ✅ **Загрузка 100%**: 8053 события за 9 запросов (~2-5 сек)
- ✅ **Защита от переполнения**: максимум 100,000 событий
- ✅ **Детальное логирование**: каждая страница + общее время
- ✅ **Integrity rate: 100%** (было 12.4%)

## 🌐 Онлайн пользователи (HTTP-based Presence)

### Как работает
1. **При входе в календарь**: OnlineUsers отправляет heartbeat каждые 30 сек
2. **Presence в KV Store**: TTL 60 секунд (автоудаление если нет heartbeat)
3. **При закрытии календаря**: explicit `DELETE /presence/leave/:workspaceId`
4. **Batch запросы**: WorkspaceListScreen загружает онлайн для всех воркспейсов
5. **Кэширование**: TTL 45 секунд, мгновенное отображение

### Отображение
- **В карточках воркспейсов**: только пользователи ВНУТРИ воркспейса
- **Текущий пользователь**: зеленый градиент с меткой "(вы)"
- **Другие пользователи**: синий градиент
- **Аватарки**: для ВСЕХ из presence данных

### Оптимизации
- ✅ **Batch оптимизация**: 1 запрос вместо N → снижение нагрузки в 15 раз
- ✅ **Graceful leave**: мгновенное удаление при закрытии календаря
- ✅ **Двухуровневая защита от "мигания"**: очистка кэша + временная блокировка

## 🔄 Автообновление данных (Simple Polling)

### Что обновляется
- **События**: каждые 10 секунд (меняются чаще всего)
- **Сотрудники**: каждые 15 секунд
- **Департаменты**: каждые 15 секунд
- **Проекты**: каждые 15 секунд

### Как работает
1. Приложение запрашивает данные с сервера по таймеру
2. Сравнивает с текущими данными через `JSON.stringify`
3. Если есть изменения → обновляет UI и кэш
4. Если было локальное изменение < 2 секунд назад → пропускает polling

### Преимущества
- ✅ **Работает везде** (только HTTP, не нужен WebSocket)
- ✅ **Простая отладка** (обычные HTTP запросы в DevTools)
- ✅ **Предсказуемое поведение** (18 req/min/user)
- ✅ **Синхронизация всех данных** между пользователями
