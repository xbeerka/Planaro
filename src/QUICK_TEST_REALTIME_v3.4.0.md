## ⚡ Быстрый старт (5 минут)

### 1. Открыть приложение

Откройте приложение в браузере и войдите под пользователем с email `@kode.ru`

### 2. Проверить консоль браузера

**Если Realtime ДОСТУПЕН** (всё работает ✅):
```
🔌 Загрузка @supabase/supabase-js...
✅ Supabase клиент инициализирован
🖱️ Подключение к Realtime Presence для workspace: 1
👤 Пользователь: Иван Иванов ivan@kode.ru
📡 Realtime статус: SUBSCRIBED
✅ Подключено к Realtime Presence
```

**Если Realtime НЕДОСТУПЕН** (ТЕКУЩАЯ СИТУАЦИЯ ⚠️):
```
⚠️ @supabase/supabase-js недоступен в Figma Make
⚠️ Supabase Realtime отключен - используйте HTTP polling
⚠️ Supabase Realtime недоступен - курсоры отключены
```

**ЭТО НОРМАЛЬНО!** Приложение работает стабильно в режиме v3.3.6.

### 3. Тест с двумя пользователями

1. **Откройте два браузера** (или incognito + обычный)
2. **Войдите под разными пользователями** (`user1@kode.ru` и `user2@kode.ru`)
3. **Откройте ОДИН workspace** в обоих браузерах
4. **Двигайте мышью** в первом браузере
5. **Ожидаемый результат**:
   - Во втором браузере появляется курсор с именем первого пользователя
   - Курсор плавно следует за мышью (задержка ~50-100ms)
   - При остановке мыши курсор исчезает через 5 секунд

### 4. Тест graceful leave

1. **Закройте первый браузер** (или вкладку)
2. **Ожидаемый результат**:
   - Во втором браузере курсор должен исчезнуть через 5 секунд

## 🔍 Диагностика проблем

### Проблема 1: `@supabase/supabase-js` недоступен

**Что делать**:
- ✅ Это НЕ критично - приложение работает в режиме v3.3.6
- ✅ Delta Sync работает (HTTP polling)
- ✅ OnlineUsers работает (HTTP heartbeat)
- ❌ Курсоры отключены (не важно для продакшена)

**Как исправить** (опционально):
- Уточнить у разработчиков Figma Make доступность `@supabase/supabase-js`
- Или оставить как есть (текущая версия стабильна)

### Проблема 2: Realtime подключается но курсоры не появляются

**Проверить в консоли**:
```javascript
// Должно быть:
📡 Realtime статус: SUBSCRIBED
✅ Подключено к Realtime Presence

// НЕ должно быть:
📡 Realtime статус: CHANNEL_ERROR
❌ Ошибка подключения к Realtime
```

**Если `CHANNEL_ERROR`**:
1. Проверить что в Supabase Dashboard включен Realtime для таблиц
2. Проверить RLS политики для `workspace_members`
3. Проверить что текущий пользователь есть в `workspace_members`

### Проблема 3: Realtime работает но курсоры "тормозят"

**Проверить**:
- Throttle настроен на 50ms (норма)
- Проверить сетевую задержку в DevTools → Network → WS (WebSocket)
- Если задержка >200ms - проблема с интернетом
