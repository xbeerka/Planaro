# Quick Test v2.3.5 - Генерация 100% заполнение

## 🎯 Цель теста

Проверить что алгоритм генерации заполняет **100% ячеек календаря** для каждого сотрудника.

## ✅ Чеклист быстрой проверки

### 1. Подготовка (1 мин)
- [ ] Откройте любой воркспейс с сотрудниками
- [ ] Убедитесь что есть хотя бы 1 проект
- [ ] Откройте консоль браузера (F12)

### 2. Генерация событий (2-3 мин)
- [ ] Нажмите кнопку **"Сгенерировать события"** (зелёная кнопка с иконкой загрузки)
- [ ] Подтвердите диалог (должен быть текст: "100% заполнение календаря БЕЗ ПУСТОТ")
- [ ] Следите за progress bar:
  - Обработано сотрудников: должно расти до 100%
  - Заполнено ячеек: должно достичь 100%
  - Загружено на сервер: должно показать финальное количество

### 3. Проверка консоли (1 мин)
В консоли для КАЖДОГО сотрудника должно быть:
```
✅ [Имя сотрудника]: создано X событий, заполнено 208/208 ячеек (100.0%)
```

**НЕ должно быть**:
```
⚠️ Остались пустые ячейки для [Имя]: N шт.
```

### 4. Визуальная проверка календаря (2 мин)
- [ ] Каждая строка (сотрудник) заполнена на 100% БЕЗ пустых промежутков
- [ ] События разных размеров (не все одинаковые)
- [ ] События разных цветов (случайные проекты)
- [ ] Нет "зависших" пустых строк

### 5. Проверка количества событий (30 сек)
- [ ] В консоли финальное сообщение должно показать:
  ```
  ✅ Перезагружено N событий с сервера
  ```
- [ ] N должно быть примерно `кол-во сотрудников × 5-15` (зависит от размеров)

## ❌ Что проверять если что-то не так

### Если заполнение < 100%:
1. Проверьте консоль - есть ли `⚠️ Остались пустые ячейки`
2. Проверьте количество итераций (должно быть < 1000)
3. Откройте issue с скриншотом консоли

### Если статус показывает "всё загружено" но строки пустые:
1. Проверьте консоль - сколько событий создано vs загружено
2. Есть ли ошибки в логах (красные ❌)
3. Проверьте что endpoint `/events` возвращает все события

### Если прогресс "зависает":
1. Подождите 30 секунд (может быть задержка между batch-ами)
2. Проверьте Network tab - есть ли pending запросы
3. Проверьте что сервер отвечает (кнопка "Проверить снова" если сервер недоступен)

## 🎉 Критерии успеха

**PASS** если:
- ✅ Все строки заполнены на 100%
- ✅ В консоли для всех сотрудников: `заполнено 208/208 ячеек (100.0%)`
- ✅ Нет предупреждений `⚠️ Остались пустые ячейки`
- ✅ Финальное количество событий соответствует ожиданиям

**FAIL** если:
- ❌ Хотя бы одна строка не заполнена на 100%
- ❌ Есть предупреждения о пустых ячейках
- ❌ Генерация "зависла" на середине
- ❌ Большое расхождение между созданными и загруженными событиями

## 📊 Ожидаемые результаты

Для воркспейса с **50 сотрудниками**:
- Всего ячеек: `50 × 208 = 10,400`
- Ожидаемое количество событий: `50 × 5-15 = 250-750` (зависит от размеров)
- Среднее событие: 3-8 недель × 1-2 юнита
- Время генерации: 10-20 секунд (зависит от производительности)
- Время загрузки: 5-10 секунд (batch по 50 событий, 500мс задержка)
