# 🐛 Исправление: Пропадающий промежуточный шаг в истории Undo/Redo

## Проблема

При выполнении последовательных операций промежуточный шаг "исчезает" из истории:

**Сценарий:**
1. Создал 1 → saveHistory ✅
2. Создал 2 → saveHistory ✅  
3. Создал 3 → saveHistory ✅
4. Изменил 3 (drag) → saveHistory ✅ ... **НО ЭТО ПРОИСХОДИТ СЛИШКОМ ПОЗДНО!**
5. Следующая операция → saveHistory ✅ (перезаписывает шаг 4)

**Результат:** Шаги "Создал 3" и "Изменил 3" объединяются в один!

---

## Причина

### Порядок операций при Drag/Resize

```typescript
// В /hooks/useEventInteractions.ts (строки 220-232)

// ✅ 1. Оптимистичное обновление UI (МГНОВЕННО)
onEventsUpdate(prev => prev.map(e => e.id === updatedEvent.id ? updatedEvent : e));

// ✅ 2. Сохранение в историю (МГНОВЕННО)
onSaveHistory(
  events.map(e => e.id === updatedEvent.id ? updatedEvent : e),
  eventZOrder,
  projects
);

// ✅ 3. Обновление в БД (АСИНХРОННО С DEBOUNCE 2000ms!)
onEventUpdate(updatedEvent.id, updatedEvent);
```

### Что происходит:

1. **t=0ms:** Drag событие → `onSaveHistory()` вызван ✅
2. **t=0ms:** `onEventUpdate()` → `queueEventUpdate()` → **ждёт 2000ms** ⏱️
3. **t=500ms:** Пользователь делает следующую операцию
4. **t=500ms:** Следующая операция → `saveHistory()` ЕЩЁ РАЗ ✅
5. **t=2000ms:** `flushPendingUpdates()` выполняет BATCH save
6. **РЕЗУЛЬТАТ:** Шаг 1 перезаписан шагом 2 в истории! ❌

---

## Решение

### Сократить Debounce Delay с 2000ms до 500ms

**Файл:** `/contexts/SchedulerContext.tsx` (строка 192)

**Было:**
```typescript
},
2000 // ⏱️ Debounce delay 2 секунды (быстрее!)
);
```

**Стало:**
```typescript
},
500 // ⏱️ Debounce delay 500ms (компромисс между производительностью и сохранением истории)
);
```

### Почему 500ms?

- ✅ **Производительность:** Достаточно для накопления быстрых изменений (multiple drag)
- ✅ **История:** Достаточно быстро чтобы не пропустить промежуточные шаги
- ✅ **UX:** Пользователь не успеет сделать следующую операцию за 500ms
- ✅ **Серверная нагрузка:** Снижает количество запросов (batch updates)

---

## Новый порядок операций

### При Drag/Resize (с 500ms debounce):

```
t=0ms:   Drag start → onSaveHistory() ✅ Шаг 1 сохранён
t=0ms:   onEventUpdate() → queueEventUpdate() ⏱️ Ждёт 500ms
t=500ms: flushPendingUpdates() → BATCH save на сервер ✅
t=600ms: Пользователь делает следующую операцию
t=600ms: onSaveHistory() ✅ Шаг 2 сохранён (НЕ перезаписывает шаг 1!)
```

**РЕЗУЛЬТАТ:** Оба шага сохранены в истории! ✅

## Тестирование

### Сценарий 1: Быстрые последовательные операции

1. Создай событие → wait 100ms
2. Drag событие → wait 100ms
3. Resize событие → wait 100ms
4. Создай новое событие

**Ожидаемый результат:**
- История: [начальное, +создал, +drag, +resize, +создал]
- 4 шага Undo вернут к начальному состоянию
- 4 шага Redo вернут к финальному состоянию

### Сценарий 2: Медленные операции (больше 500ms между)

1. Создай событие → **wait 1000ms**
2. Drag событие → **wait 1000ms**
3. Resize событие → **wait 1000ms**
4. Создай новое событие

**Ожидаемый результат:**
- История: [начальное, +создал, +drag, +resize, +создал]
- 4 шага Undo вернут к начальному состоянию
- 4 шага Redo вернут к финальному состоянию
- **Все промежуточные шаги сохранены!**

### Сценарий 3: Множественные drag (< 500ms)

1. Drag событие на неделю 1
2. Drag событие на неделю 2 (через 100ms)
3. Drag событие на неделю 3 (через 100ms)
4. Drag событие на неделю 4 (через 100ms)
5. Остановка на 600ms

**Ожидаемый результат:**
- История: [начальное, +drag на неделю 4]
- **Промежуточные drag объединены в один шаг** (это ПРАВИЛЬНО!)
- 1 шаг Undo вернёт к начальной позиции
- BATCH save отправит только финальную позицию (оптимизация!)

## Дополнительные улучшения (будущие)

### 1. Адаптивный Debounce Delay

Вместо фиксированного 500ms использовать адаптивный:

```typescript
// Если пользователь активен (делает операции) → короткий delay (200ms)
// Если пользователь неактивен (пауза) → длинный delay (500ms)
const dynamicDelay = isUserActive ? 200 : 500;
```

### 2. Флаг "Завершённость операции"

```typescript
// В useEventInteractions.ts

// При pointerup
onSaveHistory(...);
await onEventUpdate(...); // ✅ Ждём завершения (НЕ debounced!)

// Это гарантирует что saveHistory вызван ПОСЛЕ сохранения на сервер
```

### 3. Умное объединение истории

```typescript
// В useHistory.ts

// Если два последовательных шага изменяют ОДНО событие
// И разница по времени < 200ms
// → Объединить в один шаг

// Это полезно для множественных drag одного события
```
