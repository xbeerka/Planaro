## 🧪 Тест-кейсы

### ✅ Тест 1: Undo не возвращает удалённое событие

**Шаги**:
1. Открыть календарь воркспейса
2. Создать новое событие (например, проект "Test" на неделе 20)
3. Кликнуть на событие → "Удалить событие"
4. Нажать **Ctrl+Z** (Undo)
5. Событие вернулось ✅
6. Нажать **Ctrl+Shift+Z** (Redo)
7. Событие снова удалилось ✅
8. **Подождать 30 секунд** (Full Sync)

**Ожидаемый результат**:
- ✅ Событие НЕ вернулось после Full Sync
- ✅ В логах: `🗑️ Найдено 1 удалённых событий: ["e12345"]`
- ✅ В логах: `✅ Событие удалено на сервере: e12345`
- ✅ В логах: `✅ Full Sync: загружено N событий (было N)` (количество НЕ изменилось)

**Критерий успеха**: Событие остаётся удалённым

## 📊 Метрики производительности

### Timing синхронизации удалений

**Измеряем**:
1. Время от Undo/Redo до начала синхронизации
2. Время удаления событий на сервере
3. Время до очистки пометок

**Ожидаемые значения**:
- Начало синхронизации: **< 50ms** (мгновенно после Undo/Redo)
- Удаление на сервере: **100-500ms** (зависит от количества событий)
- Очистка пометок: **10 секунд** (фиксированный timeout)

**Как проверить**:
```javascript
// В Chrome DevTools Console
const startTime = performance.now();
// ... нажать Ctrl+Z (Undo)
// ... дождаться "✅ Undo: удалённые события успешно синхронизированы"
const endTime = performance.now();
console.log(`Синхронизация заняла: ${endTime - startTime}ms`);
```

## 🐛 Потенциальные проблемы

### Проблема 1: "Cannot read property 'id' of undefined"
**Причина**: `previousEvents` не сохранены до Undo/Redo  
**Решение**: Убедиться что `const previousEvents = events;` вызывается ДО `setEvents()`

### Проблема 2: События всё равно возвращаются
**Причина**: `deletedEventIdsRef` не очищает пометки, или Full Sync не фильтрует  
**Решение**: Проверить логи, убедиться что `deletedEventIdsRef.current.add()` вызывается

### Проблема 3: React warning остался
**Причина**: Где-то ещё используется `setHasCachedData`  
**Решение**: Поиск по файлам: `setHasCachedData` → заменить на `hasCachedDataRef.current = true`
