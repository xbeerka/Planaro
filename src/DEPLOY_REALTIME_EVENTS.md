**⚠️ БЕЗ ЭТОГО ФУНКЦИЯ НЕ БУДЕТ РАБОТАТЬ!**

#### Шаг 1: Включить Realtime для таблицы `events`

1. Откройте **Supabase Dashboard**: https://app.supabase.com
2. Выберите ваш проект
3. Перейдите: **Database → Replication** (левое меню)
4. Найдите таблицу **`events`** в списке
5. **Включите** переключатель в колонке **"Realtime"**
6. Дождитесь зелёной галочки ✅

![Supabase Realtime Toggle](https://supabase.com/docs/img/realtime-replication.png)

#### Шаг 2: Проверить публикацию (опционально)

Если возникли проблемы, выполните в SQL Editor:

```sql
-- Проверить что таблица events добавлена в репликацию
SELECT * FROM pg_publication_tables 
WHERE pubname = 'supabase_realtime' 
AND tablename = 'events';

-- Если результат пустой, добавить вручную:
ALTER PUBLICATION supabase_realtime ADD TABLE events;
```

## 🧪 Тестирование после деплоя

### Тест 1: Проверка подключения

1. Откройте календарь любого воркспейса
2. Откройте **Developer Console** (F12)
3. Найдите в логах:

```
🔄 EventsRealtimeSync: инициализация для workspace 123
🔌 Realtime статус подписки: SUBSCRIBED
✅ Успешно подписались на realtime обновления events для workspace 123
```

**Если видите ошибку**:
```
❌ Ошибка подписки на realtime канал
```

→ Проверьте что Realtime включен в Dashboard (Шаг 1)

### Тест 2: Синхронизация между пользователями

1. Откройте один воркспейс в **двух разных вкладках** браузера
2. **Вкладка 1**: Перетащите любое событие на другую неделю
3. **Вкладка 2**: Событие должно **мгновенно** переместиться (100-300ms)

**Ожидаемые логи**:

**Вкладка 1** (выполняет изменение):
```
🔒 Регистрируем локальное обновление для игнорирования эхо
🔄 Отправка обновления события на сервер: 456
✅ Событие обновлено на сервере: 456
📨 Realtime событие получено: UPDATE {...}
⏭️ Пропуск своего изменения для события 456  ← ВАЖНО!
```

**Вкладка 2** (получает изменение):
```
📨 Realtime событие получено: UPDATE {...}
✏️ Обновление события от другого пользователя: 456  ← ВАЖНО!
```

### Тест 3: Создание и удаление

- **Создание**: Создайте событие во вкладке 1 → должно появиться во вкладке 2
- **Удаление**: Удалите событие во вкладке 1 → должно исчезнуть во вкладке 2

## ❌ Troubleshooting

### Проблема: События не синхронизируются

**Причина**: Realtime не включен для таблицы `events`

**Решение**:
1. Перейдите в Dashboard → Database → Replication
2. Включите Realtime для таблицы `events`
3. Обновите страницу приложения (F5)
4. Проверьте логи - должен быть `SUBSCRIBED`

### Проблема: Дублирование событий

**Причина**: Не работает игнорирование собственных изменений

**Симптомы** в логах:
```
📨 Realtime событие получено: UPDATE
✏️ Обновление события от другого пользователя: 456
```
→ НЕ должно быть для своих изменений! Должен быть `⏭️ Пропуск`

**Решение**:
1. Очистите кэш браузера и перезагрузите страницу
2. Проверьте что `__registerLocalEventUpdate` вызывается перед API запросом
3. Проверьте что `eventId` передаётся как `number`, не `string`

### Проблема: Большая задержка (>1 секунда)

**Причина**: Проблемы с Supabase сервером или сетью

**Решение**:
1. Проверьте статус Supabase: https://status.supabase.com/
2. Проверьте пинг: `curl -i https://{project-id}.supabase.co/rest/v1/`
3. Проверьте что Realtime включен в вашем тарифном плане

## 📊 Мониторинг

### Метрики для отслеживания

1. **WebSocket соединения**:
   - Dashboard → Settings → API → Realtime
   - Смотрите кол-во активных подключений

2. **Realtime events sent**:
   - Dashboard → Usage → Realtime
   - График отправленных событий

3. **Ошибки в логах**:
   ```
   ❌ Ошибка подписки на realtime канал
   ⏱️ Таймаут подписки на realtime канал
   ```

### Ожидаемые показатели

- **Latency**: 100-300ms (99 percentile)
- **Messages/sec**: ~0.1-10 (зависит от активности)
- **Bandwidth**: ~500 bytes на событие
- **Reconnects**: <1 раз в час
