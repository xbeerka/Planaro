## 🎯 Основное изменение

### Проблема
При быстром нажатии Ctrl+Z несколько раз подряд возникала race condition:
- Второй Undo запускался ДО завершения первого
- Конфликты синхронизации с сервером
- Ошибки, дублирование запросов

### Решение
Блокировка одновременных операций через `isUndoRedoInProgressRef`:
```typescript
const isUndoRedoInProgressRef = useRef<boolean>(false);

const handleUndo = useCallback(async () => {
  if (isUndoRedoInProgressRef.current) return; // ← Блокировка
  isUndoRedoInProgressRef.current = true;
  
  try {
    // ... логика undo ...
  } finally {
    isUndoRedoInProgressRef.current = false; // ← Гарантированное снятие
  }
}, [...]);
```

## 🎓 Важные уроки

### 1. Async операции требуют блокировки
Быстрые хоткеи (Ctrl+Z) могут вызвать функцию ДО завершения предыдущего вызова.  
**Решение**: `useRef` для флага блокировки (без ре-рендеров)

### 2. finally блок критичен
Гарантирует очистку даже при ошибках.  
**Результат**: Предотвращает "зависание" системы

### 3. Логирование с эмоджи
🔄, ⏸️, 🔓 помогают быстро найти нужные логи в консоли.  
**Результат**: Быстрая диагностика проблем
