# Quick Test - Race Condition Fix

Быстрый тест для проверки что race condition исправлена.

---

## ✅ Главный тест - Initial Load

1. **Очистите кэш браузера** (Ctrl+Shift+Delete)
2. Откройте приложение
3. Войдите в воркспейс
4. Откройте консоль (F12)

**Ожидаемый результат:**

```
✅ НЕТ предупреждений типа:
   ⚠️ Throttle: Пропуск дубликата "events-delta-sync-22" (уже выполняется 1ms)
   ⚠️ Delta Sync: пропущен (throttle)

✅ Видите только нормальные логи:
   🔄 Delta Sync: загрузка изменённых событий...
   🔄 Projects Sync: загрузка проектов...
   🔄 Resources Sync: загрузка сотрудников...
   🔄 Departments Sync: загрузка департаментов...
```

**Если видите блокировки с временем 0-1ms** - race condition НЕ исправлена!

## ❌ Что ПРОВЕРИТЬ если тест ПРОВАЛИЛСЯ

### Проблема: Видите блокировки с временем 0-1ms

```
⚠️ Throttle: Пропуск дубликата "events-delta-sync-22" (уже выполняется 1ms)
```

**Причина**: Race condition не исправлена

**Решение**:
1. Проверьте что в `/utils/requestThrottle.ts` логика проверки + регистрации внутри `throttledRequest()`
2. Проверьте что НЕТ вызовов `canMakeRequest()` и `registerRequest()` перед регистрацией
3. Перезагрузите страницу (Ctrl+F5)

### Проблема: Видите долгие запросы (>10s)

```
🐌 events-delta-sync-22: 15s ⚠️ МЕДЛЕННО!
```

**Причина**: Сервер медленный

**Решение**:
1. Проверьте логи Edge Function
2. Оптимизируйте SQL запросы
3. Добавьте индексы в БД

### Проблема: Высокая частота запросов (>50 req/min)

```
⚠️ events-delta-sync: 75 req/min
```

**Причина**: Утечка setInterval (компоненты не размонтируются)

**Решение**: Проверьте useEffect cleanup функции в `SchedulerContext.tsx`
