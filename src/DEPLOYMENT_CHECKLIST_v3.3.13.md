## ⚠️ Rollback Plan

### Если возникли проблемы

**1. Симптомы для rollback**:
- Cleanup не удаляет реальные orphaned события
- События удаляются слишком агрессивно
- Новые ошибки в консоли

**2. Быстрый rollback**:
```bash
git revert HEAD
git push origin main
```

**3. Или откатить конкретный файл**:
```bash
git checkout HEAD~1 -- contexts/SchedulerContext.tsx
git commit -m "rollback: откат исправления v3.3.13 (проблемы в production)"
git push origin main
```

**4. Уведомить команду**:
- Описать симптомы
- Приложить логи/скриншоты
- Создать issue для исследования
