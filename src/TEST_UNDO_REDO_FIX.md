# 🧪 Тест исправления: Защита истории от событий без проектов

## 📋 Описание проблемы

**До исправления**:
- При сохранении изменений в модалках Users/Projects → вызывался `resetHistory(events, eventZOrder)` БЕЗ проектов
- История сохранялась с событиями но без проектов
- При попытке Undo → система правильно блокировала восстановление, но выдавала ошибку

**Ошибка в консоли**:
```
❌ История: КРИТИЧЕСКАЯ ОШИБКА - попытка восстановить state с событиями но без проектов!
❌ История: это приведёт к удалению всех событий. Отменяем Undo.
```

## ✅ Исправление

1. Все вызовы `resetHistory()` теперь передают проекты
2. Добавлена защита в `saveHistory()` - блокирует сохранение events без projects
3. Детальное логирование для диагностики

## 🧪 Тестовый сценарий

### Тест 1: Сохранение пользователей → Undo
**Цель**: Проверить что история сохраняется с проектами после изменений в UsersManagementModal

**Шаги**:
1. Открой календарь воркспейса с событиями
2. Открой модалку "Сотрудники" (иконка пользователя в хедере)
3. Добавь нового сотрудника или измени существующего
4. Нажми "Сохранить"
5. Открой консоль браузера (F12)
6. Нажми `Ctrl+Z` (Undo)

**Ожидаемый результат**:
```
📝 Сброс истории после изменений сотрудников (с проектами)
📝 История: СБРОС
📝 История: UNDO - текущий index X, всего записей Y
📝 История: UNDO - возвращаем state с N событиями, M проектами (index X-1)
```

**НЕ должно быть**:
```
❌ История: КРИТИЧЕСКАЯ ОШИБКА - попытка восстановить state с событиями но без проектов!
```

```
📝 История: UNDO - текущий index X, всего записей Y
📝 История: UNDO - возвращаем state с N событиями, M проектами (index X-1)
```

**НЕ должно быть**:

### Тест 3: Создание события → Undo → Redo
**Цель**: Проверить что обычный Undo/Redo работает корректно

**Шаги**:
1. Открой календарь воркспейса
2. Создай новое событие (двойной клик на ячейке)
3. Заполни форму и сохрани
4. Нажми `Ctrl+Z` (Undo) - событие должно исчезнуть
5. Нажми `Ctrl+Shift+Z` или `Ctrl+Y` (Redo) - событие должно вернуться
6. Проверь логи в консоли

**Ожидаемый результат**:
```
📝 История: СОХРАНЕНИЕ
📝 История: N записей, index: N-1
📝 История: UNDO - возвращаем state с M событиями, K проектами (index N-2)
📝 История: REDO - возвращаем state с M+1 событиями, K проектами (index N-1)
```

---

### Тест 4: Множественные действия → Undo несколько раз
**Цель**: Проверить что история работает при множественных операциях

**Шаги**:
1. Создай 3 новых события
2. Нажми `Ctrl+Z` три раза подряд
3. Проверь что все события исчезли
4. Проверь логи - НЕ должно быть ошибок

**Ожидаемый результат**:
- Каждое Undo корректно восстанавливает предыдущее состояние
- В логах видны проекты при каждом Undo
- Нет ошибок про "события без проектов"

---

### Тест 5: Попытка сохранить события без проектов (защита saveHistory)
**Цель**: Проверить что новая защита в saveHistory блокирует некорректное состояние

**Искусственный тест** (только для разработчиков):
```typescript
// В консоли браузера
saveHistory([{ id: 'e1', ... }], new Map(), []); // Попытка сохранить события без проектов
```

**Ожидаемый результат**:
```
❌ История: КРИТИЧЕСКАЯ ОШИБКА - попытка сохранить события без проектов!
❌ История: это приведёт к некорректному Undo/Redo. Отменяем сохранение.
```

И история НЕ должна сохраниться (проверить через canUndo).

---

## 📊 Критерии успеха

✅ **Все тесты пройдены** если:
1. Нет ошибок про "события без проектов" при обычном использовании
2. Undo/Redo работает корректно после сохранения в модалках
3. В логах видно "с проектами" при сбросе истории
4. saveHistory блокирует сохранение некорректного состояния

❌ **Тест провален** если:
1. Появляется ошибка про "события без проектов" при обычном Undo
2. Undo/Redo не работает после модалок
3. История сохраняется с пустым массивом проектов когда есть события

## 🔍 Отладка

### Полезные логи для проверки

**При сбросе истории** (должно быть "с проектами"):
```
📝 Сброс истории после изменений сотрудников (с проектами)
📝 Сброс истории после изменений проектов (с проектами)
```

**При Undo/Redo** (должны быть указаны проекты):
```
📝 История: UNDO - возвращаем state с 42 событиями, 12 проектами (index 4)
📝 История: REDO - возвращаем state с 43 событиями, 12 проектами (index 5)
```

**При сохранении истории**:
```
📝 История: СОХРАНЕНИЕ
📝 История: 15 записей, index: 14
```

### Проверка через DevTools

1. Открой консоль (F12)
2. Отфильтруй логи по "История"
3. Выполни действия из тестов
4. Проверь что нет красных ошибок
5. Проверь что в логах Undo/Redo указаны проекты
