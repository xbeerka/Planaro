# 🐛 Исправление: Неправильное определение временных ID в Undo/Redo

## Проблема

При Undo/Redo все события из БД определялись как "временные" и пытались создаться заново на сервере!

**Логи:**
```
🔄 UNDO/REDO: Событие e37316 имеет временный ID, нужно создать
🔄 UNDO/REDO: Событие e37320 имеет временный ID, нужно создать
...
🔄 UNDO/REDO: Событий для создания на сервере: 14
🔄 UNDO/REDO: Событий для обновления на сервере: 0  ❌ ПРОБЛЕМА!
```

**Симптомы:**
1. ✅ Создал событие
2. ✅ Drag событие
3. ❌ Undo 2 раза → событие удалилось (шаг drag пропущен)
4. ❌ Redo не работает правильно

---

## Причина

### Неправильная проверка временного ID

**Файл:** `/contexts/SchedulerContext.tsx` (строка 1143)

**Было:**
```typescript
// ❌ НЕПРАВИЛЬНО: ВСЕ ID из БД начинаются с 'e'!
const isTemporaryId = event.id.startsWith('e');

// Примеры ID из БД: e37316, e37320, e37345
// ВСЕ они проходят проверку и считаются временными! ❌
```

**Проблема:** ID из Supabase имеют формат `e37316` (буква "e" + число). Проверка `startsWith('e')` определяет их как "временные"!

**Результат:**
- События из БД пытаются создаться заново → duplicate key error
- Undo/Redo работает неправильно
- История ломается

---

## Решение

### Правильная проверка временного ID

**Стало:**
```typescript
// ✅ ПРАВИЛЬНО: Только настоящие временные ID!
const isTemporaryId = event.id.startsWith('ev_temp_');

// Примеры:
// ev_temp_1763499847802 → TRUE (временный)
// e37316 → FALSE (реальный из БД)
// e37320 → FALSE (реальный из БД)
```

### Обновлённая логика

```typescript
events.forEach(event => {
  // Проверяем формат ID: временный ID начинается с "ev_temp_"
  const isTemporaryId = event.id.startsWith('ev_temp_');
  
  if (isTemporaryId) {
    console.log(`🔄 UNDO/REDO: Событие ${event.id} имеет временный ID, нужно создать`);
    eventsToCreate.push(event); // → BATCH CREATE операция
  } else {
    console.log(`🔄 UNDO/REDO: Событие ${event.id} имеет реальный ID, нужно обновить`);
    eventsToUpdate.push(event); // → BATCH UPDATE операция
  }
});
```

## Дополнительная проверка

### Форматы ID в системе:

1. **Временные ID (клиент):**
   - Формат: `ev_temp_` + timestamp
   - Пример: `ev_temp_1763499847802`
   - Где создаётся: `createEvent()` в SchedulerContext

2. **Реальные ID (Supabase):**
   - Формат: `e` + serial number
   - Пример: `e37316`, `e37320`, `e37345`
   - Где создаётся: PostgreSQL sequence в таблице `events`

3. **Проверка:**
```typescript
// ✅ ПРАВИЛЬНО
'ev_temp_1763499847802'.startsWith('ev_temp_') // → true
'e37316'.startsWith('ev_temp_')                // → false

// ❌ НЕПРАВИЛЬНО (старая логика)
'ev_temp_1763499847802'.startsWith('e')        // → true
'e37316'.startsWith('e')                       // → true (ОШИБКА!)
```
