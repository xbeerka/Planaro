## 🧪 Testing Checklist

### 1. Генерация событий
- [ ] Открыть приложение
- [ ] Авторизоваться через email @kode.ru
- [ ] Создать воркспейс с минимум 3 сотрудниками
- [ ] Создать минимум 2 проекта
- [ ] Нажать кнопку **"Сгенерировать эвенты"**
- [ ] Подтвердить действие

### 2. Проверка в консоли
- [ ] Открыть DevTools → Console (F12)
- [ ] Найти логи генерации
- [ ] Проверить для КАЖДОГО сотрудника:
  ```
  ✅ Иванов Иван: создано X событий, заполнено 208/208 ячеек (100.0%)
  ```
- [ ] Убедиться что:
  - [ ] **Все** сотрудники имеют `208/208 ячеек (100.0%)`
  - [ ] Количество событий варьируется: от 1 до 12
  - [ ] Нет предупреждений "⚠️ Остались пустые ячейки"

### 3. Визуальная проверка
- [ ] Прокрутить календарь по горизонтали (все 52 недели)
- [ ] Проверить каждого сотрудника:
  - [ ] Все 4 юнита заполнены цветными блоками (события)
  - [ ] Нет серых промежутков (пустых ячеек)
  - [ ] События имеют разные размеры (короткие, средние, длинные)
  - [ ] События имеют разную высоту (1-4 юнита)

### 4. Функциональное тестирование
После генерации проверить основные функции:

#### Drag & Drop
- [ ] Выбрать событие
- [ ] Перетащить на другую неделю
- [ ] Убедиться что событие переместилось
- [ ] Проверить сохранение в БД (обновить страницу)

#### Resize
- [ ] Навести на событие
- [ ] Увидеть ручки resize (4 направления)
- [ ] Изменить ширину события (левая/правая ручка)
- [ ] Изменить высоту события (верхняя/нижняя ручка)
- [ ] Убедиться что размер изменился
- [ ] Проверить сохранение в БД

#### Scissors (ножницы)
- [ ] Нажать кнопку **"Ножницы"** в тулбаре (или Ctrl+X)
- [ ] Навести на событие
- [ ] Увидеть вертикальные линии на границах недель
- [ ] Кликнуть на линию
- [ ] Убедиться что событие разделилось на две части
- [ ] Проверить что обе части сохранились в БД

#### Undo/Redo
- [ ] Переместить событие
- [ ] Нажать **Ctrl+Z** (Undo)
- [ ] Убедиться что событие вернулось на место
- [ ] Нажать **Ctrl+Shift+Z** (Redo)
- [ ] Убедиться что событие снова переместилось

#### Context Menu
- [ ] Правый клик на событие
- [ ] Выбрать **"Edit"**
- [ ] Изменить проект или размер
- [ ] Сохранить
- [ ] Правый клик на другое событие
- [ ] Выбрать **"Delete"**
- [ ] Убедиться что событие удалено

### 5. Очистка событий
- [ ] Нажать кнопку **"Очистить все эвенты"**
- [ ] Подтвердить действие
- [ ] Убедиться что все события удалены (пустой календарь)
- [ ] Проверить что событий нет в БД (обновить страницу)

### 6. Повторная генерация
- [ ] Снова нажать **"Сгенерировать эвенты"**
- [ ] Проверить что результат отличается от первого раза (случайность работает)
- [ ] Проверить что опять **100% заполнение** для всех сотрудников
