## 📋 Краткое описание

Исправлены две критические проблемы с системой Undo/Redo:
1. **Pending состояние после Undo** - события показывали полоски загрузки после отмены действия
2. **Лишний toast warning** - раздражающее уведомление при попытке undo

---

## 🐛 Исправленные баги

### 1. Pending состояние после Undo

**Проблема:**
- Пользователь создаёт событие → drag события → быстрый Undo (< 2 сек)
- Событие возвращается к исходной позиции, НО показывает полоски загрузки
- Визуальный артефакт сбивает с толку

**Причина:**
- Debounced save (2 сек задержка) продолжает выполняться после Undo
- Pending операция в очереди → событие показывает pending state

**Исправление:**
- Добавлен `await flushPendingChanges()` в начало `handleUndo` и `handleRedo`
- Все pending изменения сохраняются ПЕРЕД undo/redo
- Очередь пуста → нет pending операций → нет полосок

**Результат:**
- ✅ События мгновенно восстанавливаются без артефактов
- ✅ Нет визуального "мигания"
- ✅ UX стал предсказуемым

### 2. Лишний toast warning

**Проблема:**
- При попытке undo временного события (< 500ms после создания)
- Появлялся toast "Подождите, дождитесь завершения создания событий"
- Раздражал пользователя

**Исправление:**
- Убрали toast уведомление
- Оставили только console.log для диагностики

**Результат:**
- ✅ Меньше визуального шума
- ✅ Toast только для критических ошибок
