## 📊 Технические характеристики

| Параметр | Значение |
|----------|----------|
| Задержка broadcast | ~100-300ms |
| Время перезагрузки событий | ~500-800ms |
| **Итоговая задержка** | **~600-1100ms** |
| Размер сообщения | <100 байт |
| Технология | WebSocket (fallback: long-polling) |
| Нагрузка на сервер | Минимальная |
| Стоимость | Бесплатно (до 200 подключений) |

## ❓ FAQ

### **Q: Нужно ли что-то настраивать на сервере?**
**A:** Нет! Realtime Broadcast работает только на клиенте. Серверная часть не требуется.

### **Q: Что если Realtime не включен?**
**A:** Приложение продолжит работать, но без автообновления. Пользователи увидят только свои изменения.

### **Q: Сколько это стоит?**
**A:** Бесплатно на Supabase Free tier:
- До 200 одновременных подключений
- До 2 млн сообщений в месяц
- Для 10-20 пользователей это бесплатно

### **Q: Как отключить если нужно?**
**A:** Закомментируйте useEffect с Realtime в `/contexts/SchedulerContext.tsx` (строки ~348-390).
