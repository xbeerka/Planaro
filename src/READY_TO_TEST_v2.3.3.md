## 🧪 Как протестировать (2 минуты)

### 1. Откройте календарь
- Войдите в систему
- Выберите воркспейс с 50+ сотрудниками

### 2. Генерация событий
```
1. Нажмите "Генерировать события" (в ТЕСТОВЫЕ КНОПКИ)
2. Подтвердите действие
3. Откройте консоль браузера (F12)
```

### 3. Проверьте логи
**Должны увидеть**:
```
🧪 Создание XXX событий для YYY сотрудников...
📤 Загрузка пачки 1 (100 событий)...
✅ Пачка загружена: 100 событий
📊 Прогресс: 100 / XXX (успешно: 100, ошибок: 0)
⏳ Задержка 2 секунды перед следующей пачкой...
...
🔄 Перезагрузка всех событий с сервера...
✅ Перезагружено XXX событий с сервера
✅ Генерация тестовых событий завершена без ошибок
```

**НЕ должно быть**:
```
❌ ReferenceError: loadEvents is not defined
```

### 4. Проверьте UI
- ✅ Прогресс-бар показывает два этапа (генерация + загрузка)
- ✅ Финальный toast: "Успешно создано XXX событий"
- ✅ События отображаются в календаре
- ✅ Весь процесс занимает ~20-30 секунд (для 500 событий)

## 🐛 Если что-то не так

### Ошибка: loadEvents is not defined
- ❌ Bugfix не применён
- Проверьте что файл `/components/scheduler/SchedulerMain.tsx` обновлён
- Должна быть строка: `const { eventsApi } = await import('../../services/api');`

### События не все загрузились
1. Проверьте финальный toast - сколько успешно, сколько ошибок
2. Проверьте консоль на логи `❌ Ошибка загрузки пачки`
3. Проверьте логи сервера в Supabase Dashboard

### Слишком долго загружается
1. Проверьте консоль - delay должен быть 2 секунды (не 30!)
2. Проверьте что используется batch endpoint (`📤 Загрузка пачки...`)
3. Для 500 событий должно быть ~25 секунд общее время
