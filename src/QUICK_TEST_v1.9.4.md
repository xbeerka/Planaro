# Быстрый тест автообновления v1.9.4

## 🧪 Как протестировать расширенный polling

### Подготовка

1. Откройте приложение в **двух разных браузерах** (или в обычном и инкогнито режиме)
2. Войдите под **двумя разными пользователями** с email @kode.ru
3. Откройте **один и тот же воркспейс** в обоих браузерах

### Тест 1: События (10 секунд)

**В браузере A:**
1. Создайте новое событие
2. Перетащите событие на другую неделю

**В браузере B:**
- ✅ Событие должно появиться через 0-10 секунд
- ✅ Перемещение должно синхронизироваться через 0-10 секунд

**Логи в консоли B:**
```
🔄 Polling: проверка обновлений событий...
✅ События: обнаружены изменения, обновление UI
```

---

### Тест 2: Сотрудники (15 секунд)

**В браузере A:**
1. Откройте модальное окно "Управление сотрудниками"
2. Добавьте нового сотрудника "Тест Тестович"
3. Нажмите "Сохранить"

**В браузере B:**
- ✅ Новый сотрудник должен появиться в календаре через 0-15 секунд
- ✅ Новая строка с сотрудником появится автоматически

**Логи в консоли B:**
```
🔄 Polling: проверка обновлений сотрудников...
✅ Сотрудники: обнаружены изменения, обновление UI
```

**В браузере A:**
1. Измените имя сотрудника на "Новое Имя"
2. Нажмите "Сохранить"

**В браузере B:**
- ✅ Имя должно обновиться через 0-15 секунд

---

### Тест 3: Департаменты (15 секунд)

**В браузере A:**
1. Откройте модальное окно "Управление отделами"
2. Добавьте новый отдел "QA"
3. Нажмите "Сохранить"

**В браузере B:**
- ✅ Новый отдел должен появиться в календаре через 0-15 секунд
- ✅ Отдел появится в структуре календаря

**Логи в консоли B:**
```
🔄 Polling: проверка обновлений департаментов...
✅ Департаменты: обнаружены изменения, обновление UI
```

**В браузере A:**
1. Переименуйте отдел "QA" → "Testing"
2. Нажмите "Сохранить"

**В браузере B:**
- ✅ Название отдела должно обновиться через 0-15 секунд

**В браузере A:**
1. Измените порядок отделов (drag & drop)
2. Нажмите "Сохранить"

**В браузере B:**
- ✅ Порядок отделов должен синхронизироваться через 0-15 секунд

---

### Тест 4: Проекты (15 секунд)

**В браузере A:**
1. Откройте модальное окно "Управление проектами"
2. Добавьте новый проект "Мобильное приложение"
3. Выберите цвет
4. Нажмите "Сохранить"

**В браузере B:**
- ✅ Новый проект должен появиться в селекте "Проект" через 0-15 секунд
- ✅ Если создать событие с этим проектом, цвет будет правильный

**Логи в консоли B:**
```
🔄 Polling: проверка обновлений проектов...
✅ Проекты: обнаружены изменения, обновление UI
```

**В браузере A:**
1. Измените цвет проекта на другой
2. Нажмите "Сохранить"

**В браузере B:**
- ✅ Цвет должен обновиться через 0-15 секунд
- ✅ Существующие события с этим проектом изменят цвет

---

### Тест 5: Защита от дублирования

**В браузере A:**
1. Создайте событие
2. **Сразу же** (в течение 2 секунд) проверьте консоль

**Логи в консоли A:**
```
✅ Событие создано: e123
⏭️ Пропуск polling событий - недавно было локальное изменение
```

**Ожидаемое поведение:**
- ✅ Polling событий пропускается в течение 2 секунд после создания
- ✅ Это предотвращает лишние запросы к серверу
- ✅ После 2 секунд polling возобновляется нормально

**То же самое работает для:**
- Создание/редактирование сотрудника → пропуск polling сотрудников
- Создание/редактирование отдела → пропуск polling департаментов
- Создание/редактирование проекта → пропуск polling проектов

---

### Тест 6: Нет изменений = нет обновлений

**Проверка:**
1. Не делайте никаких изменений в течение 1 минуты
2. Смотрите логи в консоли

**Ожидаемое поведение:**
```
🔄 Polling: проверка обновлений событий...
🔄 Polling: проверка обновлений сотрудников...
🔄 Polling: проверка обновлений департаментов...
🔄 Polling: проверка обновлений проектов...
```

**Логи НЕ должны содержать:**
```
❌ НЕ должно быть: "✅ обнаружены изменения"
```

**Почему:**
- Polling выполняется, но UI не обновляется
- Сравнение через `JSON.stringify` предотвращает лишние ре-рендеры
- Это экономит ресурсы браузера

## 📊 Проверка нагрузки на сервер

### Откройте DevTools → Network

**Фильтр:** XHR  
**Временной промежуток:** 1 минута

**Ожидаемые запросы:**
- `/resources` - 4 запроса (каждые 15 секунд)
- `/departments` - 4 запроса (каждые 15 секунд)
- `/projects` - 4 запроса (каждые 15 секунд)
- `/events` - 6 запросов (каждые 10 секунд)

**Итого:** ~18 запросов за минуту ✅

## 🐛 Возможные проблемы

### Проблема: Изменения не синхронизируются

**Проверить:**
1. Оба пользователя в одном воркспейсе?
2. Есть ли интернет соединение?
3. Сервер работает? (проверьте `/health`)
4. Есть ли ошибки в консоли?

### Проблема: Слишком частые обновления

**Проверить:**
1. Нет ли ошибок в логах "Пропуск polling"?
2. Не зацикливается ли что-то?
3. Проверьте Network tab на лишние запросы

### Проблема: Данные не обновляются в UI

**Проверить:**
1. Есть ли логи "✅ обнаружены изменения"?
2. Если логи есть, но UI не меняется → ошибка рендеринга
3. Перезагрузите страницу
