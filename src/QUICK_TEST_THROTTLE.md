# Quick Test - Request Throttling

Быстрый тест для проверки работы системы throttling и мониторинга запросов.

---

## ✅ Шаг 1: Проверка debug commands

1. Откройте приложение в браузере
2. Откройте консоль разработчика (F12)
3. Вы должны увидеть подсказку:

```
💡 TIP: Введите debugHelp() для справки по debug командам
```

4. Введите команду:

```javascript
debugHelp()
```

**Ожидаемый результат:**

```
============================================================
🔧 DEBUG COMMANDS:

   debugRequests()  - Статистика запросов (req/min)
   debugThrottle()  - Активные throttled запросы
   debugHelp()      - Эта справка

Пример:
   > debugRequests()
   📊 events-delta-sync: 15 req/min
   📊 projects-sync: 4 req/min
============================================================
```

---

## ✅ Шаг 2: Проверка статистики запросов

Подождите 60 секунд (чтобы накопилась статистика), затем введите:

```javascript
debugRequests()
```

**Ожидаемый результат (пример):**

```
============================================================
📊 REQUEST MONITOR REPORT (последняя минута):
   Всего запросов: 28
   ✅ events-delta-sync-w123: 15 req/min
   ✅ events-full-sync-w123: 2 req/min
   ✅ projects-sync-w123: 4 req/min
   ✅ resources-sync-w123: 4 req/min
   ✅ departments-sync-w123: 4 req/min
   ✅ presence-fetch-w123: 2 req/min
============================================================
```

**Что проверять:**

- ✅ Нет запросов с частотой > 50 req/min
- ✅ events-delta-sync: ~15 req/min (норма для 4 секундного интервала)
- ✅ projects/resources/departments: ~4 req/min (норма для 15 секундного интервала)

**Если видите:**

```
⚠️ events-delta-sync-w123: 75 req/min
```

Это означает что есть **утечка** (несколько setInterval работают одновременно).

## ✅ Шаг 3: Проверка активных запросов

В любой момент времени введите:

```javascript
debugThrottle()
```

**Ожидаемый результат:**

```
============================================================
🔒 ACTIVE THROTTLED REQUESTS:
   Активных запросов: 2 / 10 (MAX_CONCURRENT)
   ✅ Нет активных запросов
============================================================
```

**Или (если запросы выполняются):**

```
============================================================
🔒 ACTIVE THROTTLED REQUESTS:
   Активных запросов: 3 / 10 (MAX_CONCURRENT)
============================================================
```

**Что проверять:**

- ✅ Активных запросов < 8 (норма)
- ⚠️ Активных запросов >= 8 (близко к лимиту)
- ❌ Активных запросов >= 10 (превышен лимит)

## ❌ Возможные проблемы

### Проблема 1: Высокая частота запросов

```
⚠️ events-delta-sync-w123: 75 req/min
```

**Причина**: Несколько setInterval работают одновременно (утечка).

**Решение**: Проверить что useEffect корректно возвращает cleanup функцию.

### Проблема 2: Превышен лимит активных запросов

```
⚠️ Throttle: Превышен лимит одновременных запросов (10/10)
```

**Причина**: Слишком много долгих запросов.

**Решение**: Уменьшить MAX_CONCURRENT с 10 до 5 в `/utils/requestThrottle.ts`.

### Проблема 3: Debug commands не работают

```
Uncaught ReferenceError: debugHelp is not defined
```

**Причина**: Модуль `/utils/debugCommands.ts` не импортирован.

**Решение**: Проверить что в `/App.tsx` есть `import './utils/debugCommands';`.
