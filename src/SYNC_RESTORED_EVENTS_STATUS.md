**Алгоритм**:
1. Проверяет какие события отсутствуют на сервере (`loadedEventIds`)
2. Очищает пометки удаления для восстанавливаемых событий
3. Создаёт batch операцию `op: 'create'` с существующими ID
4. Отправляет на сервер через `/events/batch`
5. Обновляет `loadedEventIds` и state событий
6. Помечает локальное изменение для блокировки синхронизации

## 🎯 Как это работает

### Сценарий: Пользователь удалил событие → Undo

1. **Удаление события**:
   - Событие удаляется из state: `setEvents(prev => prev.filter(...))`
   - Событие удаляется с сервера: `DELETE /events/:id`
   - ID удаляется из `loadedEventIds`
   - ID добавляется в `deletedEventIdsRef` для защиты от "воскрешения"

2. **Пользователь нажимает Ctrl+Z**:
   - `handleUndo()` вызывает `historyUndo()` из useHistory
   - Восстанавливается предыдущее состояние событий (включая удалённое)
   - `setEvents(state.events)` добавляет событие обратно в UI

3. **Автоматическая синхронизация**:
   - `syncRestoredEventsToServer(uniqueEvents)` вызывается автоматически
   - Функция проверяет: событие НЕ в `loadedEventIds` → нужно создать на сервере
   - Отправляется batch запрос с `op: 'create'` и **существующим ID**
   - Сервер выполняет UPSERT: создаёт событие с тем же ID что был раньше
   - `loadedEventIds` обновляется, событие теперь считается "существующим на сервере"

4. **Защита от удаления**:
   - Full Sync через 30 секунд НЕ удалит это событие, т.к. оно есть в `loadedEventIds`
   - Delta Sync проигнорирует его (локальное изменение < 2 сек назад)
