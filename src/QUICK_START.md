# 🚀 Быстрый старт - Planaro v2.3.7

**Время на запуск**: ~5 минут  
**Статус**: ✅ Production Ready

---

## 1️⃣ Первый запуск

### Шаг 1: Откройте приложение
```
URL: https://your-domain.com
```

### Шаг 2: Авторизация
1. Введите **корпоративный email @kode.ru**
2. Введите пароль **ИЛИ** получите OTP код
3. Нажмите "Войти"

✅ **Сессия сохраняется на 30 дней** - не нужно входить каждый раз

---

## 2️⃣ Создание воркспейса

### Шаг 1: Список воркспейсов
После входа вы увидите экран "Управление рабочими пространствами"

### Шаг 2: Создать новый воркспейс
1. Нажмите **"➕ Создать воркспейс"**
2. Введите **название** (например, "2025")
3. Выберите **год** (например, 2025)
4. Нажмите **"Создать"**

✅ **Воркспейс создан** - можно начинать работу

---

## 3️⃣ Настройка структуры

### Добавление департаментов
1. Откройте воркспейс из списка
2. Нажмите **"👥 Департаменты"**
3. Нажмите **"+ Добавить департамент"**
4. Введите название (например, "Backend")
5. Добавьте ещё департаменты (можно добавить несколько пустых строк)
6. Нажмите **"💾 Сохранить"**

### Добавление проектов
1. Нажмите **"📁 Проекты"**
2. Нажмите **"+ Добавить проект"**
3. Введите название (например, "Проект Alpha")
4. Выберите цвета (автогенерация по клику на превью)
5. Добавьте ещё проекты
6. Нажмите **"💾 Сохранить"**

### Добавление сотрудников
1. Нажмите **"👤 Сотрудники"**
2. Нажмите **"+ Добавить сотрудника"**
3. Введите **имя**, **email**, выберите **департамент**, **грейд**, **компанию**
4. Добавьте ещё сотрудников
5. Нажмите **"💾 Сохранить"**

✅ **Структура готова** - можно создавать события

---

## 4️⃣ Работа с событиями

### Способ 1: Через модальное окно
1. Нажмите **"➕ Создать событие"** в тулбаре
2. Выберите **сотрудника** и **проект**
3. Укажите **неделю**, **продолжительность**, **высоту**
4. Нажмите **"Создать"**

### Способ 2: Drag & Drop
1. Наведите на существующее событие
2. **Зажмите левую кнопку мыши**
3. **Перетащите** на новую позицию
4. **Отпустите** - событие сохранится автоматически

### Способ 3: Resize
1. Наведите на **край события** (появятся ручки)
2. **Зажмите и тяните** ручку
3. **Отпустите** - новый размер сохранится

### Режим ножниц (разделение событий)
1. Нажмите **"✂️ Ножницы"** в тулбаре
2. **Кликните на границу недели** внутри события
3. Событие разделится на два
4. Нажмите **"✂️ Ножницы"** ещё раз для выхода

---

## 5️⃣ Генерация тестовых данных

### Автоматическое заполнение календаря
1. Убедитесь что есть **сотрудники** (хотя бы 10)
2. Убедитесь что есть **проекты** (хотя бы 3)
3. Нажмите **"🧪 Генерация"** в тулбаре
4. Подтвердите генерацию
5. Дождитесь завершения (~30-60 секунд для 85 сотрудников)

**Результат**: Каждый сотрудник получит события на 100% календаря (без пустот)

**Ожидаемые логи**:
```
✅ Генерация завершена без ошибок:
   - Успешно отправлено: 8053
   - Реально создано в БД: 8053
   - Потеряно: 0
   - Integrity rate: 100.0%
```

## 6️⃣ Полезные функции

### Горячие клавиши
- **Ctrl+Z** - Отмена последнего действия
- **Ctrl+Shift+Z** - Повтор действия
- **Space + drag** - Панорамирование календаря
- **Ctrl + scroll** - Зум
- **?** - Справка по всем хоткеям

### Фильтрация
1. Нажмите **"🔍 Фильтры"** в тулбаре
2. Выберите **департаменты** для отображения
3. Выберите **проекты** для подсветки
4. Нажмите **"Применить"**

**Эффект**: 
- Скрыты сотрудники из других департаментов
- События других проектов - серые и полупрозрачные

### Настройка размеров
1. Нажмите на иконку **размеров** (📏)
2. Используйте слайдеры:
   - **Ширина недели**: 48-220px
   - **Высота строки**: 48-144px
3. Изменения сохраняются автоматически в cookies

### Очистка событий
1. Нажмите **"🗑️ Очистить"** в тулбаре
2. Подтвердите удаление
3. Все события воркспейса будут удалены

⚠️ **ВНИМАНИЕ**: Это действие нельзя отменить!

## 7️⃣ Совместная работа

### Онлайн пользователи
- **В карточках воркспейсов**: видны все пользователи ВНУТРИ воркспейса
- **Текущий пользователь**: выделен зелёным с меткой "(вы)"
- **Другие пользователи**: синие аватарки

### Автообновление данных
Все изменения синхронизируются автоматически:
- **События**: каждые 10 секунд
- **Сотрудники, департаменты, проекты**: каждые 15 секунд

**Пример**:
1. Пользователь А создаёт событие
2. Через 10 секунд Пользователь Б видит это событие у себя
3. Никаких дополнительных действий не требуется

## 8️⃣ Профиль пользователя

### Изменение имени и аватарки
1. Нажмите на **аватарку** в правом верхнем углу
2. Выберите **"Редактировать профиль"**
3. Измените **имя** (отображается другим пользователям)
4. Загрузите **аватарку** (макс 5MB, автоматический ресайз до 256x256px)
5. Нажмите **"Сохранить"**

✅ **Страница перезагрузится** через 2 секунды для обновления токена

## 🆘 Частые проблемы

### Сервер недоступен
**Проблема**: При запуске показывается красный экран "Сервер недоступен"

**Решение**:
```bash
# Задеплойте Edge Function
supabase functions deploy make-server-73d66528

# Проверьте health check
curl https://{PROJECT_ID}.supabase.co/functions/v1/make-server-73d66528/health
```

### События не загружаются (< 100%)
**Проблема**: Создано 8053 событий, но загружено меньше

**Решение**: Проверьте версию - должна быть **v2.3.7** с пагинацией

**Проверка версии**:
```bash
# Логи сервера должны показывать:
📦 Загрузка страницы 1: range(0, 999)...
📦 Загрузка страницы 2: range(1000, 1999)...
...
✅ Загрузка завершена: 8053 событий за 9 запросов
```

### Онлайн пользователи не отображаются
**Проблема**: В карточках воркспейсов не видны аватарки

**Решение**:
1. Убедитесь что пользователь ВНУТРИ воркспейса (открыл календарь)
2. Проверьте что сервер деплоился (presence endpoint работает)
3. Проверьте логи:
```
📡 Heartbeat от user@kode.ru для workspace 123
✅ Presence сохранён с TTL 60 сек
```

### Аватарка не появилась после загрузки
**Проблема**: Загрузил аватарку, но она не отображается

**Решение**: Страница должна **перезагрузиться через 2 секунды** после сохранения (для обновления токена)

Если не перезагрузилась - обновите вручную (F5)
