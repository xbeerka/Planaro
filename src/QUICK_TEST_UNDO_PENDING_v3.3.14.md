# 🧪 Быстрый тест: Undo Pending Flush v3.3.14

## Сценарий 1: Drag → Undo → БЕЗ полосок

1. **Открой календарь**
   - Зайди в воркспейс
   - Открой любой департамент

2. **Создай событие**
   - Двойной клик на ресурс
   - Событие создаётся с временным ID `ev_temp_XXX`
   - Через ~500ms получает реальный ID с сервера

3. **Быстрый drag (< 2 сек)**
   - Drag события на другую неделю
   - Debounced save начинается (2 сек задержка)
   - НЕ жди окончания!

4. **Быстрый Undo**
   - Нажми `Ctrl+Z` (или `Cmd+Z`)
   - **БЫЛ БАГ**: событие возвращается, но показывает полоски загрузки
   - **ИСПРАВЛЕНО**: событие возвращается мгновенно, БЕЗ полосок ✅

5. **Проверь консоль**
   ```
   ✅ UNDO: Pending изменения сохранены перед undo
   🔄 UNDO/REDO: ↩️ Undo начат - мгновенное восстановление из истории
   ✅ UNDO/REDO: ✅ Undo завершён успешно
   ```

## Сценарий 2: Нет toast warning

1. **Создай событие**
   - Двойной клик
   - Событие с временным ID `ev_temp_XXX`

2. **Быстрый Undo (< 500ms)**
   - Нажми `Ctrl+Z` ДО того как событие сохранится
   - **БЫЛ БАГ**: Toast уведомление "Подождите, дождитесь завершения создания событий"
   - **ИСПРАВЛЕНО**: БЕЗ toast, только console.log ✅

3. **Проверь консоль**
   ```
   ⏸️ UNDO: Заблокировано - есть события в процессе создания
   ```

## Сценарий 3: Resize → Undo → БЕЗ полосок

1. **Создай событие**
   - Двойной клик
   - Дождись сохранения на сервере

2. **Быстрый resize**
   - Drag правую ручку (расширить событие)
   - Debounced save начинается

3. **Быстрый Undo**
   - Нажми `Ctrl+Z` ДО окончания save
   - **ИСПРАВЛЕНО**: событие возвращается БЕЗ полосок ✅

## Ожидаемый результат

**✅ Все сценарии должны работать:**
- БЕЗ полосок загрузки после undo
- БЕЗ toast warning при блокировке
- Только console.log для диагностики
- События мгновенно восстанавливаются

## Как это работает

```typescript
// 1. Flush pending ПЕРЕД undo
await flushPendingChanges(); // ← Сохраняет все изменения!

// 2. Проверка pending событий
const hasPendingEvents = events.some(e => e.id.startsWith('ev_temp_'));
if (hasPendingEvents) {
  console.warn('⏸️ UNDO: Заблокировано');
  return; // ← БЕЗ toast!
}

// 3. Undo с ПУСТОЙ очередью
const state = historyUndo();
setEvents(state.events); // ← Нет pending → нет полосок!
```

## Диагностика

Если видишь полоски после undo:
1. Проверь консоль: есть ли `✅ UNDO: Pending изменения сохранены`?
2. Проверь сеть: есть ли BATCH запрос перед undo?
3. Проверь `pendingChangesRef`: он должен быть пустой после flush
