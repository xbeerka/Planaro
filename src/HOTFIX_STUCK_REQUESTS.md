# Hotfix - Застрявшие запросы блокируют throttle

**Дата**: 2025-12-05  
**Версия**: 1.0.3  
**Тип**: Hotfix

---

## 🐛 Проблема

```
⚠️ Throttle: Пропуск дубликата "presence-batch-all-workspaces" (уже выполняется 15000ms)
⚠️ Batch запрос: пропущен (throttle)
```

**Причина**: 

Запрос `presence-batch-all-workspaces` выполняется **слишком долго** (15 секунд) и блокирует следующий запрос с тем же ID. Возможные причины:

1. **Запрос реально медленный** - Edge Function обрабатывает большой batch
2. **Запрос завис** - таймаут или сетевая проблема
3. **Ошибка без завершения** - exception до вызова `completeRequest()`

**Старая логика**:

Cleanup для "застрявших" запросов выполнялся **каждые 60 секунд**. Если запрос завис на 15 секунд, он блокирует следующий запрос на **45 секунд** (до следующего cleanup).

---

## ✅ Решение

Добавлена **автоматическая очистка застрявших запросов** и **логирование медленных запросов**.

### 1. Логирование медленных запросов

Теперь throttle логирует запросы которые выполняются **более 5 секунд**:

```typescript
// В throttledRequest()
const startTime = Date.now();
const result = await request();

const elapsed = Date.now() - startTime;
if (elapsed > 5000) {
  console.warn(`🐌 Throttle: Медленный запрос "${requestId}" (${Math.round(elapsed/1000)}s)`);
}
```

**Результат**: Видим какие запросы тормозят систему.

### 2. Автоматическая очистка застрявших запросов

Новая функция `cleanupStuckRequests()` принудительно завершает запросы которые:

- **НЕ завершены** (completed = false)
- **Выполняются более 20 секунд**

```typescript
cleanupStuckRequests(): void {
  const now = Date.now();
  const STUCK_THRESHOLD = 20000; // 20 секунд
  
  for (const [id, request] of this.pendingRequests.entries()) {
    if (!request.completed && (now - request.timestamp > STUCK_THRESHOLD)) {
      console.error(`⚠️ Throttle: ЗАСТРЯВШИЙ запрос "${id}" (${Math.round((now - request.timestamp)/1000)}s) → принудительно завершён`);
      this.pendingRequests.delete(id);
    }
  }
}
```

### 3. Частая очистка застрявших запросов

Cleanup для застрявших запросов теперь выполняется **каждые 10 секунд** (было 60 секунд):

```typescript
// Периодическая очистка завершённых запросов каждые 60 секунд
setInterval(() => {
  requestThrottle.cleanup();
}, 60000);

// Периодическая очистка застрявших запросов каждые 10 секунд
setInterval(() => {
  requestThrottle.cleanupStuckRequests();
}, 10000);
```

**Результат**: Застрявшие запросы блокируют систему максимум на **20-30 секунд** (было 60+ секунд).

## 🔧 Исправленный файл

### `/utils/requestThrottle.ts`

**Изменения:**

1. Добавлена функция `cleanupStuckRequests()` - очистка застрявших запросов
2. Изменена `cleanup()` - теперь очищает только завершённые запросы
3. Добавлен `setInterval` для `cleanupStuckRequests()` каждые 10 секунд
4. Добавлено логирование медленных запросов в `throttledRequest()` (> 5 сек)

**Параметры:**

- `STUCK_THRESHOLD = 20000` (20 секунд) - застрявшие запросы
- `STALE_THRESHOLD = 30000` (30 секунд) - устаревшие завершённые запросы
- Cleanup каждые 60 секунд (завершённые)
- Cleanup каждые 10 секунд (застрявшие)

## 🔍 Диагностика медленных запросов

Если видите много медленных запросов:

```
🐌 Throttle: Медленный запрос "presence-batch-all-workspaces" (15s)
🐌 Throttle: Медленный запрос "events-delta-sync-22" (8s)
🐌 Throttle: Медленный запрос "projects-sync-22" (7s)
```

**Возможные причины:**

1. **Edge Function перегружен** - много одновременных запросов
2. **База данных медленная** - сложные SQL запросы
3. **Сетевые проблемы** - медленное соединение
4. **Холодный старт** - Edge Function только запустился

**Решения:**

1. Уменьшить частоту polling (увеличить интервалы)
2. Оптимизировать SQL запросы на сервере
3. Добавить индексы в БД
4. Уменьшить размер batch запросов
