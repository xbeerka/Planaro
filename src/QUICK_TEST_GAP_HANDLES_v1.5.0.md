# Quick Test: Gap Handles v1.5.0

## 🎯 Цель
Проверить работу двустороннего resize границ между событиями через gap handles

---

## ✅ Тестовый чеклист

### 1. Подготовка данных

- [ ] Открыть календарь воркспейса
- [ ] Создать 2 события **на одном ресурсе и одной неделе**:
  - Event A: unitStart=0, unitsTall=2
  - Event B: unitStart=2, unitsTall=2
  - Они должны **касаться** (нижняя граница A = верхняя граница B)

### 2. Вертикальный gap (сверху-снизу)

- [ ] **Зажать Cmd/Ctrl** клавишу
- [ ] Навести курсор на **промежуток между Event A и Event B**
- [ ] Должна появиться **синяя горизонтальная линия с 3 кружками**
- [ ] Курсор должен измениться на `ns-resize` (стрелки вверх-вниз)
- [ ] **Перетащить handle вверх** (на -1 unit):
  - Event A должно стать: unitStart=0, unitsTall=**1** (уменьшилось)
  - Event B должно стать: unitStart=**1**, unitsTall=**3** (сдвинулось и увеличилось)
- [ ] **Перетащить handle вниз** (на +1 unit):
  - Event A должно стать: unitStart=0, unitsTall=**3** (увеличилось)
  - Event B должно стать: unitStart=**3**, unitsTall=**1** (сдвинулось и уменьшилось)
- [ ] Изменения должны сохраниться в БД
- [ ] **Undo (Cmd+Z)** - изменения должны откатиться
- [ ] **Redo (Cmd+Shift+Z)** - изменения должны вернуться

### 3. Горизонтальный gap (слева-справа)

- [ ] Создать 2 события **на одном ресурсе**:
  - Event C: startWeek=0, weeksSpan=5, unitStart=0, unitsTall=2
  - Event D: startWeek=5, weeksSpan=5, unitStart=0, unitsTall=2
  - Они должны **касаться** (правая граница C = левая граница D)
- [ ] **Зажать Cmd/Ctrl** клавишу
- [ ] Навести курсор на **промежуток между Event C и Event D** (граница недели 5)
- [ ] Должна появиться **синяя вертикальная линия с 3 кружками**
- [ ] Курсор должен измениться на `ew-resize` (стрелки влево-вправо)
- [ ] **Перетащить handle влево** (на -1 week):
  - Event C должно стать: startWeek=0, weeksSpan=**4** (уменьшилось)
  - Event D должно стать: startWeek=**4**, weeksSpan=**6** (сдвинулось и увеличилось)
- [ ] **Перетащить handle вправо** (на +1 week):
  - Event C должно стать: startWeek=0, weeksSpan=**6** (увеличилось)
  - Event D должно стать: startWeek=**6**, weeksSpan=**4** (сдвинулось и уменьшилось)

### 4. Валидация границ

- [ ] Попробовать уменьшить событие до 0 units/weeks
  - Gap handle **не должен** уменьшить событие меньше 1 unit/week
- [ ] Попробовать вытащить событие за пределы сетки (unit > 15 или week > 51)
  - Gap handle **не должен** позволить выйти за пределы

### 5. Многонедельные события

- [ ] Создать события которые занимают несколько недель:
  - Event E: startWeek=0, weeksSpan=**5**, unitStart=0, unitsTall=2
  - Event F: startWeek=0, weeksSpan=**5**, unitStart=2, unitsTall=2
  - Они касаются по вертикали на всех 5 неделях
- [ ] **Зажать Cmd/Ctrl**
- [ ] Gap handle должен **растянуться на все 5 недель**
- [ ] Drag handle изменяет оба события одновременно

### 6. События на разных неделях (горизонтальный gap)

- [ ] Создать события с частичным вертикальным перекрытием:
  - Event G: startWeek=0, weeksSpan=3, unitStart=**0**, unitsTall=4
  - Event H: startWeek=3, weeksSpan=3, unitStart=**1**, unitsTall=4
  - Они касаются по горизонтали и перекрываются по вертикали (units 1-4)
- [ ] **Зажать Cmd/Ctrl**
- [ ] Gap handle должен появиться только на **перекрывающихся units** (1-4)
- [ ] Высота handle = 3 units

### 7. Отсутствие gaps

- [ ] События НЕ касаются (есть промежуток) - **handle НЕ должен появиться**
- [ ] События на разных ресурсах - **handle НЕ должен появиться**
- [ ] **Отпустить Cmd/Ctrl** - все handles должны **исчезнуть**

### 8. Интеграция с другими фичами

- [ ] **Scissors mode** - gap handles НЕ мешают режиму ножниц
- [ ] **Comment mode** - gap handles НЕ мешают режиму комментариев
- [ ] **Drag события** - gap handles появляются только при Cmd, не мешают drag
- [ ] **Resize события** - gap handles НЕ мешают обычному resize

### 9. Delta Sync

- [ ] После gap drag подождать 2 секунды
- [ ] Delta Sync НЕ должен перезаписать изменения
- [ ] После 4 секунд Delta Sync должен возобновиться

### 10. Производительность

- [ ] Календарь с **100+ событиями**
- [ ] Зажать Cmd/Ctrl - gaps должны **мгновенно** вычислиться
- [ ] Drag gap handle - **плавное** движение без лагов

---

## 🐛 Известные Edge Cases

1. **События перекрываются но не касаются** → gap НЕ создаётся ✅
2. **События касаются по диагонали** → gap НЕ создаётся (пока не поддерживается) ✅
3. **Более 2 событий касаются в одной точке** → создаётся gap для каждой пары ✅
4. **Gap drag отменён (Esc)** → события восстанавливаются ✅
5. **Невалидное изменение** → drag игнорируется ✅

---

## 📊 Ожидаемый результат

- ✅ Gap handles появляются только при зажатой Cmd/Ctrl
- ✅ Handles рендерятся на всю ширину/высоту пересечения событий
- ✅ Drag изменяет оба события одновременно
- ✅ Валидация предотвращает некорректные изменения
- ✅ Undo/Redo работает корректно
- ✅ Polling блокируется на 2 секунды
- ✅ Производительность не страдает

## 🔧 Debug

Если что-то не работает:

1. Проверь консоль браузера:
   ```
   🎯 Gap drag начат: { type: 'vertical', event1: '...', event2: '...' }
   ✅ Gap drag завершён с изменениями: { delta: -1, ... }
   ✅ Gap изменения сохранены на сервер
   ```

2. Проверь что `isCtrlPressed = true` (в React DevTools)

3. Проверь что gaps вычислились: `eventGaps.length > 0`

4. Проверь что события **касаются**:
   - Вертикально: `event1.unitStart + event1.unitsTall === event2.unitStart`
   - Горизонтально: `event1.startWeek + event1.weeksSpan === event2.startWeek`
