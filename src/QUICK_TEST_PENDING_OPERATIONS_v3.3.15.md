# Быстрый тест проверки pending операций (v3.3.15)

## Как проверить что исправление работает

### Тест 1: Toast показывается при pending операции

1. Открой консоль браузера (F12)
2. Создай событие (клик на календаре)
3. **СРАЗУ** нажми Ctrl+Z (в течение 500ms)
4. **Ожидаемый результат**:
   - ✅ В консоли: `⏳ hasPendingOperations: 1 операций в очереди: create:ev_temp_...`
   - ✅ В консоли: `⏸️ UNDO: Заблокировано - есть pending операции`
   - ✅ Toast появляется: "Подождите - Дождитесь завершения сохранения изменений"
   - ✅ Undo заблокирован

### Тест 2: Блокировка снимается после сохранения

1. Создай событие
2. Подожди 2 секунды (событие полностью сохранится)
3. Нажми Ctrl+Z
4. **Ожидаемый результат**:
   - ✅ В консоли: `✅ Удалена pending операция для события ev_temp_...`
   - ✅ НЕТ лога `⏳ hasPendingOperations`
   - ✅ НЕТ toast уведомления
   - ✅ Undo выполняется (событие исчезает)

### Тест 3: Нет "вечной блокировки"

1. Создай событие → подожди 2 сек → drag → подожди 2 сек
2. Нажми Ctrl+Z (undo drag)
3. Подожди 2 сек
4. Нажми Ctrl+Z ещё раз (undo create)
5. **Ожидаемый результат**:
   - ✅ Первый Undo работает (событие возвращается на исходное место)
   - ✅ Второй Undo работает (событие исчезает)
   - ✅ НЕТ "вечной блокировки"
   - ✅ НЕТ постоянного toast

### Тест 4: Проверка логов pending операций

Открой консоль и выполни:

```javascript
// Создай событие
// В консоли должно появиться:
⏳ createEvent: событие ev_temp_1732005123456_78901 добавлено в pending queue (op: create)

// Через 500ms начнётся batch save:
📦 BATCH v4.0.0: отправка 1 изменений на сервер...
📦 BATCH: событие ev_temp_1732005123456_78901 → create (isTemporary=true, isLoaded=false)

// После успешного create:
✅ BATCH: успешно - created: 1, updated: 0
🔄 BATCH: замена ev_temp_1732005123456_78901 → e123
✅ Удалена pending операция для события ev_temp_1732005123456_78901

// Если нажать Ctrl+Z СРАЗУ после создания (до замены ID):
⏳ hasPendingOperations: 1 операций в очереди: create:ev_temp_1732005123456_78901
⏸️ UNDO: Заблокировано - есть pending операции
```

## Отладка проблем

### Проблема: Toast не показывается

**Проверка 1**: Открой консоль, найди строку:
```
⏸️ UNDO: Заблокировано - есть pending операции
```

Если этой строки НЕТ → `hasPendingOperations()` вернул `false`

**Проверка 2**: Найди строку:
```
⏳ hasPendingOperations: N операций в очереди: ...
```

Если этой строки НЕТ → pending операций нет

**Проверка 3**: Проверь timing:
```
1. createEvent вызван в 12:00:00.000
2. Ctrl+Z нажат в 12:00:00.100 (менее 500ms)
3. hasPendingOperations должен вернуть true
```

### Проблема: "Вечная блокировка"

**Проверка**: Найди строку:
```
✅ Удалена pending операция для события XXXXX
```

Если этой строки НЕТ для вашего события → операция не была удалена из очереди

**Причина**: Событие НЕ сохранилось на сервере (ошибка batch save)

**Решение**: Проверь логи сервера, есть ли ошибки

## Timeline создания события

```
T+0ms:    createEvent() вызван
          ├─ Создан tempEvent с id = ev_temp_1732005123456_78901
          ├─ pendingOps.addPending(tempId, 'create', ...)
          ├─ queueEventUpdate(tempId, tempEvent)
          └─ return tempEvent

T+500ms:  Debounced save срабатывает
          ├─ BATCH request отправлен
          └─ Ожидание ответа...

T+700ms:  BATCH response получен
          ├─ results.created = [{ id: 'e123', ... }]
          ├─ setEventsState: ev_temp_... → e123
          ├─ loadedEventIds.add('e123')
          └─ pendingOps.removePending('ev_temp_...') ← БЛОКИРОВКА СНЯТА

T+800ms:  Ctrl+Z нажат
          ├─ hasPendingOperations() → false (операция удалена)
          ├─ НЕТ toast
          └─ Undo выполняется
```

## Важные моменты

1. **Проверка идёт по pending операциям**, а не по временным ID в state
2. **Блокировка автоматически снимается** после `pendingOps.removePending()`
3. **Toast показывается через queueMicrotask** чтобы избежать React warning
4. **Логирование детальное** для диагностики проблем
