# 🚀 Быстрый старт Realtime Broadcast

## ✅ Что нужно сделать (2 шага)

### Шаг 1: Проверьте Supabase Dashboard

1. Откройте ваш проект в **Supabase Dashboard**
2. **Settings** → **API**
3. Найдите **"Realtime"** секцию
4. Убедитесь что переключатель **"Enable Realtime"** включен ✅

**Скриншот где найти:**
```
Dashboard > Settings > API > Realtime Configuration
  ┌─────────────────────────────────────┐
  │ ☑️ Enable Realtime                  │
  └─────────────────────────────────────┘
```

---

### Шаг 2: Деплой кода

Если вы используете git + автодеплой:

```bash
git add .
git commit -m "feat: Realtime Broadcast для автообновления"
git push
```

Код обновится автоматически через ваш деплой сервис.

## 🧪 Проверка работы

### Тест с двумя пользователями:

1. Откройте приложение в **Chrome** (пользователь A)
2. Откройте в **Firefox** или **режиме инкогнито** (пользователь B)
3. Войдите **под разными @kode.ru email**
4. Откройте **один и тот же воркспейс**
5. **Пользователь A**: создайте событие
6. **Пользователь B**: должен увидеть событие **автоматически** (~1 секунда)

### В консоли вы увидите:

**У пользователя A** (создает событие):
```
📤 Отправка broadcast: create от пользователя abc-123
✅ Broadcast отправлен
📨 Получен broadcast: { action: 'create', userId: 'abc-123' }
⏭️ Пропуск своего broadcast (state уже обновлен)
```

**У пользователя B** (получает обновление):
```
📨 Получен broadcast: { action: 'create', userId: 'abc-123' }
🔔 Изменение от другого пользователя (create), перезагрузка...
🔄 Перезагрузка событий после изменений другого пользователя...
✅ События обновлены: 42
```
