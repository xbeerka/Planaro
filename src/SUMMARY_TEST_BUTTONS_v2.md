# 🧪 Тестовые кнопки генерации и очистки событий v2.2

## 📋 Обзор изменений

Реализованы две тестовые кнопки в Toolbar для массовых операций с событиями:
1. **"Сгенерировать эвенты"** (зеленая) - заполняет календарь до 10 событий на сотрудника
2. **"Очистить все эвенты"** (красная) - удаляет все события одним запросом

## ✨ Новые возможности

### 1. Генерация с заполнением календаря (макс 10 на сотрудника)

**Алгоритм генерации**:
```typescript
// Для каждого сотрудника:
// 1. Создаем сетку занятости: 52 недели × 4 юнита
// 2. Заполняем все ячейки событиями:
//    - Ищем свободную позицию
//    - Определяем максимальный размер события
//    - Выбираем случайный размер (короткий/длинный, узкий/широкий)
//    - Помечаем ячейки как занятые
//    - Повторяем пока есть свободные ячейки ИЛИ пока не достигнем лимита 10 событий
```

**Характеристики событий**:
- **Заполнение**: 100% доступного пространства (52 недели × 4 юнита = 208 ячеек)
- **Лимит**: максимум 10 событий на сотрудника
- **Длительность**:
  - 70% коротких (1-4 недели)
  - 30% длинных (5+ недель)
- **Высота**:
  - 80% однострочных (1 юнит)
  - 20% многострочных (2+ юнита)
- **Проекты**: случайные из текущего воркспейса
- **Результат**: красиво заполненный календарь без пропусков

**Оптимизации**:
- Батчинг по 50 событий
- Защита от бесконечного цикла (1000 попыток на сотрудника)
- Детальное логирование для каждого сотрудника
- Показ среднего количества событий на сотрудника

### 2. Оптимизированная очистка одним запросом

**Серверный endpoint**:
```typescript
DELETE /make-server-73d66528/events/clear/:workspaceId
```

**Что делает**:
1. Проверяет авторизацию
2. Подсчитывает количество событий
3. Удаляет ВСЕ события одним SQL запросом: `DELETE FROM events WHERE workspace_id = X`
4. Обновляет workspace summary
5. Возвращает количество удаленных событий

**Преимущества**:
- Один SQL запрос вместо N
- Быстро даже для тысяч событий
- Транзакционная безопасность
- Точный подсчет удаленных
