# 🚨 СРОЧНОЕ ИСПРАВЛЕНИЕ: Heartbeat Errors

## Проблема
```
⚠️ Heartbeat: сетевая ошибка - сервер может быть недоступен (попытка 1) - повтор через 30 сек
⚠️ OnlineUsers: сетевая ошибка, сервер может быть недоступен
```

## ⚡ Быстрое решение (2 команды)

### 1. Деплой сервера
```bash
supabase functions deploy make-server-73d66528
```

### 2. Проверка
```bash
curl https://YOUR_PROJECT_ID.supabase.co/functions/v1/make-server-73d66528/health
```

**Замените `YOUR_PROJECT_ID` на ваш Project ID** (найдите его в консоли браузера или в Supabase Dashboard → Settings → API)

## 🎯 Ожидаемый результат

### После деплоя:

### После проверки:

1. Откройте Supabase Dashboard: https://supabase.com/dashboard
2. Выберите ваш проект
3. Edge Functions → make-server-73d66528 → Logs
4. Ищите красные строки (ошибки)
5. Скопируйте ошибку и исправьте код

### Вариант D: 401 Unauthorized
Обновите ключи в `/utils/supabase/info.tsx`:
1. Supabase Dashboard → Settings → API
2. Скопируйте Project URL и anon/public key
3. Вставьте в файл

## 🎓 Почему это произошло?

### Причина
После изменений в коде сервера (`/supabase/functions/server/index.tsx`) нужно ВСЕГДА делать деплой. Без деплоя изменения остаются только локально и сервер на Supabase продолжает работать со старым кодом.

### Что изменилось
В последних коммитах были добавлены:
- Cache-busting для аватарок (`?t=timestamp`)
- Улучшенное логирование user_metadata
- Force refresh для токенов в ProfileModal

Эти изменения находятся в `/supabase/functions/server/index.tsx` и требуют деплоя.

### Как избежать в будущем
**ВСЕГДА** после изменений в `/supabase/functions/server/index.tsx` выполняйте:
```bash
supabase functions deploy make-server-73d66528
```
