# Быстрое тестирование: Блокировка Undo для pending событий (v3.3.12)

## 🎯 Что исправлено
**Проблема**: При быстром Undo сразу после создания события оно исчезало, но потом "воскресало" из БД через Delta Sync.

**Решение**: Undo/Redo теперь блокируется если есть события в процессе создания (со спиннером).

---

## ⚡ Быстрый тест (30 секунд)

### Тест 1: Блокировка Undo для pending события
```
1. Открой любой воркспейс
2. Дважды кликни на пустую ячейку
   → Создаётся событие со спиннером
3. СРАЗУ нажми Ctrl+Z (не дожидаясь исчезновения спиннера)

✅ Ожидается:
   - Появляется жёлтый toast "Подождите, дождитесь завершения создания событий"
   - Undo НЕ выполняется
   - Событие остаётся на календаре
   - Через ~500ms спиннер исчезает → событие получает реальный ID

❌ Было (до исправления):
   - Событие исчезает
   - Через 4 секунды "воскресает" из БД
```

### Тест 2: Нормальный Undo после завершения создания
```
1. Создай событие (двойной клик)
2. ПОДОЖДИ пока спиннер исчезнет (~500ms)
3. Нажми Ctrl+Z

✅ Ожидается:
   - Undo выполняется нормально
   - Событие удаляется
   - НЕ "воскресает"
```

### Тест 3: Множественные pending события
```
1. Быстро создай 3 события подряд (все со спиннерами)
2. Нажми Ctrl+Z

✅ Ожидается:
   - Toast "Подождите..."
   - Undo блокирован пока ВСЕ события не получат реальные ID
```

## 🔧 Технические детали для диагностики

### Как определить pending событие
```typescript
// Pending события имеют временный ID:
const isPending = event.id.startsWith('ev_temp_');

// Примеры ID:
// - ev_temp_1700000000000_abc123 ← pending
// - ev_temp_1700000001234_xyz789 ← pending
// - ev123 ← реальный ID (НЕ pending)
// - ev456 ← реальный ID (НЕ pending)
```

### Где смотреть код
```typescript
// Файл: /components/scheduler/SchedulerMain.tsx
// Строки: ~403-412 (handleUndo)
// Строки: ~502-511 (handleRedo)

const handleUndo = useCallback(async () => {
  // ✅ v3.3.12: Проверка pending событий
  const hasPendingEvents = events.some(e => e.id.startsWith('ev_temp_'));
  if (hasPendingEvents) {
    console.warn('⏸️ UNDO: Заблокировано - есть события в процессе создания');
    showToast({
      title: 'Подождите',
      description: 'Дождитесь завершения создания событий',
      variant: 'warning'
    });
    return; // блокируем Undo
  }
  // ... остальная логика
}, [events, showToast, ...]);
```

## 🏆 Критерии успеха

### ✅ Исправление работает если:
1. При быстром Undo после создания показывается toast "Подождите..."
2. Undo блокирован пока спиннер видимый
3. После исчезновения спиннера Undo работает нормально
4. События НЕ "воскресают" из БД
5. В консоли видны логи блокировки

### ❌ Исправление НЕ работает если:
1. События исчезают при Ctrl+Z (во время спиннера)
2. События "воскресают" через 4 секунды
3. Нет toast уведомления
4. Undo выполняется даже при pending событиях
