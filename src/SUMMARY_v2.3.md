## 📝 Что изменилось

### Файл: `/components/scheduler/SchedulerMain.tsx`

#### Функция `handleGenerateTestEvents()`

**Было:**
```typescript
const MAX_EVENTS_PER_RESOURCE = 10;

while (attempts < maxAttempts && createdCount < MAX_EVENTS_PER_RESOURCE) {
  // Создаём события пока не достигли лимита
  // Могут остаться пустые ячейки
}
```

**Стало:**
```typescript
const targetEventCount = Math.floor(Math.random() * 12) + 1; // 1-12

while (hasFreeCell() && iterations < maxIterations) {
  // Заполняем ВСЕ ячейки без пустот
  
  if (createdCount >= targetEventCount - 1) {
    // Последнее событие - максимального размера
    width = maxWidth;
    height = maxHeight;
  } else {
    // Случайный размер
    width = random(1, maxWidth);
    height = random(1, maxHeight);
  }
}
```

## 🧮 Алгоритм

### Прогрессивное заполнение

```
1. Для каждого сотрудника:
   - Случайное количество событий: 1-12
   
2. Создаём сетку занятости 52×4

3. Пока есть свободные ячейки:
   a) Найти первую свободную ячейку (слева-направо, сверху-вниз)
   b) Определить максимальные размеры от этой позиции
   c) ЕСЛИ это последнее событие (достигли лимита):
        → Создать МАКСИМАЛЬНЫЙ прямоугольник (заполнить ВСЁ)
      ИНАЧЕ:
        → Создать событие случайного размера
   d) Пометить ячейки как занятые
   e) Перейти к следующей ячейке

4. Результат: ВСЕ 208 ячеек заполнены
```

### Гарантия полного заполнения

**Ключевая логика:**
```typescript
// Когда осталось создать 1 событие - делаем его максимального размера
if (createdCount >= targetEventCount - 1) {
  width = maxWidth;   // Всё доступное по горизонтали
  height = maxHeight; // Всё доступное по вертикали
}
```

Это гарантирует что последнее событие **заполнит ВСЕ оставшиеся пробелы**.

Множество маленьких событий, последнее заполняет всё оставшееся.
