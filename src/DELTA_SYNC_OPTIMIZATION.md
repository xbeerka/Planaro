# Delta Sync Optimization - Исправление проблем

## Дата: 2025-11-18

## 🐛 Проблемы

### 1. Batch Update с временными ID (КРИТИЧНО)
**Симптомы:**
```
❌ [Supabase] ❌ BATCH update: событие e37103 не найдено в БД
❌ [Supabase] ❌ BATCH update: событие e37102 не найдено в БД
```

**Причина:**
- Система пыталась обновлять события с временными ID (начинаются с "e" и имеют timestamp: `e1732005123456789`)
- Временные ID используются для оптимистичных обновлений до создания события в БД
- Race condition: событие создано локально → добавлено в debounced queue → batch update пытается обновить до того, как событие было создано в БД

**Решение:**
```typescript
// Фильтруем временные ID перед отправкой batch update
const validChanges = Array.from(changes.entries()).filter(([id]) => {
  const numericPart = id.replace('e', '');
  const isTemporaryId = numericPart.length > 10; // Временные ID: e1732005123456789 (timestamp)
  
  if (isTemporaryId) {
    console.log(`⏭️ BATCH: пропуск временного ID ${id} (событие еще не создано в БД)`);
    return false;
  }
  return true;
});

if (validChanges.length === 0) {
  console.log('⏭️ BATCH: нет валидных изменений для отправки (все временные ID)');
  return;
}
```

**Эффект:**
- ✅ Batch update больше не пытается обновлять несуществующие события
- ✅ Временные ID автоматически заменяются на реальные при создании события
- ✅ Нет ошибок "Event not found"

### 2. Delta Sync работает слишком часто
**Симптомы:**
```
🔄 Delta Sync: загрузка изменений... { "since": "2025-11-18T08:58:18.010Z" }
📥 Delta Sync: получено 0 изменений
// ... каждые 2 секунды, даже когда нет изменений
```

**Причина:**
- Фиксированный интервал 4 секунды (слишком часто)
- Нет умной логики: система делала запросы даже когда нет активности
- Лишняя нагрузка на сервер и сеть

**Решение:**
```typescript
const SYNC_INTERVAL = 10000; // ⚡ 10 секунд (быстрая проверка)
const IDLE_SYNC_INTERVAL = 30000; // 💤 30 секунд (медленная проверка при отсутствии изменений)
const FULL_SYNC_INTERVAL = 60000; // 🔄 Полная синхронизация каждую минуту

let emptyResponsesCount = 0; // Счётчик пустых ответов подряд
let currentInterval = SYNC_INTERVAL;

// В delta sync:
if (changedEvents.length > 0) {
  // Есть изменения
  emptyResponsesCount = 0;
  
  // Возвращаемся к быстрому интервалу
  if (currentInterval !== SYNC_INTERVAL) {
    console.log('⚡ Delta Sync: возврат к быстрому интервалу (обнаружены изменения)');
    currentInterval = SYNC_INTERVAL;
  }
} else {
  // Нет изменений
  emptyResponsesCount++;
  
  // Если 3 раза подряд нет изменений → переключаемся на медленный интервал
  if (emptyResponsesCount >= 3 && currentInterval !== IDLE_SYNC_INTERVAL) {
    console.log('💤 Delta Sync: переключение на медленный интервал (нет активности)');
    currentInterval = IDLE_SYNC_INTERVAL;
  }
}
```

**Стратегия:**
1. **Начинаем с быстрого интервала** (10 сек) - для быстрого обнаружения изменений
2. **При отсутствии активности** (3 пустых ответа подряд) → переход на медленный интервал (30 сек)
3. **При обнаружении изменений** → возврат к быстрому интервалу (10 сек)
4. **Full Sync каждую минуту** - для обнаружения удалений другими пользователями

**Эффект:**
- ✅ Снижение нагрузки на сервер в 3 раза при отсутствии активности
- ✅ Быстрое обнаружение изменений (10 сек) при активной работе
- ✅ Автоматическая адаптация к активности пользователей
- ✅ Меньше логов в консоли при простое

## 📊 Нагрузка на сервер

### До оптимизации:
- Delta Sync: каждые 4 секунды
- **15 запросов/минуту** (постоянно)

### После оптимизации:
- Delta Sync (активный режим): каждые 10 секунд → **6 запросов/минуту**
- Delta Sync (idle режим): каждые 30 секунд → **2 запроса/минуту**
- Full Sync: каждые 60 секунд → **1 запрос/минуту**

### Экономия:
- **При активной работе**: 15 → 6 req/min (**-60%**)
- **При простое**: 15 → 2 req/min (**-87%**)
