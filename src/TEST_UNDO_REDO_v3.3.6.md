# Тестирование Undo/Redo синхронизации (v3.3.6)

## 🎯 Цель

Проверить что **измененные события** (высота, позиция, проект) больше **НЕ возвращаются** к старому состоянию после Undo/Redo.

---

## ✅ Тест-кейс 1: Изменение высоты события

### Шаги
1. Создайте событие (высота **1 unit**)
2. Измените высоту на **2 units** (resize) → **Сохраните**
3. Удалите событие → **Сохраните**
4. Нажмите **Undo** (событие должно восстановиться с высотой **1 unit**)
5. Подождите **35 секунд** (Full Sync сработает)

### Ожидаемый результат
- ✅ Событие остается с высотой **1 unit** (НЕ возвращается к 2 units)
- ✅ В консоли:
  ```
  🔄 Undo/Redo: синхронизация восстановленных событий с сервером...
  📦 BATCH: 1 create + 0 update
  ✅ BATCH CREATE: создано 1 событий на сервере
  ```

### Логи для диагностики
```javascript
console.log('🔄 Событие X не найдено на сервере, нужно создать');
console.log('📦 BATCH CREATE: подготовка 1 событий для создания...');
console.log('✅ Восстановленные события успешно синхронизированы с сервером');
```

### Шаги

### Ожидаемый результат

## ✅ Тест-кейс 4: Undo → Redo → Full Sync

### Шаги
1. Создайте событие (высота **1 unit**, неделя **10**)
2. Измените высоту на **3 units**, переместите на неделю **30** → **Сохраните**
3. Нажмите **Undo** (должно вернуться к высоте **1 unit**, неделя **10**)
4. Нажмите **Redo** (должно вернуться к высоте **3 units**, неделя **30**)
5. Подождите **35 секунд** (Full Sync сработает)

### Ожидаемый результат
- ✅ Событие остается с высотой **3 units**, неделя **30** (НЕ возвращается к 1 unit)
- ✅ В консоли (после Redo):
  ```
  ↪️ Redo: МГНОВЕННОЕ восстановление из истории
  🔄 Событие X найдено на сервере, нужно обновить
  📦 BATCH: 0 create + 1 update
  ✅ BATCH UPDATE: обновлено 1 событий на сервере
  ```

## ❌ Что НЕ должно происходить

### ❌ Full Sync возвращает старые данные
**Проблема (до v3.3.6)**:
```
1. Изменил высоту на 2 units → сохранил
2. Undo → высота вернулась к 1 unit
3. Подождал 30 секунд
4. ❌ ОШИБКА: высота вернулась к 2 units (данные с сервера)
```

**Исправлено (v3.3.6)**:
```
1. Изменил высоту на 2 units → сохранил
2. Undo → высота вернулась к 1 unit
3. ✅ МГНОВЕННО: высота обновилась на сервере (UPDATE запрос)
4. Подождал 30 секунд
5. ✅ ПРАВИЛЬНО: высота остается 1 unit (правильные данные с сервера)
```

### ❌ События пропадают после Undo
**Не должно быть**:
- События исчезают после Undo
- События дублируются после Redo
- События появляются с неправильной высотой/позицией

**Должно быть**:
- Все события на месте
- Правильная высота и позиция
- Данные синхронизированы с сервером

## 📊 Производительность

### Оптимизация batch запросов
- ✅ **1 batch запрос** вместо N последовательных UPDATE
- ✅ CREATE + UPDATE в одном запросе
- ✅ Минимальное количество данных (только измененные поля)

### Пример логов
```javascript
📦 BATCH: всего операций для отправки: 7
📦 BATCH: 2 create + 5 update
✅ BATCH CREATE: создано 2 событий на сервере
✅ BATCH UPDATE: обновлено 5 событий на сервере
📦 Восстановленные события: [...]
```

## 🎓 Что изменилось в v3.3.6

### До (v3.3.5)
```typescript
// ❌ Синхронизировались только СОЗДАНИЯ новых событий
const eventsToCreate = events.filter(e => !loadedEventIds.current.has(e.id));
if (eventsToCreate.length === 0) return; // ОШИБКА!
```

### После (v3.3.6)
```typescript
// ✅ Синхронизируются CREATE + UPDATE
const eventsToCreate: SchedulerEvent[] = [];
const eventsToUpdate: SchedulerEvent[] = [];

events.forEach(event => {
  const existsOnServer = loadedEventIds.current.has(event.id);
  if (!existsOnServer) {
    eventsToCreate.push(event); // Создать
  } else {
    eventsToUpdate.push(event); // ✅ Обновить!
  }
});

// Batch запрос: op: 'create' + op: 'update'
const operations = [
  ...eventsToCreate.map(e => ({ op: 'create', ... })),
  ...eventsToUpdate.map(e => ({ op: 'update', ... })) // ✅ КРИТИЧНО!
];
```
