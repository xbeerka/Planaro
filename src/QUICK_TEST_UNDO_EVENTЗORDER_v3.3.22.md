# 🧪 Quick Test: Undo/Redo eventZOrder Fix (v3.3.22)

**Цель**: Проверить что Undo/Redo корректно восстанавливает события с eventZOrder

---

## ✅ Тест 1: Простой Drag → Undo

### Шаги:
1. Открой календарь воркспейса с несколькими событиями
2. Перетащи любое событие в другую строку (drag)
3. Подожди 2 секунды (пока сохранится на сервере)
4. Нажми **Ctrl+Z** (Undo)

### Ожидаемый результат:
✅ Событие возвращается на исходную позицию  
✅ События НЕ удаляются  
✅ В консоли:
```
📝 pushState: добавлено в историю (past: 1, present: X событий, Y проектов, Z z-order)
🔄 UNDO: восстановлено X событий, Y проектов, Z z-order
```

❌ НЕ должно быть:
```
🔄 UNDO/REDO: ↩️ Восстановлено 0 событий, 0 проектов
🔄 UNDO/REDO: Найдено N удалённых событий
```

## 🐛 Известные проблемы (если тесты ПАДАЮТ)

### Проблема 1: События удаляются при Undo
**Причина**: История пустая или не инициализирована  
**Решение**: Проверь что в консоли есть `📝 resetHistory` после загрузки

### Проблема 2: TypeError: state.eventZOrder is undefined
**Причина**: Старая версия useOptimisticHistory без eventZOrder  
**Решение**: Обнови `/hooks/useOptimisticHistory.ts` (см. `/UNDO_REDO_EVENTЗORDER_FIX_v3.3.22.md`)

### Проблема 3: История реинициализируется при Undo
**Причина**: Сброс флага при `events.length === 0`  
**Решение**: Проверь что в `SchedulerMain.tsx` убрана проверка `events.length === 0`
