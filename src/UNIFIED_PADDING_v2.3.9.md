# 🎯 Унифицированные отступы для событий v2.3.9

## Проблема

До версии v2.3.9 одиночные события имели **разные визуальные отступы**:

```
┌─────────────────────┐
│  Проект №1          │ ← Справа: 2px отступ
│                     │
│                     │ ← Снизу: 4px отступ (gap)
└─────────────────────┘
```

**Причина**: `cellPadding = gap / 2`
- `gap = 4px` (между юнитами)
- `cellPaddingLeft = 2px`
- `cellPaddingRight = 2px`

Это создавало **визуальную асимметрию** – справа было 2px, а снизу 4px.

---

## Решение

Изменили формулу: **`cellPadding = gap`**

```
┌─────────────────────┐
│  Проект №1          │ ← Справа: 4px отступ
│                     │
│                     │ ← Снизу: 4px отступ (gap)
└─────────────────────┘
```

**Результат**: Одиночные события имеют **одинаковые отступы 4px со всех сторон** ✅

---

## Детали реализации

### До изменений (v2.3.8 и ранее)
```typescript
const cellPadding = isPerformanceMode ? 0 : gap / 2;
```

**При rowHeight = 144px:**
- `gap = 4px`
- `cellPadding = 2px`
- Визуально: справа 2px, снизу 4px ❌

---

### После изменений (v2.3.9)
```typescript
const cellPadding = isPerformanceMode ? 0 : gap;
```

**При rowHeight = 144px:**
- `gap = 4px`
- `cellPadding = 4px`
- Визуально: справа 4px, снизу 4px ✅

## Визуальные примеры

### Одиночное событие
```
     4px
   ┌────┐
4px│████│4px  ← Симметричные отступы
   │████│
   └────┘
     4px
```

### Два события разных недель
```
     4px      8px      4px
   ┌────┐           ┌────┐
4px│Ev1 │4px   4px │Ev2 │4px
   │    │           │    │
   └────┘           └────┘
     4px              4px
```

**Между событиями**: `4px (paddingRight) + 4px (paddingLeft) = 8px`

### Склеенные события (одного проекта)
```
     4px            4px
   ┌────────────────┐
4px│Event1│Event2  │4px  ← Padding убирается при склейке
   │      │        │
   └────────────────┘
        4px
```

**При склейке**: `padding = 0`, события плотно прилегают друг к другу ✅

## Режим производительности

В режиме производительности **все отступы отключены**:
```typescript

const cellPadding = isPerformanceMode ? 0 : gap;
```

**Результат**: События плотно заполняют всё доступное пространство (0px отступы)

## Дополнение: Математически правильные внутренние скругления (v2.3.10)

### Проблема
После внедрения v2.3.9 внутренние скругления были равны внешним:
```
innerRadius = borderRadius = 10px
```

Это **математически некорректно**, так как не учитывает отступ между событиями.

### Решение
**Новая формула**: `innerRadius = borderRadius + gap`

```
innerRadius = 10px + 4px = 14px
```

### Визуальное сравнение

**До (v2.3.9)**:
```
┌─────────────────┐
│ Event 1    10px │◄─ Внешний radius
│              ╭──┘
│              │ 10px ◄─ Внутренний radius (слишком острый!)
│              ╰──┐
│ Event 2        │
└────────────────┘
```

**После (v2.3.10)**:
```
┌─────────────────┐
│ Event 1    10px │◄─ Внешний radius
│            ╭────┘
│            │ 14px ◄─ Внутренний radius (компенсирует gap 4px)
│            ╰────┐
│ Event 2        │
└────────────────┘
```

### Таблица значений

| Row Height | Внешний radius | gap  | Внутренний radius | Разница |
|------------|----------------|------|-------------------|---------|
| ≤ 48px     | 4px            | 1px  | **5px**           | +1px    |
| ≤ 80px     | 6px            | 1px  | **7px**           | +1px    |
| ≤ 112px    | 8px            | 2px  | **10px**          | +2px    |
| ≥ 144px    | 10px           | 4px  | **14px**          | +4px    |

### Режим производительности
В режиме производительности всё остается 0px:
```typescript
const innerRadius = displayMode === 'performance' ? 0 : borderRadius + config.gap;
```

## Дополнение: Pixel-Perfect позиционирование (v2.3.11)

### Проблема
После внедрения v2.3.9-v2.3.10 при малых размерах (`rowHeight = 48px`) использовался `gap = 0.5px`, что создавало **дробные координаты Y**.

**Пример**:
```
gap = 0.5px
unitContentH = (48 - 3*0.5 - 2) / 4 = 11.125px ❌
unitStride = 11.625px ❌

Event unit 1: top = 11.625px ❌
Event unit 2: top = 23.25px ❌
Event unit 3: top = 34.875px ❌
```

Это приводило к субпиксельному рендерингу, события выглядели "размытыми" или смещенными.

### Решение
1. **Минимальный gap увеличен с 0.5px до 1px**
2. **Округление вычислений**: `unitContentH = Math.floor(...)`

```typescript
const unitContentH = Math.floor(
  (eventRowH - (UNITS - 1) * gap - rowPaddingTop - rowPaddingBottom) / UNITS
);
```

**Результат** при rowHeight = 48px:
```
gap = 1px
unitContentH = Math.floor((48 - 3*1 - 2) / 4) = 10px ✅
unitStride = 11px ✅

Event unit 1: top = 11px ✅
Event unit 2: top = 22px ✅
Event unit 3: top = 33px ✅
```

### Таблица точных значений

| Row Height | gap | unitContentH (до) | unitContentH (после) | unitStride (до) | unitStride (после) |
|------------|-----|-------------------|----------------------|-----------------|-------------------|
| 48px       | 0.5→1px | 11.125px ❌ | **10px** ✅ | 11.625px ❌ | **11px** ✅ |
| 80px       | 1px | 17px ✅ | **17px** ✅ | 18px ✅ | **18px** ✅ |
| 112px      | 2px | 24px ✅ | **24px** ✅ | 26px ✅ | **26px** ✅ |
| 144px      | 4px | 32px ✅ | **32px** ✅ | 36px ✅ | **36px** ✅ |

### Преимущества

✅ **Pixel-perfect рендеринг** - события всегда на целых пикселях  
✅ **Нет размытия** - субпиксельный рендеринг устранен  
✅ **Идеальное выравнивание** - события разной высоты визуально ровные  
✅ **Перфекционизм в деталях** - все координаты Y целые числа
