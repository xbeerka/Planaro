# Undo/Redo: Синхронизация измененных событий (v3.3.6)

## Проблема

### Сценарий воспроизведения
1. Пользователь изменил высоту события (unitsTall) → сохранил на сервер
2. Удалил событие → сохранил на сервер
3. Сделал **Undo** → событие восстановилось с **ПРЕЖНЕЙ** высотой (до изменения)
4. **НО!** На сервере высота осталась **ИЗМЕНЕННОЙ** (из шага 1)
5. Через **~30 секунд** Full Sync загрузил данные с сервера
6. **Результат**: высота события вернулась к измененному состоянию (до Undo)

### Корневая причина

**Функция `syncRestoredEventsToServer()`** синхронизировала только:
- ✅ Создание новых событий (которых нет на сервере)
- ❌ **НЕ** синхронизировала изменения существующих событий

```typescript
// ❌ СТАРАЯ ЛОГИКА (НЕПРАВИЛЬНАЯ)
const eventsToCreate = events.filter(event => {
  const existsOnServer = loadedEventIds.current.has(event.id);
  return !existsOnServer; // Только события которых НЕТ на сервере
});

if (eventsToCreate.length === 0) {
  console.log('✅ Все восстановленные события уже существуют на сервере');
  return; // ❌ ОШИБКА: события ЕСТЬ на сервере, но с УСТАРЕВШИМИ данными!
}
```

### Почему удаленные события работали правильно?

**Функция `syncDeletedEventsToServer()`** работала правильно:
1. Помечала удаленные события в `deletedEventIdsRef`
2. Удаляла их на сервере
3. Full Sync **НЕ** возвращал удаленные события (защита через `deletedEventIdsRef`)

### Почему измененные события НЕ работали?

**Функция `syncRestoredEventsToServer()`** НЕ обновляла существующие события:
1. Событие существовало на сервере (в `loadedEventIds`)
2. Функция пропускала его (`return !existsOnServer`)
3. Full Sync загружал устаревшие данные с сервера
4. Локальные изменения (высота, позиция) **перезаписывались**

## Решение

### ✅ Новая логика синхронизации

Разделяем события на **ДВЕ группы**:
1. **eventsToCreate** - событий НЕТ на сервере (удалены другим пользователем)
2. **eventsToUpdate** - события ЕСТЬ на сервере (но могут быть с устаревшими данными)

```typescript
// ✅ НОВАЯ ЛОГИКА (ПРАВИЛЬНАЯ)
const eventsToCreate: SchedulerEvent[] = [];
const eventsToUpdate: SchedulerEvent[] = [];

events.forEach(event => {
  const existsOnServer = loadedEventIds.current.has(event.id);
  if (!existsOnServer) {
    console.log(`🔄 Событие ${event.id} не найдено на сервере, нужно создать`);
    eventsToCreate.push(event);
  } else {
    console.log(`🔄 Событие ${event.id} найдено на сервере, нужно обновить`);
    eventsToUpdate.push(event); // ✅ КРИТИЧНО: синхронизируем изменения!
  }
});
```

### ✅ Batch операции (CREATE + UPDATE)

Отправляем **ВСЕ** операции в одном batch запросе:

```typescript
const operations: BatchOperation[] = [];

// ЧАСТЬ 1: Операции создания
if (eventsToCreate.length > 0) {
  eventsToCreate.forEach(e => {
    operations.push({
      op: 'create',
      id: e.id,
      data: {
        id: e.id,
        resourceId: e.resourceId,
        startWeek: e.startWeek,
        weeksSpan: e.weeksSpan,
        unitStart: e.unitStart,
        unitsTall: e.unitsTall, // ✅ Высота события
        projectId: e.projectId
      },
      workspace_id: workspaceId
    });
  });
}

// ЧАСТЬ 2: Операции обновления
if (eventsToUpdate.length > 0) {
  eventsToUpdate.forEach(e => {
    operations.push({
      op: 'update', // ✅ КРИТИЧНО: обновляем существующие события!
      id: e.id,
      data: {
        resourceId: e.resourceId,
        startWeek: e.startWeek,
        weeksSpan: e.weeksSpan,
        unitStart: e.unitStart,
        unitsTall: e.unitsTall, // ✅ Синхронизируем высоту!
        projectId: e.projectId
      },
      workspace_id: workspaceId
    });
  });
}

console.log(`📦 BATCH: ${eventsToCreate.length} create + ${eventsToUpdate.length} update`);
```

### ✅ Логирование

```typescript
console.log(`✅ BATCH CREATE: создано ${results.created.length} событий на сервере`);
console.log(`✅ BATCH UPDATE: обновлено ${results.updated.length} событий на сервере`);
```
