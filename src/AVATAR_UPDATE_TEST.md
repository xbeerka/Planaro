## Тестирование (5 минут)

### Шаг 1: Загрузка новой аватарки
1. Войти в систему
2. Открыть выпадающее меню профиля (правый верхний угол)
3. Нажать "Редактировать профиль"
4. Кликнуть на аватарку → выбрать новое изображение
5. Проверить что preview обновился
6. Нажать "Сохранить"

**Ожидаемое:**
- Toast: "Профиль обновлён!"
- НЕТ перезагрузки страницы (токен обновляется на лету)
- Модалка закрывается
- Аватарка в хедере обновляется мгновенно

### Шаг 2: Проверка в консоли браузера
Должны быть логи:
```
💾 Сохранение профиля...
📤 Загрузка аватара на сервер...
✅ Аватар загружен: https://...?t=1729540800000
💾 Обновление профиля на сервере...
✅ Профиль успешно обновлён
🔄 Обновление токена для получения свежих user_metadata...
✅ Новый токен получен с обновленными user_metadata
✅ Токен обновлен, все компоненты получат свежие данные
```

Перейти в: Edge Functions → make-server-73d66528 → Logs

Должны быть:

💾 Загрузка файла avatars/user_123_1729540800000.jpg в Supabase Storage...

### Шаг 4: Проверка отображения для других пользователей
1. Открыть любой воркспейс (войти в календарь)
2. Подождать 30 секунд (heartbeat)
3. Второй пользователь должен увидеть обновленную аватарку в списке онлайн пользователей в течение 15 секунд

**Проверить в логах:**
```
💓 Heartbeat от user@kode.ru в workspace 123
   user_metadata: {"name":"John Doe","avatar_url":"https://...?t=1729540800000"}
✅ Presence сохранён: presence:123:uuid-here
   avatarUrl: https://...?t=1729540800000
```

### Шаг 5: Проверка после жесткой перезагрузки
1. Нажать Ctrl+Shift+R (Windows) или Cmd+Shift+R (Mac)
2. Войти в систему снова (если вышло)
3. Проверить что аватарка сохранилась

**Проверить:**
- Аватарка в хедере
- Аватарка в списке онлайн пользователей (внутри воркспейса)
- Аватарка в карточках воркспейсов (если пользователь онлайн там)

## Troubleshooting

### ❌ Аватарка не обновилась после сохранения
**Проверить:**
1. Console → ищи `force_refresh: true` в логах
2. Network → проверь что `/auth/session` вызывается с `force_refresh: true`
3. Server logs → проверь что есть `FORCE_REFRESH` лог

**Решение:** Убедись что ProfileModal.tsx строка 177 содержит `force_refresh: true`

### ❌ Старая аватарка в кэше
**Проверить:**
1. Network → проверь URL аватарки, должен быть `?t=...` параметр
2. Storage → проверь что файл обновился в Supabase Storage

**Решение:** Жесткая перезагрузка (Ctrl+Shift+R) или очистить кэш браузера

### ❌ Аватарка не появляется у других пользователей
**Проверить:**
1. Server logs → ищи heartbeat с avatarUrl
2. Presence TTL (60 сек) - подожди минуту после закрытия календаря

**Решение:** Убедись что OnlineUsers отправляет heartbeat каждые 30 секунд

### ❌ Toast "Профиль обновлён!" но аватарка не меняется
**Проверить:**
1. Console → проверь что `handleTokenRefresh` вызывается
2. IndexedDB → проверь что `auth_access_token` обновился

**Решение:** Перезагрузи страницу и проверь логи снова

## Success criteria

✅ Аватарка обновляется мгновенно без перезагрузки страницы

✅ Heartbeat отправляет avatarUrl в presence
✅ Другие пользователи видят обновленную аватарку в течение 15 секунд
✅ После жесткой перезагрузки аватарка сохраняется
