# Release Notes v3.3.20

## 🎯 Краткое описание

Исправлены два критических бага системы Undo/Redo:
1. ✅ **Undo блокируется при pending событиях** (симметрия с Redo)
2. ✅ **Redo разблокируется после загрузки события** (обновление истории)

## 🐛 Исправленные баги

### Баг 1: Можно нажать Undo когда событие грузится

**Проблема:**
- Создал событие → сразу нажал Ctrl+Z (пока грузится)
- Undo срабатывал → событие удалялось
- createEvent завершался → событие "воскресало" из БД

**Причина:**
- В `handleUndo` отсутствовала проверка на временные ID
- В `handleRedo` эта проверка была → несимметричная логика

**Исправление:**
```typescript
// Добавлена проверка в handleUndo (строка 424-444)
const hasPendingEvents = events.some(e => e.id.startsWith('ev_temp_'));
if (hasPendingEvents) {
  console.log('⏸️ UNDO: Заблокировано - есть события в процессе создания');
  // ... toast + return
}
```

**Результат:**
- ✅ Undo блокируется при pending событиях
- ✅ Toast: "Подождите - Дождитесь завершения создания событий"
- ✅ События НЕ "воскресают" из БД

---

### Баг 2: Нельзя нажать Redo когда событие загрузилось

**Проблема:**
- Создал событие → дождался загрузки
- Ctrl+Z (Undo) → Ctrl+Y (Redo)
- Toast: "Подождите - Дождитесь завершения создания событий"
- Redo заблокирован навсегда (toast висит)

**Причина:**
- State обновлялся: `ev_temp_123` → `e12345` ✅
- История НЕ обновлялась: остаётся `ev_temp_123` ❌
- При Redo восстанавливается событие с временным ID из истории
- Проверка `events.some(e => e.id.startsWith('ev_temp_'))` → блокировка

**Исправление:**
```typescript
// В syncRestoredEventsToServer (строка 1349-1370)
if (updateHistoryEventId && results.created.length > 0) {
  console.log(`📝 История: обновление ID для ${results.created.length} созданных событий...`);
  
  const tempToRealIdMap = new Map<string, string>();
  eventsToCreate.forEach((tempEvent, index) => {
    const createdEvent = results.created[index];
    if (createdEvent) {
      tempToRealIdMap.set(tempEvent.id, createdEvent.id);
      console.log(`   ${tempEvent.id} → ${createdEvent.id}`);
    }
  });
  
  tempToRealIdMap.forEach((realId, tempId) => {
    updateHistoryEventId(tempId, realId);
  });
  
  console.log(`📝 История: обновлено ${tempToRealIdMap.size} ID`);
}
```

**Результат:**
- ✅ История обновляется после создания на сервере
- ✅ Временный ID заменяется на реальный ID во ВСЕЙ истории
- ✅ При Redo восстанавливается событие с реальным ID
- ✅ Redo НЕ блокируется (toast не показывается)

## 🔍 Технические детали

### Почему нужны обе проверки?

**hasPendingOperations()** - проверяет очередь debounced сохранений:
- Блокирует Undo/Redo если есть несохранённые изменения в очереди
- Защищает от потери изменений

**hasPendingEvents** - проверяет временные ID в state:
- Блокирует Undo/Redo если событие создаётся на сервере
- Защищает от "воскрешения" событий (баг v3.3.12)
- Предотвращает восстановление событий с временными ID

### Почему важно обновлять историю?

**Без обновления истории:**
```
Создание → ev_temp_123 (state + история)
Сохранение → e12345 (state), ev_temp_123 (история) ← ПРОБЛЕМА!
Undo → восстановление из истории
Redo → восстановление ev_temp_123 из истории ← БЛОКИРОВКА!
```

**С обновлением истории:**
```
Создание → ev_temp_123 (state + история)
Сохранение → e12345 (state + история) ← ПРАВИЛЬНО!
Undo → восстановление из истории
Redo → восстановление e12345 из истории ← РАБОТАЕТ!
```
