## 📝 Технические детали

### Мемоизация config (критично!)

**Было:**
```typescript
const config = useMemo(() => createLayoutConfig(weekPx, eventRowH), [weekPx, eventRowH]);
```

**Стало:**
```typescript
const config = useMemo(
  () => createLayoutConfig(weekPx, eventRowH, filteredDepartments, filteredResources),
  [weekPx, eventRowH, filteredDepartments, filteredResources]
);
```

**Почему важно:**
- `config` используется в `useEventInteractions`, `renderEvents`, `handleCellMouseMove`
- Без мемоизации создавался новый объект каждый рендер → ре-рендеры всех зависимых компонентов
- Теперь пересоздаётся только когда реально меняются зависимости

---

### Мемоизация renderEvents

**Было:**
```typescript
const renderEvents = () => {
  const filtered = visibleEvents.filter(...);
  const sorted = sortEvents(filtered, filteredResources);
  // ... сортировка по z-order
  return eventsWithZOrder.map(event => <SchedulerEventComponent ... />);
};
```

**Стало:**
```typescript
const sortedEventsWithZOrder = useMemo(() => {
  const filtered = visibleEvents.filter(...);
  const sorted = sortEvents(filtered, filteredResources);
  // ... сортировка по z-order
  return eventsWithZOrder;
}, [visibleEvents, filteredResources, eventZOrder]);

const renderEvents = useCallback(() => {
  return sortedEventsWithZOrder.map(event => <SchedulerEventComponent ... />);
}, [sortedEventsWithZOrder, config, projects, ...]);
```

**Почему важно:**
- Фильтрация и сортировка событий - дорогая операция
- Теперь выполняется только когда меняются события, ресурсы или z-order
- Сам рендер стабилизирован через useCallback

## 🎯 Дальнейшие возможности оптимизации

### Если нужно ещё больше производительности:

1. **Виртуализация событий** (react-window/react-virtualized):
   - Рендерить только видимые события в viewport
   - Актуально при >1000 событий на экране

2. **Web Workers для сортировки**:
   - Перенести sortEvents в отдельный поток
   - Актуально при >5000 событий

3. **Throttle для handleCellMouseMove**:
   - Сейчас вызывается на каждое движение мыши
   - Можно throttle до 60fps (16ms)

4. **Lazy loading департаментов**:
   - Загружать только развёрнутые департаменты
   - Актуально при >50 департаментах

5. **Debounce для polling**:
   - Отменять polling если пользователь активно взаимодействует с UI
   - Возобновлять через 5 сек после последнего действия
