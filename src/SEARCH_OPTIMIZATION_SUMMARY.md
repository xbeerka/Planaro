### 2. **Двусторонний поиск + Обратная транслитерация** ⭐ НОВОЕ

#### Новая возможность:
```typescript
// Теперь работает в обе стороны!
smartSearch('ЛайтФинанс', 'LiteFinance')    // ✅ true
smartSearch('KfqnAbyfyc', 'LiteFinance')    // ✅ true (неправильная раскладка)
smartSearch('LiteFinance', 'ЛайтФинанс')    // ✅ true (обратное направление)

// ✨ ГЛАВНОЕ: Обратная транслитерация (Cyrillic → Latin)
smartSearch('Фидбэк', 'Feedback')           // ✅ true (было: ❌ false)
smartSearch('Проджект', 'Project')          // ✅ true (было: ❌ false)
smartSearch('ЛайтФинанс', 'LiteFinance')    // ✅ true (было: ❌ false)
```

#### Технические детали:
- Генерируются варианты **и для запроса, и для цели**
- Проверка: каждый вариант запроса против каждого варианта цели
- Покрывает все кейсы: RU→EN, EN→RU, транслитерацию в обе стороны
- **Новая функция**: `transliterateToLatin()` для обратной транслитерации

### 3. **Производительность**

## 🆕 Что нового в v7.0.0 - АГРЕССИВНАЯ ФОНЕТИКА

### Короткие запросы теперь работают!

**Проблема:** "ла", "лай", "лайт" не находили "LiteFinance" ❌

**Корень проблемы:**
```typescript
"лайт" → transliterateToLatin → "lajt"
       → normalizePhonetic → "lait" (aj→ai)

"LiteFinance" → normalizePhonetic → "litefinance"

isFuzzyMatch("lait", "lite"):
- distance = 2 (a→i, i→e)
- maxDistance = 1 (для 4 символов)
- НЕ НАЙДЕНО ❌
```

**Решение:**
✅ **Упрощение дифтонгов в фонетике** - `ai→i`, `ei→i`, `oi→i`

**Как работает теперь:**
```typescript
"лайт" → transliterateToLatin → "lajt"
       → normalizePhonetic: 
         1. aj → ai (lajt → lait)
         2. ai → i  (lait → lit)  ← НОВОЕ!

"LiteFinance" → normalizePhonetic → "litefinance"

isFuzzyMatch("lit", "litefinance"):
- Точное вхождение: "litefinance".includes("lit") → ✅ НАЙДЕНО!
```

**Новые фонетические правила:**
```typescript
.replace(/aj/g, 'ai')    // lajt → lait (шаг 1)
.replace(/ai/g, 'i')     // lait → lit (шаг 2) ← НОВОЕ!
.replace(/ei/g, 'i')     // близко к i ← НОВОЕ!
.replace(/oi/g, 'i')     // близко к i ← НОВОЕ!
.replace(/yi/g, 'i')     // близко к i ← НОВОЕ!
```

**Результат - ЛЮБОЙ частичный ввод работает:**
```typescript
"л"    → НЕ НАЙДЕТ (< 3 символа, требуем точное совпадение)
"ла"   → НЕ НАЙДЕТ (< 3 символа, требуем точное совпадение)
"лай"  → находит "LiteFinance" ✅ (3 символа, фонетика работает!)
"лайт" → находит "LiteFinance" ✅
"фид"  → находит "Feedback" ✅
"бэк"  → находит "Feedback" ✅
"мит"  → находит "Meeting" ✅
```

**Почему это работает:**
1. Транслитерация приводит к латинице: "лай" → "laj"
2. Фонетика упрощает: "laj" → "lai" → "li"
3. Точное вхождение: "litefinance".includes("li") → ✅

**Баланс точности:**
- Короткие запросы (< 3 символа): только точное совпадение
- Средние запросы (3-4 символа): 1 ошибка + агрессивная фонетика
- Длинные запросы (5+ символов): 2-3 ошибки

## 🆕 Что нового в v6.0.0 - СОРТИРОВКА ПО РЕЛЕВАНТНОСТИ

### Проблема с ложными срабатываниями решена!

**Проблема 1:** Запрос "сбер" находил нерелевантные проекты (MosTech, Delivery, etc.) ❌  
**Проблема 2:** Нет сортировки - самые релевантные проекты терялись в списке ❌

**Решение:**
1. ✅ **Ужесточены допуски** - для 4 символов: 1 ошибка (было: 2)
2. ✅ **Новая функция `getMatchScore()`** - вычисляет релевантность совпадения
3. ✅ **Автоматическая сортировка в SimpleEventModal** - самые точные совпадения показываются первыми

**Как работает getMatchScore:**
```typescript
// Чем МЕНЬШЕ score, тем ЛУЧШЕ совпадение
getMatchScore('сбер', 'Сбер Бытовые')     → 0   (точное начало - идеально!)
getMatchScore('сбер', 'Сберегательный')   → 1   (начало с вариантами)
getMatchScore('лайт', 'LiteFinance')      → 11  (частичное + фонетика)
getMatchScore('сбер', 'MosTech')          → 1000 (нет совпадения)
```

**Приоритеты сортировки:**
1. **Score 0** - Точное начало оригинальной строки
2. **Score 1** - Точное начало с вариантами (транслитерация, смена раскладки)
3. **Score 10+** - Частичное вхождение (10 + среднее расстояние Левенштейна)
4. **Score 1000** - Нет совпадения (не показывается)

**Ужесточенные допуски:**
| Длина токена | Было | Стало | Зачем |
|--------------|------|-------|-------|
| 3 символа    | 1    | 1     | Без изменений |
| 4 символа    | 2    | **1** | ✅ Меньше ложных срабатываний |
| 5-6 символов | 2    | 2     | Без изменений |
| 7-8 символов | 3    | **2** | ✅ Меньше ложных срабатываний |
| 9+ символов  | 4    | **3** | ✅ Меньше ложных срабатываний |

**Интеграция в SimpleEventModal:**
```typescript
const filteredProjects = useMemo(() => {
  if (!searchQuery.trim()) {
    // Пустой поиск → показываем все с сортировкой по использованию
    return sortedProjects;
  }
  
  // Фильтруем + вычисляем score для каждого проекта
  const matches = sortedProjects
    .filter(project => smartSearch(searchQuery, project.name))
    .map(project => ({
      project,
      score: getMatchScore(searchQuery, project.name)
    }));
  
  // Сортируем по score (чем меньше, тем лучше)
  matches.sort((a, b) => a.score - b.score);
  
  return matches.map(m => m.project);
}, [sortedProjects, searchQuery]);
```

**Результат:**
```typescript
Запрос: "сбер"
→ Результаты (отсортированные):
   1. Сбер Бытовые        (score: 0)  ← точное начало
   2. Сберегательный банк (score: 1)  ← начало с вариантом
   3. (MosTech, Delivery - НЕ НАЙДЕНЫ благодаря ужесточению!)
```

**Преимущества:**
- ✅ Нет ложных срабатываний
- ✅ Самые релевантные проекты показываются первыми
- ✅ Интуитивный порядок результатов
- ✅ Поиск работает быстро (мемоизация через useMemo)
