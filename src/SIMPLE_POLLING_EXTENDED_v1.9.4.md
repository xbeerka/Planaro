# Simple Short Polling - Extended Auto-Update v1.9.4

## 📋 Обзор

Расширена система автообновления Simple Short Polling. Теперь приложение автоматически обновляет не только события, но и пользователей, департаменты и проекты.

## 🎯 Что было добавлено

### Автообновление всех сущностей

| Сущность | Интервал | Задержка | Ref для локальных изменений |
|----------|----------|----------|----------------------------|
| **События** | 10 сек | 0-10 сек | `lastLocalChangeRef` |
| **Сотрудники** | 15 сек | 0-15 сек | `lastResourcesChangeRef` |
| **Департаменты** | 15 сек | 0-15 сек | `lastDepartmentsChangeRef` |
| **Проекты** | 15 сек | 0-15 сек | `lastProjectsChangeRef` |

### Почему разные интервалы?

- **События (10 сек)** - меняются чаще всего (drag & drop, создание, редактирование)
- **Сотрудники, Департаменты, Проекты (15 сек)** - меняются реже, можно снизить нагрузку на сервер

## 🔧 Технические детали

### Защита от дублирования

Все CRUD операции теперь устанавливают соответствующий `ref` при изменении:

```typescript
// Пример для проектов
const createProject = useCallback(async (data: CreateProjectData) => {
  // ... создание проекта
  
  // Отмечаем что было локальное изменение
  lastProjectsChangeRef.current = Date.now();
}, [accessToken, workspaceId]);
```

### Логика polling

Каждый polling проверяет:
1. **Защита от дублирования**: было ли локальное изменение < 2 секунд назад
2. **Проверка изменений**: сравнение через `JSON.stringify`
3. **Обновление только при изменениях**: минимизация ре-рендеров
4. **Автоматическое обновление кэша**: синхронизация IndexedDB

### Пример кода (сотрудники)

```typescript
// 🔄 Simple Short Polling - автообновление сотрудников каждые 15 секунд
useEffect(() => {
  if (!accessToken || !workspaceId || isLoadingResources) return;

  console.log('🔄 Запуск автообновления сотрудников (polling каждые 15 секунд)');

  const pollResources = async () => {
    try {
      // Проверяем было ли локальное изменение в последние 2 секунды
      const timeSinceLastChange = Date.now() - lastResourcesChangeRef.current;
      if (timeSinceLastChange < 2000) {
        console.log('⏭️ Пропуск polling сотрудников - недавно было локальное изменение');
        return;
      }

      console.log('🔄 Polling: проверка обновлений сотрудников...');
      const data = await resourcesApi.getAll(accessToken, workspaceId);
      
      // Проверяем изменились ли сотрудники
      setResources(prev => {
        if (JSON.stringify(prev) !== JSON.stringify(data)) {
          console.log('✅ Сотрудники: обнаружены изменения, обновление UI');
          // Обновляем кэш
          setStorageJSON(`cache_resources_${workspaceId}`, data);
          return data;
        }
        return prev;
      });
    } catch (error) {
      console.error('❌ Ошибка polling сотрудников:', error);
    }
  };

  // Первый polling через 15 секунд после загрузки
  const timeoutId = setTimeout(pollResources, 15000);
  
  // Затем каждые 15 секунд
  const intervalId = setInterval(pollResources, 15000);

  return () => {
    clearTimeout(timeoutId);
    clearInterval(intervalId);
    console.log('🔌 Автообновление сотрудников остановлено');
  };
}, [accessToken, workspaceId, isLoadingResources]);
```

## 📊 Нагрузка на сервер

### До обновления (v1.9.3)
- Только события: 6 запросов/минуту

### После обновления (v1.9.4)
- События: 6 запросов/минуту (10 сек)
- Сотрудники: 4 запроса/минуту (15 сек)
- Департаменты: 4 запроса/минуту (15 сек)
- Проекты: 4 запроса/минуту (15 сек)

**Итого**: 18 запросов/минуту на пользователя (умеренная нагрузка)

### Оптимизации
- ✅ Пропуск polling после локальных изменений (защита от дублирования)
- ✅ Обновление только при реальных изменениях (JSON.stringify)
- ✅ Разные интервалы для разных типов данных (10/15 сек)
- ✅ Кэширование в IndexedDB (мгновенная загрузка при открытии)

## 🎨 UX эффекты

### Что увидят пользователи

1. **Автообновление списка сотрудников**
   - Пользователь A добавляет нового сотрудника → Пользователь B видит его через 0-15 секунд
   - Пользователь A редактирует имя → Пользователь B видит обновление через 0-15 секунд

2. **Автообновление департаментов**
   - Пользователь A создает новый отдел → Пользователь B видит его через 0-15 секунд
   - Пользователь A переименовывает отдел → Пользователь B видит новое название через 0-15 секунд

3. **Автообновление проектов**
   - Пользователь A создает проект → Пользователь B видит его через 0-15 секунд
   - Пользователь A меняет цвет проекта → Пользователь B видит новый цвет через 0-15 секунд

4. **События (уже работало в v1.9.3)**
   - Drag & drop события → другие видят через 0-10 секунд
   - Создание/удаление → другие видят через 0-10 секунд

### Плавность работы

- ❌ **Нет конфликтов**: polling пропускается если пользователь только что сделал изменение
- ✅ **Нет лишних обновлений**: UI обновляется только если данные реально изменились
- ✅ **Нет мерцания**: сравнение через JSON.stringify предотвращает лишние ре-рендеры
- ✅ **Мгновенный фидбек**: локальные изменения применяются мгновенно, polling только для синхронизации
